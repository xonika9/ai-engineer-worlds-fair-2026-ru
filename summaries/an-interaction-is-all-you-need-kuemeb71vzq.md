## 1. О чём доклад
Воркшоп о том, что function calling — основа почти всех современных агентных возможностей: от памяти и делегирования до асинхронных воркфлоу и мультиагентной маршрутизации. Докладчик показывает на живом коде, как из простого вызова функций собрать всё это, почти не прибегая к сложным фреймворкам.

## 2. Главные идеи
- Function calling прошёл путь от простых завершений текста до генерации структурированных действий, которые разработчик выполняет и возвращает модели.
- Агент — это цикл: получить ответ модели, обработать tool call, выполнить код, вернуть результат, повторять.
- Память, делегирование, асинхронные задачи, handoff между агентами — всё это реализуется через функции и немного состояния.
- Для real‑time нужна возможность неблокирующего вызова функций, когда модель продолжает говорить, пока задача выполняется фоном.

## 3. Практические выводы
- Пишите функции понятно, как для человека; применяйте инженерные практики, enum'ы и чёткие схемы.
- Для десятков функций лучше разбить их на группы и передавать управление между агентами (handoff/триаж), либо дообучать модель.
- Асинхронное делегирование: создавайте задачу, возвращайте ID сразу, позже проверяйте статус — так диалог не блокируется.
- Даже простейшая петля (while True + вызов API + обработка tool calls) покрывает большинство сценариев.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- OpenAI API, GPT-3.5/4/o1/o3, Real‑time API, function calling

- Pydantic AI (упомянут как альтернатива)
- Twilio (демо звонков)

- Подходы: handoff, router/triage, динамическая подгрузка функций, генерация инструментов самим агентом

## 5. Что полезно разработчику
- Базовая реализация агента умещается в ~70 строк, можно не тащить фреймворк.
- Функция `functions_to_schema` для автоматического преобразования Python‑функций в JSON‑схему tool calling.
- Примеры: хранение памяти в списке и файле, передача контекста другому агенту (handoff), асинхронный запуск фоновых задач.
- В real‑time API уже встроена поддержка асинхронных функций — можно обновлять конфигурацию и добавлять свои обработчики.

## 6. Что можно применить в агентных продуктах
- Триаж‑агент маршрутизирует пользователя к специализированным агентам (почта, календарь) — удобно при большом наборе инструментов.
- Паттерн create_task / check_task для тяжелых вычислений или обращения к медленным API без остановки диалога.
- Функция «stay silent» в голосовом агенте — модель сама решает, закончил ли пользователь мысль, не полагаясь только на VAD.
- (Осторожно) агент, который во время сессии генерирует и подключает новые функции через `exec`.
