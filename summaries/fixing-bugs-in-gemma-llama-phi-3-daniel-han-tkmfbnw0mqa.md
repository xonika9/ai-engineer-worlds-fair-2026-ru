## 1. О чём доклад
Доклад о типичных ошибках при файн-тюнинге открытых языковых моделей — Gemma, Llama 3 и Phi‑3. Основной фокус на восьми проблемах, найденных в Llama 3, которые ломают дообучение и инференс.

## 2. Главные идеи
- Двойной BOS‑токен в чат‑шаблоне Llama 3 незаметно попадает в обучение и резко снижает точность инференса.
- Спецтокены (untrained tokens) базовой модели дают NaN в градиентах, если применить instruct‑шаблон.
- Один и тот же токен‑идентификатор для pad и eos маскирует конец последовательности и вызывает бесконечную генерацию.
- Конвертация в llama.cpp / GGUF работает корректно только на CPU из‑за различий в float‑16 precision.
- Добавление системного промпта может заметно улучшить fine‑tuned модель.
- Неправильный экспорт в Ollama (несовпадение чат‑шаблона) — частая причина неработоспособности модели.

## 3. Практические выводы
- Перед дообучением проверять чат‑шаблон на лишний BOS.
- Для Llama 3 base никогда не использовать instruct‑шаблон; обучать LM head и embedding с learning rate ×0.1.
- Явно задавать pad‑токен, не совпадающий с eos.
- Конвертировать модель в GGUF только на CPU.
- Включать системный промпт в обучающие данные.
- Генерировать Modelfile для Ollama автоматически, чтобы избежать ручной подгонки шаблона.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- unsloth (опенсорс‑инструмент для файн‑тюнинга)
- Hugging Face (transformers, chat_template)
- Ollama
- llama.cpp / GGUF
- Google Colab
- QLoRA (4‑bit training, rank, alpha)
- Gradient checkpointing с оффлоадом в системную RAM
- Метод: обучение LM head и embedding‑слоёв с уменьшенной скоростью обучения

## 5. Что полезно разработчику
- Проверять token‑id и шаблон до старта обучения.
- Настраивать LoRA: rank 16–128, alpha = rank×2, обучать все линейные слои (QKV, down, up, gate).
- Держать batch size небольшим, увеличивать gradient accumulation.
- Удалять untrained‑токены при усреднении эмбеддингов, чтобы не искажать среднее.
- Использовать готовые колабы для автоматического экспорта в Ollama.

## 6. Что можно применить в агентных продуктах
- Стабильный fine‑tuning без NaN и бесконечных ответов критичен для агентов; корректные токены и шаблоны повышают надёжность.
- Системный промпт позволяет направлять поведение агента.
- Увеличение контекста через градиентный оффлоад даёт агенту работать с длинными документами без резкого роста времени.

## 7. Стоит ли смотреть полностью: yes / maybe / no
**yes** — если вы занимаетесь fine‑tuning открытых моделей, особенно Llama 3; доклад даёт конкретные ошибки и готовые приёмы.
