## О чём доклад
Доклад «Skills are the New SDKs» (Elvin Aghammadzada, DataRobot) о том, как навыки (skills) меняют разработку AI-агентов и создают новую «моду» (moat) на основе плавности опыта, а не трения. Рассматриваются ограничения контекста современных LLM, концепция контекстной инженерии, сравнение с MCP, а также растущая экосистема навыков, где навыки становятся основной единицей расширения агентов.

## Главные идеи
- *«The longer context doesn't mean better performance… after 25% usage of the context window, performance starts to degrade»* (контекстная гниль / context rot).  
- Контекст — ограниченный бюджет; важно прогрессивное раскрытие: загружать только метаданные навыка в промпт, а полное тело и скрипты — по требованию.  
- Навыки создают «экосистему плавности» (fluency moat) — вместо защитного удержания через сложность замены, компания достигает лояльности качеством опыта от намерения до результата.  
- Навыки дополняют MCP: MCP отвечает за действия и изолированные процессы, а навыки — за способ рассуждения, самокоррекцию и прогрессивную загрузку инструкций.  
- Чек-лист для enterprise-платформ пополнился «teachability» — способностью агента-навеса (harness) быстро усваивать операционные знания через навыки.

## Практические выводы
- Проектируйте навыки с тремя уровнями: frontmatter (индекс, <100 токенов), markdown-тело (~5K токенов, загружается при активации) и скрипты (исполняемые или примеры).  
- Чтобы не попасть в «dumb zone» (после 40% окна), используйте прогрессивное раскрытие и индексацию навыков.  
- Пишите навыки вручную: сгенерированные LLM навыки ухудшают производительность и тратят больше токенов.  
- Относитесь к навыкам как к софту — версионируйте, тестируйте, оценивайте.  
- Остерегайтесь prompt injection и отсутствия изоляции при выполнении навыков на машине агента.

## Упомянутые инструменты, фреймворки, компании, продукты и подходы
- DataRobot, Anthropic, Claude Code (Cloud Code), Codex, GitHub Copilot, Gemini CLI  
- Model Context Protocol (MCP)

- Реестры навыков (Claude Agent, маркетплейсы)

## Что полезно разработчику
- **AI-агенты всех типов**: прогрессивное раскрытие контекста через навыки; трёхуровневая модель загрузки.  
- **Harness**: «teachability» — новый критерий платформы: насколько легко навес осваивает домен через навыки.  
- **Sandboxes**: риск — навыки работают без изоляции (в отличие от MCP), требуют контроля.  
- **Tool execution**: исполняемые скрипты в навыке, вывод возвращается в контекст; примеры кода для обучения агента.  
- **Developer tools**: навыки как артефакты разработки — нужны версионирование, тестирование, «оценка качества».  
- **Evals**: сигнал, что LLM-генерированные навыки способны ухудшать метрики — основа для тестов.  
- **Observability**: упоминание требований к логированию и трейсингу в чеклисте платформ (косвенно).  
- **агентном продукте**: можно внедрить модель навыков с frontmatter-индексацией и прогрессивной загрузкой; измерять «teachability» как ключевую метрику опыта; контролировать бюджет контекста.

## Что можно применить в агентных продуктах
- Внедрить формат навыков с YAML/frontmatter для индексации и прогрессивной загрузки тела и скриптов, чтобы не перегружать контекст с самого начала.  
- Добавить в продукт оценку «teachability» — скорость усвоения новой доменной логики.  
- Реализовать песочницу для выполнения кода с учётом рисков, либо гибридно использовать MCP для тяжёлых/изолированных процессов.  
- Построить реестр/маркетплейс навыков с версионированием и тестами на качество.  
- Использовать идею «fluency moat» в позиционировании — опыт от намерения до результата как конкурентное преимущество.
