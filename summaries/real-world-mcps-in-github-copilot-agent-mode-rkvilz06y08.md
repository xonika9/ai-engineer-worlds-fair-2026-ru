## 1. О чём доклад

Практический обзор использования Model Context Protocol (MCP) в Agent Mode GitHub Copilot. Джон Пек из Microsoft показывает, как подключить MCP-серверы к Copilot внутри VS Code, чтобы агент получал данные из баз, автоматизировал работу с репозиторием и выполнял сквозные задачи – от прочтения спецификации до создания PR.

## 2. Главные идеи

- Agent Mode превращает Copilot из чата в автономного агента: он сам принимает решения, спрашивает разрешение на выполнение команд в терминале и может реализовать приложение по одному README.
- MCP – открытый протокол, позволяющий LLM обращаться к внешним источникам данных или сервисам. Конфигурация задаётся в настройках IDE, агент выбирает нужный сервер под задачу.
- Безопасность через явное разрешение: перед каждым вызовом MCP или терминальной команды Copilot запрашивает разрешение. Postgres MCP работает только в режиме чтения, что защищает данные.
- Использование GitHub MCP позволяет в одном промпте закоммитить изменения, создать ветку и PR – без ручного git.

## 3. Практические выводы

- Описывайте задачу максимально полно в README или copilot-instructions: структура проекта, схема БД, API, диаграммы (текстовые или изображения).
- При работе с MCP явно указывайте в промпте, какой сервер использовать (*use the Postgres MCP*), иначе агент может не догадаться.
- Добавьте в `.github/copilot-instructions.md` требование вести changelog всех действий – это даёт прозрачный лог работы агента, помимо git-истории.
- Для GitHub MCP обязательно укажите personal access token в конфигурации.
- Начинайте итеративно: первый промпт редко бывает идеальным, Agent Mode рассчитан на доработку.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Продукты и компании:** GitHub Copilot (Agent Mode), VS Code, Microsoft, GitHub Copilot Coding Agent (назначение задач Copilot’у через issues).
- **Протоколы и спецификации:** Model Context Protocol (MCP).
- **MCP-серверы:** Postgres MCP (read‑only), GitHub MCP server.
- **Файлы и конфигурации:** `.github/copilot-instructions.md`, MCP-конфигурация в `settings.json`, personal access token.
- **Подходы:** генерация mock-данных из реальной БД, управление репозиторием через MCP, использование README как спецификации.

## 5. Что полезно разработчику

- Умение быстро конфигурировать MCP через «one‑click config» в VS Code и запускать локальные серверы прямо из редактора.
- Возможность получать из базы данных реальную схему и данные для тестов, избегая ручной подготовки mock‑файлов.
- Автоматизация рутинных действий с git и GitHub (коммиты, ветки, PR) прямо внутри агентного цикла.
- Приём «changelog в инструкции» помогает при code review и отладке того, что наделал агент.

## 6. Что можно применить в агентных продуктах

- Интеграция MCP-шлюза для доступа к внутренним API, базам и сервисам компании, чтобы агент работал с актуальными корпоративными данными.
- Агентная автоматизация полного Dev‑цикла: от чтения issue / спецификации до создания протестированного и задокументированного PR с changelog.
- Использование read‑only MCP для безопасного извлечения контекста без риска мутации production‑среды.

## 7. Стоит ли смотреть полностью: maybe
