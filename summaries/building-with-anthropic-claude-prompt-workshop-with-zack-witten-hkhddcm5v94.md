## 1. О чём доклад
Живой воркшоп по промпт-инжинирингу от Зака Виттена («промпт-доктора» Anthropic). Зак итеративно улучшает промпты, присланные зрителями в Slack, прямо в консоли Anthropic и показывает работающие приёмы для Claude.

## 2. Главные идеи
- Чётко разделяйте контекст, инструкции и ввод пользователя XML-тегами; инструкции размещайте ближе к концу промпта.
- Для гарантированного вывода используйте assistant-prefill (например, открывающую скобку `{` для JSON) и stop sequences.
- Примеры (few-shot) важнее любых текстовых ограничений: один идеальный пример с контрастной парой (хорошо/плохо) даёт больше, чем длинные инструкции.
- Рассуждение (chain-of-thought) помещайте до финального ответа; для снижения галлюцинаций сначала извлеките цитаты из документа, а потом стройте суммаризацию.
- Сложную логику (переключение персонажей, оценку) выносите в код, не пытайтесь всё решить одним промптом.

## 3. Практические выводы
- Оборачивайте контекст в `<document>`, инструкции — в `<instructions>`, ответ требуйте в `<json>`, `<summary>` и т.п.
- Чтобы модель всегда отдавала JSON без лишнего текста: prefill `{` и добавьте закрывающий тег в stop sequences.
- Не пишите «будь кратким» — укажите конкретное число предложений (например, 1-2).
- Системный промпт используйте только для роли, остальное кладите в user-сообщение.
- Для задач с изображениями обрезайте лишнее, повышайте разрешение и сначала просите модель описать картинку.
- Лучше потратить время на подбор безупречных примеров, чем бесконечно шлифовать инструкции.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Инструменты/продукты:** Anthropic Console, Claude 3.5 Sonnet / Haiku, API (prefill, stop sequences), Claude.ai, Quad for Sheets.
- **Подходы:** XML-теги, few-shot prompting, chain-of-thought, negative prompting (лёгкое касание), model grading, logprobs, prefill assistant.

## 5. Что полезно разработчику
- Prefill — детерминированный способ задать начало ответа; всегда комбинируйте с тегами и stop sequences.
- Парсите ответ регулярками по тегам, не полагайтесь на «Only output JSON» в тексте промпта.
- Пишите промпты в IDE с Copilot — автодополнение ускоряет работу с вложенными тегами.
- Выносите постобработку (форматирование, извлечение) в код: это дешевле и надёжнее, чем дополнительные инструкции.

## 6. Что можно применить в агентных продуктах
- Разные персоны лучше реализовать через роутинг запроса к отдельному промпту, а не через один мега-промпт.
- Prefill + stop sequences = стабильный структурированный вывод для пайплайнов.
- Для суммаризации без выдумок: сначала модель извлекает точные цитаты, затем на их основе строит краткое резюме.
- Теги в ответе (например, `<analysis>` и `<score>`) упрощают извлечение нужных частей без лишних LLM-вызовов.
- Примеры с цепочкой рассуждений улучшают качество оценок в автоматических проверках.
