## 1. О чём доклад
Доклад раскрывает ловушку «100-инструментного агента»: загрузка полного каталога инструментов в каждый промпт ведёт к резкому падению точности, росту задержек и взрывным затратам токенов. Предлагается решение — семантическая маршрутизация с контекстной инжекцией «точно вовремя» (just-in-time), которая масштабируется на сотни инструментов без деградации.

## 2. Главные идеи
- *«At 741 tools, the accuracy will be a mere 13.6%… roughly one correct tool out of eight.»* Fat agent перегружает модель: точность падает с 78% (10 инструментов) до 13.6%.
- Семантический роутер удерживает точность выше 83% независимо от размера каталога — модель выбирает из небольшого релевантного набора.
- Контекстная инжекция сокращает токены инструментов на ~99%: с ~127k до ~1k токенов на запрос.
- *«Anthropic… reported token usage went from 150k tokens down to 2,000, which is 98.7% token reduction.»* Этот же принцип ленивой загрузки применяется к LLM-контексту.
- Задержка первого токена остаётся стабильной, тогда как у fat agent растёт с размером каталога.
- Паттерн повторяет RAG для документов: офлайн-индексация описаний инструментов, онлайн-поиск top-K, передача в модель только отобранных схем.

## 3. Практические выводы
- Маршрутизация оправдана при >50 инструментов в продакшене; для ≤20 можно обойтись прямой загрузкой.
- K=5 — сильная точка старта по умолчанию (больше — восстанавливает edge-кейсы, меньше — быстрее).
- Необходимо логирование выбора инструментов, мониторинг промахов роутера и падение на фолбек.
- Слабые описания — главный риск: их надо писать языком пользователя, включая намерение, действие и ключевые сущности.
- Редкие инструменты требуют специальной доработки описаний.
- Внедрение — это фокусированный спринт, не перестройка платформы.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Векторные БД: Chroma DB, Pinecone, Quadrant.
- Фреймворки: Anthropic MCP (on-demand tool loading), семантический роутер от Aurelio Labs, MCP Zero.
- Бенчмарки: Berkeley Function Calling Leaderboard, Toolbench, Skills bench.
- Подходы: semantic routing, just-in-time context injection, RAG для инструментов, lazy loading, nearest neighbor search.
- Компании: Prosodica (спикеры), Anthropic.

## 5. Что полезно разработчику
Прямо применимо к агентному продукту и любым агентам с большим числом инструментов (браузерные, офисные): метод семантической маршрутизации предотвращает конфузию инструментов, снижает задержки и стоимость. Для evals даны метрики (точность выбора инструмента, время первого токена, входные токены, стоимость) и наборы данных. Observability поддерживается через детальное логирование. Паттерн легко встраивается в существующую RAG-инфраструктуру, что упрощает реализацию в студийных тулчейнах.

## 6. Что можно применить в агентных продуктах
- Реализовать семантическую маршрутизацию для любых агентов студии: индексировать описания инструментов офлайн, на рантайме извлекать top‑5 по запросу, передавать модели только их.
- Использовать K=5 как базовую настройку, логировать каждый вызов и считать accuracy для тюнинга.
- Внедрить чеклист: каталог инструментов с версиями, векторный индекс, роутер, интеграция в петлю агента, оценка на тестовом сете, мониторинг.
- Улучшать описания инструментов, ориентируясь на реальные запросы пользователей, чтобы снизить риск промахов роутера.
