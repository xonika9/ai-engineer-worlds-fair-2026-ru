# User Signal Dies at the Retrieval Boundary

## 1. О чём доклад
Агенты массово проваливаются не из-за генерации, а из-за статичного retrieval (73% отказов). Сигнал от систем оценки (evals) и observability не доходит до слоя retrieval — он «умирает в дашборде». Доклад объясняет, как сделать retrieval обучаемым на исходах выполнения, чтобы агенты самоулучшались без ручного вмешательства.

## 2. Главные идеи
- Retrieval не учится на истории успехов/провалов; контекст stuffing засоряет промпт.
- Нет слоя, который бы превращал trace и eval в retrieval-руководство для будущих запусков.
- Текущие memory-системы (LangChain, Mem0) хранят факты и предпочтения, а не *reasoning*, поэтому не влияют на исходы.
- Предложена *utility score* — сходство с текущей задачей, взвешенное тем, помогала ли память раньше выполнить задачу.
- Агент учится «на лету» (Agent RX) без ретренинга и без правки промптов, обновляя навыки (skills) по мере накопления исходов.

## 3. Практические выводы
- Внедрять runtime feedback loop: observability + evals → impact on retrieval.
- Извлекать из прошлых прогонов не просто факты, а *правила-рассуждения*: *«если запрос о возврате — сначала проверь статус расчёта, чтобы избежать двойного возврата»*.
- Автоматически исключать неактуальные инструкции из системного промпта (например, устаревшие колонки в БД).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
React agent, LangChain, Mem0, Pinecone, Gartner, McKinsey, StarlightSearch, reflect, Agent RX, utility score, Tau Bench, Big Code Bench, Long TV, GPT‑5.4.

## 5. Что полезно разработчику
Utility score для переранжирования retrieval-памятей, runtime learning без дообучения модели, авто-обновляемые skills, observability, работающая как первый citizen в петле обратной связи.

## 6. Что можно применить в агентных продуктах
- Utility scoring для браузерных/офисных агентов.
- Автоматическое сохранение и переиспользование reasoning-памятей из успешных/провальных инструментальных вызовов.
- Само-обновляемые навыки агента без редактирования промпта руками.

## 7. Возможная идея для поста
«73% агентов ломаются на retrieval, потому что сигнал от оценок умирает в дашборде — как utility scoring и runtime learning решают эту проблему».
