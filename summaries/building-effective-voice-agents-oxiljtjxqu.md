## 1. О чём доклад
Обзор текущего состояния голосовых агентов: сравнение цепочечной архитектуры и нового подхода «speech-to-speech» на базе Real-time API OpenAI. Практические паттерны, компромиссы и лучшие практики при создании надёжных голосовых продуктов.

## 2. Главные идеи
- Мультимодальная эра дополняет текст: аудиомодели достигли уровня «достаточно хороши» для масштабных продакшен-приложений — быстрые, выразительные, точные.
- Speech-to-speech (Real-time API) упрощает архитектуру, снижает задержку и сохраняет семантическое понимание, в отличие от связки транскрипция + LLM + TTS.
- Ключевые компромиссы (latency, cost, accuracy, UX, integrations) диктуют выбор архитектуры: потребительским приложениям важнее скорость и UX; клиентскому сервису — точность и интеграции.
- Надёжные голосовые агенты строятся через делегирование: фронтальный голосовой агент решает простые вопросы, а через инструменты передаёт сложные задачи специализированным текстовым моделям.
- Голос означает контроль экспрессивности через промпт (тон, энтузиазм, темп). Промптинг включает пошаговые сценарии вместо абстрактных инструкций.
- Оценка качества (evals) и guardrails критичны: от наблюдаемости и человеческой разметки до синтетических разговоров; guardrails асинхронны, используют зазор между генерацией текста и озвучкой.

## 3. Практические выводы
- Для низкой задержки и выразительности выбирайте speech-to-speech; для детерминизма и интеграций — chained.
- Начинайте с минимального количества инструментов, делегируйте сложные задачи другим агентам.
- При handoff суммируйте состояние разговора, чтобы избежать потери контекста.
- Guardrails запускайте асинхронно с настраиваемым debounce (например, каждые 100 символов).
- Обязательно внедрите observability и ручную разметку данных — это самый быстрый путь к итерации промптов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- OpenAI Real-time API (новый snapshot), Agents SDK (handoffs, delegation).
- Модели: GPT-4 audio (оценка тона/темпа), GPT-4 mini, O3 mini для делегирования.
- Архитектуры: chained (transcription → LLM → TTS) и speech-to-speech.
- Компании: Lemonade (страхование), Tinder (брендовый голос в чате).

## 5. Что полезно разработчику
- Понимание компромиссов между архитектурами для разных типов продуктов.
- Паттерн делегирования через инструменты для повышения точности боевых агентов.
- Управление голосом только через промпт (выразительность, темп, эмоции).
- Асинхронные guardrails с регулируемой задержкой.
- Многоуровневая оценка: от разметки человеком до автоматических аудио-эвалов и синтетических диалогов.

## 6. Что можно применить в агентных продуктах
- Фронтальный голосовой агент (Real-time API) + инструменты, вызывающие специализированных текстовых агентов для критичных операций (возвраты, отмены).
- Детальные инструкции в промпте для формирования брендового голоса.
- Асинхронная проверка безопасности без ущерба для задержки ответа.
- Оценка тональности и темпа через аудиомодели для контроля качества речи.
- Ограничение числа инструментов у одного агента и суммаризация контекста при переключении между агентами.
