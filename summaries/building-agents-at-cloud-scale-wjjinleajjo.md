## 1. О чём доклад
Опыт и инструменты AWS для создания AI-агентов, работающих в масштабе облака (на примере Alexa Plus и Amazon Q Developer).

## 2. Главные идеи
- Model-driven подход: модели сами решают, планируют и выполняют действия, разработчик описывает **что** делать, а не **как**.
- Strands Agents (открытый Python SDK) ускоряет создание агентов до нескольких строк кода.
- MCP-серверы можно масштабировать, разворачивая их как serverless-функции (Lambda) через streamable HTTP.
- Для тысяч инструментов используется knowledge base и retrieve‑tool, чтобы не переполнять контекст модели.

## 3. Практические выводы
- Начинайте с определения промпта и инструментов, тестируйте локально, развёртывайте в облаке.
- MCP-сервер на Lambda: добавьте авторизацию, сессии в DynamoDB, используйте готовые обработчики.
- Подключайте тысячи MCP-серверов или внутренних инструментов через нативную интеграцию в Strands.
- Готовьтесь к agent‑to‑agent взаимодействиям (A2A) и мульти-агентным рабочим процессам.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Strands Agents, Amazon Bedrock, Amazon Q Developer, Alexa Plus
- MCP, A2A, FastMCP, light LLM
- AWS Lambda, DynamoDB, API Gateway, AWS SAM
- Llama, OpenAI, Meta, Anthropic (Claude 3.7 Sonnet)
- GitHub-репозиторий labmc (MCP-серверы для AWS)

## 5. Что полезно разработчику
- Быстрый старт с агентами: `pip install strands-agents`, импорт и добавление инструментов.
- 20+ готовых инструментов (файлы, API, память, RAG, multi‑agent).
- Готовый код для превращения MCP-сервера в Lambda с обработчиком сессий и авторизацией.
- Свобода выбора моделей (не только AWS).

## 6. Что можно применить в агентных продуктах
- Динамический выбор инструментов из базы знаний (retrieve) – решает проблему тысяч инструментов.
- Оркестрация специализированных экспертов (как в Alexa Plus – сотни систем).
- Быстрое прототипирование и деплой агентных сервисов (3 недели на Q‑агента для CLI).
- Мульти-агентные архитектуры: graph‑based workflows и swarms.
