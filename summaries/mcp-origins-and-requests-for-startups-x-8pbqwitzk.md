## 1. О чём доклад
Рассказ Theodora Chu (PM MCP, Anthropic) об истоках Model Context Protocol, почему его открыли, какие поворотные моменты произошли (внедрение Cursor, затем Google/Microsoft/OpenAI) и в каких направлениях стоит строить стартапы вокруг протокола.

## 2. Главные идеи
- MCP решает проблему «агентности модели»: LLM должны иметь стандартный способ взаимодействия с реальным миром, получать контекст и выполнять действия.
- Открытый протокол ускоряет экосистему, исключая необходимость отдельных партнёрств для каждой интеграции.
- *«У вас три пользователя: конечный пользователь, разработчик клиента и модель»* — сервер нужно проектировать с учётом всех трёх.
- Ставка на рост интеллекта моделей: в будущем они смогут на лету генерировать MCP-серверы; сейчас закладывается фундамент (elicitation, registry API, streamable HTTP).
- Приоритеты усилий: 80% — больше качественных серверов, 10% — упрощение сборки серверов, 10% — автоматическая генерация.

## 3. Практические выводы
- Не оборачивайте API один в один как инструменты; отталкивайтесь от реальных запросов пользователя и проектируйте инструменты под эти запросы.
- Активно используйте Inspector — самый недооценённый инструмент для отладки серверов.
- Следите за появлением реестра MCP и поддержкой elicitation — они усилят агентские сценарии.
- Открытый процесс: можно предлагать изменения и отправлять PR, участвуя в формировании стандартов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
MCP (Model Context Protocol), Anthropic, Claude, Cursor, VS Code, Sourcegraph, Google, Microsoft, OpenAI, streamable HTTP (сменил SSE), OAuth, Inspector, Registry API, GitHub-репозиторий MCP, концепция «сервер как клиент» для агентов.

## 5. Что полезно разработчику
- Как строить MCP-серверы правильно: продумывать сценарии конечного пользователя и то, как модель будет выбирать инструменты.
- Возможность создавать удалённые, а не только локальные серверы.
- Растущий запрос на инструменты для упрощения хостинга, тестирования, деплоя и отладки серверов.
- Открытая кодовая база для вклада в спецификацию и примеры.

## 6. Что можно применить в агентных продуктах
- Архитектура «сервер — тоже клиент»: агенты могут взаимодействовать друг с другом через MCP, обмениваясь запросами.
- Elicitation позволяет серверу запрашивать уточнения у пользователя (например, *«самый дешёвый или самый быстрый рейс?»*), делая агентов интерактивнее.
- Registry API даст моделям динамически находить нужные серверы без предварительной настройки.
- Автоматическая генерация серверов моделями — перспективная ниша для стартапов.
