## 1. О чём доклад
Sally Ann O’Malley из Red Hat делится опытом запуска персональных агентов OpenClaw в контейнерах — от локальной разработки до Kubernetes. Доклад-демонстрация: как безопасно управлять секретами, собирать повторяемое окружение и масштабировать агентов для команд.

## 2. Главные идеи
- Контейнеризация агентов даёт воспроизводимость, изоляцию секретов и переносимость между машинами, Mac и облачным Kubernetes.
- Двухслойная защита API-ключей: Podman/K8s secrets снаружи и OpenClaw secret refs внутри конфигурации агента.
- Подход «baseline‑образ» для рабочих мест: единый набор MCP-серверов, аутентификации и скиллов раздаётся всей команде, затем персонализируется.
- Контейнеры упрощают бэкап состояния через volumes (Podman volumes, K8s PVC).

## 3. Практические выводы
- Для запуска OpenClaw локально достаточно Podman или Docker; самодельный инсталлятор (npm run dev) ускоряет настройку.
- API-ключи стоит хранить в Podman secrets и монтировать их в контейнер, а внутри OpenClaw включать secret ref — *«я не буду показывать свои API-ключи в логах»*.
- Готовый командный образ можно собрать один раз и развернуть на всех рабочих станциях; новичок получает рабочее окружение мгновенно.
- SSH sandbox изолирует выполнение команд агента на удалённых хостах, что важно для безопасности.

## 5. Что полезно разработчику
- Контейнер избавляет от «захламления» хоста и конфликтов зависимостей.
- Инсталлятор (под Podman/Docker) автоматически создаёт OpenClaw secret refs из переданных ключей, ускоряя запуск.
- Лёгкое переключение между провайдерами (Open Router, Anthropic, локальный endpoint) без пересборки образа.

## 6. Что можно применить в агентных продуктах
- Метод baseline‑образов с утверждёнными MCP-серверами и аутентификацией — быстрый онбординг сотрудников.
- Практика двойного secret ref (внешний → внутренний) для защиты ключей в production‑подобных средах.
- Kubernetes как среда для одновременной работы десятков OpenClaw‑инстансов (пример Nvidia с model evals).
- SSH sandbox для безопасного запуска команд агента в изолированном воркспейсе.
