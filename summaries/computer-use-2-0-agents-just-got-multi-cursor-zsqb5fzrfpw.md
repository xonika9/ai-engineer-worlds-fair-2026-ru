## 1. О чём доклад
Команда Cua показывает, как за прошедший год агенты, управляющие компьютером, ушли от «старого» цикла со скриншотом и захватом всего экрана к фоновой работе. Три спикера последовательно раскрывают три слоя: драйвер, который даёт агенту «руки» в фоне на Mac, Windows и Linux; бенчмарк для проверки, что этими руками агент пользуется правильно; и инфраструктуру, которая не даёт простаивать дорогим GPU при обучении с подкреплением.

## 2. Главные идеи
- «Старый» computer use 1.0 — это человекоподобный цикл: скриншот, рассуждение, клики и ввод с захватом всего экрана; агент фактически отбирал управление у пользователя.
- Открытый драйвер (в расшифровке звучит как «cua driver») работает в фоне: он не забирает экран, а через недокументированные системные API снимает состояние конкретного окна и действует внутри него.
- Наблюдение идёт через дерево доступности плюс скриншот; действие сначала пробуется по дереву доступности, а если не выходит — выполняется фоновый клик по пикселям.
- Бенчмарк (в расшифровке — «cua bench») описывает задачу тремя частями: функция подготовки среды, эталонная траектория (oracle) и проверяющий (evaluator), причём эталон здесь — это действия в графическом интерфейсе.
- Каждой задаче доверяют только после попыток самим сломать среду и подобрать reward hacking; проходят лишь задачи, пережившие этот отбор.
- Интеллект агента измеряется не только успешностью действий, но и «моделью мира»: любой прогон можно форкнуть в любой точке и спросить у модели предсказание награды или состояния, сверив с реальным форком.

## 3. Практические выводы
- Фокус драйвера на одном окне, а не на всём рабочем столе, на бенчмарке 4K поднимает долю успешных прохождений с ~62% до ~80% при примерно на 34% меньшем расходе токенов.
- Один и тот же графический интерфейс задачи для пяти с лишним платформ можно описать в едином Python-файле через SDK бенчмарка и им же собирать пригодные данные о работе агента.
- Задачи по электротехнике (совместно со Snorkel AI) обнажают потолок: лучший агент полностью прошёл 6 из 25, все шесть — правка существующей схемы, а с чистого листа успех падает до нуля.
- При обучении с подкреплением GPU простаивает, ожидая старта или сброса песочницы; пул с автомасштабированием по спросу заранее греет нужное число песочниц, и стоимость старта уходит на дешёвую инфраструктуру, а не на дорогие GPU.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Cua (компания докладчиков), её открытый драйвер для фонового управления компьютером и бенчмарк для проверки агентов.
- Снапшоты состояния окна, деревья доступности, фоновый клик по пикселям; около восьми прикладных «харнессов» для проверки релизов на несовместимости.
- Совместный набор задач по электротехнике со Snorkel AI, где evaluator реально симулирует электрические схемы.
- Инфраструктура мгновенных песочниц (в расшифровке — «cua fleet») для Windows, Linux, Android; поддержка Mac OS заявлена как готовящаяся.
- Упомянуты Terminal-Bench и Harbor как знакомая аудитории точка отсчёта, а также ранние пользователи, присылающие правки в фреймворк.

## 5. Что полезно разработчику
Драйвер даёт способ подключить агента к операционной системе через дерево доступности и скриншоты, не отбирая экран у пользователя, что полезно при встраивании управления компьютером в собственные продукты. Бенчмарк и его SDK показывают воспроизводимый способ писать проверяемые задачи для графического интерфейса и получать из них данные для оценки агентов.

## 6. Что можно применить в агентных продуктах
- Строить фоновых агентов, работающих в рамках одного окна: это повышает надёжность и снижает расход токенов относительно захвата всего экрана.
- Оформлять задачи для оценки агента как тройку «подготовка среды — эталонная траектория — проверяющий» и прогонять их через попытки сломать среду перед включением в датасет.
- Измерять не только успешность действий, но и «модель мира» агента, форкая прогон и сверяя предсказания состояния.
- Для обучения с подкреплением закладывать пул тёплых песочниц с автомасштабированием по спросу, чтобы GPU не простаивал.
