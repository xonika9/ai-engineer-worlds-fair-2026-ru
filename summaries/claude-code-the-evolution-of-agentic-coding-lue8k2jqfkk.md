## 1. О чём доклад
Борис Черный, создатель Claude Code в Anthropic, рассказывает о быстрой эволюции моделей и о том, как команда ищет правильный UX для агентного кодинга. Ключевой тезис: модель улучшается экспоненциально, а продукт едва за ней поспевает, поэтому Claude Code сделан намеренно простым и не навязывает конкретных интерфейсов.

## 2. Главные идеи
- *«The more general model always wins»* — более общая модель всегда побеждает.
- Модель улучшается экспоненциально, и UX программирования тоже.
- Claude Code остаётся неopinionated: минимальный терминальный доступ к модели, чтобы пользователи сами находили удобные сценарии.
- Новые инструменты кодинга не только пишут код, но и способны на гораздо большее (изучение кодовой базы, планирование, использование любых CLI‑утилит).

## 3. Практические выводы
- Начинайте с codebase Q&A — это сокращает онбординг и рутину.
- Позвольте Claude освоить ваши инструменты: дайте документацию через `claude.md` или подключите через MCP.
- Давайте модели контекст: используйте `CLAUDE.md` (корень репозитория, подпапки, домашняя директория) и slash‑команды.
- Применяйте TDD: сначала тесты, потом код. Модель гораздо лучше итерирует, когда есть способ верификации (юнит‑тесты, скриншоты).
- Просите сперва исследовать и составить план, а не сразу писать код (режим plan mode по `Shift+Tab`).
- Для параллельной работы запускайте несколько терминалов или GitHub Actions; агенты могут обмениваться результатами через markdown‑файл.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Инструменты и продукты:** Claude Code (терминал, IDE‑расширение, GitHub app, SDK `claude-p`), Claude Pro / Max, GitHub Actions, MCP, `CLAUDE.md`, plan mode.
- **Исторические вехи:** Ed (первый текстовый редактор), Smalltalk‑80, Visual Basic, Eclipse, GitHub Copilot, Devin.
- **Подходы:** TDD, codebase Q&A, итеративная верификация, использование моделей как Unix‑утилит, параллельные агенты без жёсткой координации.

## 5. Что полезно разработчику
- Возможность задавать вопросы по кодовой базе и получать план перед написанием кода.
- Лёгкое обучение модели своим CLI‑инструментам и рабочим процессам.
- Быстрая итерация с автоматической проверкой результата через тесты или скриншоты.
- Простая параллелизация задач (несколько вкладок терминала, CI‑пайплайны с GitHub Actions).
- Минимальный порог входа: работает в любом терминале, включая SSH и tmux.

## 6. Что можно применить в агентных продуктах
- Минималистичный, неopinionated подход: позволить пользователю самому формировать UX.
- Предоставлять модель как Unix‑утилиту, которую можно пайпить.
- Использовать память (`CLAUDE.md`) и команды для воспроизводимых рабочих процессов.
- Встраивать plan mode, чтобы агент сначала предлагал план и ждал одобрения.
- Поддерживать параллельную работу нескольких агентов без сложной оркестрации, обмениваясь данными через файлы.
