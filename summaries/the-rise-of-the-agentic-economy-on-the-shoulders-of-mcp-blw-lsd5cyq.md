## 1. О чём доклад

Доклад о формировании агентной экономики (agentic economy) благодаря протоколу MCP (Model Context Protocol) и централизованным маркетплейсам инструментов. Проводится аналогия между зарождающимся коллективным интеллектом агентов и биологическими нейронами либо рынками: общий разум возникает из множества взаимодействующих единиц, преследующих собственные интересы. На примере платформы Apify Store показывается, как MCP и функция tool discovery позволяют агентам динамически находить, оплачивать и использовать сервисы без прямого участия человека.

## 2. Главные идеи

- *MCP уже стал стандартом для агентного взаимодействия*, обгоняя по популярности OpenAPI и Google’s A2A. Ключевая «киллер-фича» — динамическое обнаружение инструментов (tool discovery), позволяющее агенту видеть только релевантные средства в нужный момент, а не весь список из тысяч API.
- *Текущая проблема*: агенты не способны самостоятельно находить и приобретать услуги, потому что для каждого внешнего сервиса нужны отдельные токены и учётные записи, которые пока предоставляет человек-разработчик.
- *Предлагаемое решение* — централизованный маркетплейс MCP-сервисов (Apify Store), где разработчики публикуют «обёртки» над внешними API или MCP-серверами, указывают стоимость вызова и берут на себя идентификацию/оплату перед конечным сервисом. Агентам достаточно одного API-ключа и аккаунта, чтобы использовать тысячи инструментов.

## 3. Практические выводы

- Любой сторонний MCP-сервер можно опубликовать на Apify Store как Actor, даже без ведома владельца сервиса (на примере Browserbase).
- Сейчас выплаты создателям Actors превышают $4M, а общий оборот — более $0.5M в месяц; экосистема масштабируется.
- Tool discovery пока работает не идеально (нужны обходные манёвры вроде переподключения инструментов в Claude Desktop), но по мере улучшения моделей автономный подбор инструментов станет надёжным.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Apify** (actors, marketplace на 5000+ инструментов).
- **MCP** (Model Context Protocol).
- **Claude for Desktop** (Anthropic) — клиент, поддерживающий tool discovery.
- **Zapier** (MCP-сервер, доступ к 5000 приложений).
- **Browserbase** (MCP-сервер, обёрнутый как Actor без участия компании).
- **Coinbase** (стандарт x42), **Stripe**, **Mastercard**, **Visa** — в контексте попыток создать централизованные платёжные и идентификационные системы для агентов.
- **Google Trends**, **Google’s A2A** (DNS-based протокол, well-known agents.json).
- **OpenAPI**.
- **Реестры MCP-серверов** (включая «реестр реестров» от Mastra, реестр от Anthropic).

## 5. Что полезно разработчику

- Можно использовать Apify MCP server для доступа агента к 5000+ готовых веб-скрапинговых и процессинговых инструментов через единый ключ, без настройки отдельных токенов для каждого сервиса.
- Есть возможность публиковать собственные инструменты или обёртки над чужими MCP-серверами и монетизировать их на Apify Store, получая пассивный доход.
- Tool discovery избавляет от необходимости вручную прописывать и ограничивать список OpenAPI-схем в контексте агента — это снижает риск ошибок и делает систему более гибкой.

## 6. Что можно применить в агентных продуктах

- *Вложенность MCP-серверов*: один родительский сервер может предоставлять доступ к дочерним инструментам из маркетплейса, создавая многоуровневую экосистему.
- *Централизованный маркетплейс как единая точка аутентификации и биллинга*: агент с одним аккаунтом может подписываться и оплачивать десятки сервисов без решений captcha или регистрации на каждом сайте.
- *Быстрое расширение возможностей агента*: публикация коннектора к стороннему MCP-серверу без согласования с его владельцем — способ наращивать экосистему без длительных партнёрств.
