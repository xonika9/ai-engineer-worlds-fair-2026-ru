## 1. О чём доклад
Продуктовый доклад о решении проблемы сетевых изменений. Показано, как с помощью AI-агентов и графа знаний сети можно снизить количество сбоев при внесении изменений в продакшен.

## 2. Главные идеи
- Проблема: при управлении изменениями в сети часты отказы, нужен AI для снижения рисков.
- Решение из трёх компонентов: естественно-языковой интерфейс (взаимодействие с ITSM), мульти-агентная система и сетевой граф знаний (цифровой двойник).
- Граф знаний строится поверх единой схемы OpenConfig, что упрощает понимание LLM; данные из многих источников (контроллеры, телеметрия, конфиги) проходят ETL-нормализацию.
- Агенты работают с графом через специализированного query-агента, который после файнтюнинга резко сократил расход токенов и время отклика.
- Платформа агентов разрабатывается как открытый коллектив (Cisco Outshift, LangChain, Galileo и др.) с упором на интероперабельность (сайт agency.org).

## 3. Практические выводы
- Файнтюнинг агента запросов к графу дал *drastic reduction in number of tokens consumed* и ускорение.
- Слоистая архитектура графа (уровни конфигураций, control plane, data plane) позволяет агентам обращаться сразу к нужному слою без лишних вычислений.
- Оценку системы строят на extrinsic метриках, привязанных к бизнес-ценности.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- ArangoDB (выбрана для MVP), Neo4j (на исследовании), OpenConfig schema.
- Cisco Outshift, Galileo, LangChain, MCP, A2A, agency.org.
- ServiceNow (ITSM), Splunk, Batfish (инструмент тестирования конфигураций).
- React reasoning loops для агентов.

## 5. Что полезно разработчику
- Подход к построению многослойного сетевого графа знаний с единой схемой, понятной LLM.
- Опыт файнтюнинга агента запросов к графу вместо избыточного RAG-подхода для снижения токенов и задержки.
- Открытая агентная платформа с со-стандартизацией (agency.org), куда можно контрибьютить.

## 6. Что можно применить в агентных продуктах
- Интеграция агентов с ITSM (ServiceNow) через естественно-языковые обмены и автоматическую обратную фиксацию результатов.
- Цифровой двойник (граф + инструменты тестирования) для безопасной симуляции изменений до продакшена.
- Оценка качества графа через extrinsic метрики, ориентированные на пользовательскую задачу, а не на внутренние показатели графа.
