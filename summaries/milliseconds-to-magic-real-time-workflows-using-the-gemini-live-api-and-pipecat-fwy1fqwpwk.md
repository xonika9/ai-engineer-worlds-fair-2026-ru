## 1. О чём доклад

Доклад о построении голосовых AI-приложений в реальном времени с использованием Gemini Live API и фреймворка Pipecat. Спикеры (Шиста из Google и Куинн из Daily) показывают, как выглядит современный стек голосового ИИ, насколько он ещё незрел, и демонстрируют live-пример — голосового ассистента для управления списками и генерации UI.

## 2. Главные идеи

- Голос — самый естественный интерфейс, и большая часть взаимодействий с языковыми моделями будет происходить через голос.
- Выделены слои стека: LLM (DeepMind), real-time API (Gemini Live API), оркестровочные библиотеки (Pipecat) и код приложения.
- Сложные задачи (детекция окончания реплики, понимание контекста) со временем мигрируют вниз по стеку — из кода приложения во фреймворк и затем в API.
- Ни одна из проблем голосового AI не решена более чем на 50%, технология находится на раннем этапе.
- Модели всё чаще «переопределяют» логику приложения: поведение, не запланированное программистом, может оказаться удачным.

## 3. Практические выводы

- Детекция реплики должна быть настраиваемой: можно использовать встроенную на стороне сервера или внешние решения (Daily, LiveKit).
- В демо модель вела себя нестабильно: путала имена, неверно распознавала запросы, но при этом сама училась по ходу диалога, угадывая, когда нужно очищать экран.
- Голосовой агент способен динамически генерировать UI (функция `display_text`) без жёстких инструкций, полагаясь только на примеры в контексте.
- *«The way the models drive the whole application cycle is very different from my traditional programmer's view of things»* — проектирование требует иной ментальной модели.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- Gemini Live API (Google DeepMind)
- Pipecat (оркестровочный фреймворк для реального времени)
- Daily (платформа для голосовых и видео-агентов)
- LiveKit (упомянут как альтернатива для turn detection)
- Подходы: мультимодальные модели (текст, голос, изображения, видео), серверная детекция голосовой активности, динамическая генерация UI элементов в ходе диалога.

## 5. Что полезно разработчику

- Понимание того, как код приложения эволюционирует вместе с API: решение задачи turn detection сначала писалось вручную, затем ушло во фреймворк, теперь встроено в API.
- Демонстрация «хрупкости» голосовых агентов: даже без детальных промптов модель может справляться, но поведение сильно варьирует от сессии к сессии, требуется eval.
- Идея, что модель может обучаться и адаптироваться в рантайме, используя историю диалогов, иногда обходясь без код-саппорта.

## 6. Что можно применить в агентных продуктах

- Комбинирование списков, делегирование задач разным людям, контекстное уточнение запросов — всё через голос.
- Генерация простых веб-приложений или визуализаций прямо в диалоге (команда «сгенерируй приложение с Hello World»).
- Использование LLM для управления состоянием интерфейса без традиционного кода: модель сама решает, когда показать/скрыть элементы.
