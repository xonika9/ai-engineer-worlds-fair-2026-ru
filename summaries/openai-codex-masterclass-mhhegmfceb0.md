## 1. О чём доклад
Обзор OpenAI Codex — агента программной инженерии. Рассматриваются эволюция моделей, интерфейсы (приложение, IDE, CLI, Slack), плагины, автоматизации, саб‑агенты, встроенный code review и экспериментальные возможности.

## 2. Главные идеи
- Codex — не просто генератор кода, а полноценный агент: *Codex is our software engineering agent. It’s not just a coding agent… it can do everything that a software engineer would do.*
- Быстрая смена моделей (GPT‑5.2 → GPT‑5.4 и мини) и единый агентный harness постоянно улучшают поведение.
- Плагины пакуют навыки (skills), MCP‑серверы и подключения к сервисам в переиспользуемые рабочие процессы.
- Автоматизации выполняют фоновые задачи по расписанию (cron‑подобно) и экономят часы рутины.
- Саб‑агенты параллелят независимые подзадачи с индивидуальными правами доступа, моделями и инструкциями.
- Code review от Codex анализирует не только дифф, но и весь репозиторий, находя побочные эффекты.

## 3. Практические выводы
- Плагины резко сокращают ручную настройку: один бандл вместо нескольких MCP-серверов и навыков.
- Автоматизации заменяют ручную обработку почты/Slack и кроны.
- Саб‑агенты с read‑only sandbox безопасны для ревью и исследования без риска модификации кода.
- Guardian approvals снижают усталость от подтверждений, делегируя проверку привилегированных действий отдельному агенту.
- Хуки (start, after‑tool‑use, stop) внедряют кастомную логику в жизненный цикл сессии, например, принудительно продлевают долгие задачи.
- Codex способен анализировать свои прошлые сессии и предлагать новые автоматизации или саб‑агентов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
Модели: GPT‑5.2 Codex, GPT‑5.3 Codex, GPT‑5.3 Codex Spark (Cerebras), GPT‑5.4, GPT‑5.4‑mini, GPT‑5.4‑nano.  
Интерфейсы: Codex app (macOS, Windows с нативной песочницей), IDE-расширение, CLI, Slack, GitHub, веб‑интерфейс, Cloud Code plugin.  
Технологии: WebSockets, fast mode, Git work trees, план‑режим, slash‑команды.  
Плагины и интеграции: Google Drive, Slack, Gmail, Game Studio, Playwright Interactive, Imagen, Figma, Linear, Notion, Sentry (через MCP).  
Экспериментальные фичи: Guardian approvals, hooks (pre‑tool‑use, start, stop), Codex Security model, кастомные личности и инструкции.

## 5. Что полезно разработчику
- Глубокий code review, который выявляет неочевидные последствия изменений; *100% of pull requests across all OpenAI repos… are reviewed by Codex code review by default.*
- Генерация визуальных проектов с Playwright для отладки и Imagen для ассетов.
- Создание собственных саб‑агентов под конкретные задачи (тестирование, анализ уязвимостей, генерация отчётов) и выбор быстрых/лёгких моделей для простых операций.
- Анализ собственных сессий Codex для автоматического создания навыков и агентов на основе повторяющихся действий.
- Хуки для автоматизации рутин: авто‑продолжение долгих задач, документирование каждого вызова инструментов.

## 6. Что можно применить в агентных продуктах
- Ролевые саб‑агенты (worker, explorer, reviewer) с разными правами доступа (read‑only, sandbox, write).
- Плагины как способ упаковки навыков, MCP-серверов и внешних приложений в один переиспользуемый компонент.
- Guardian approvals — модель «сторожевого» агента, проверяющего опасные операции перед выполнением.
- Хуки как универсальный механизм для инъекции пользовательского поведения на ключевых этапах агентного цикла.
- Разнесение нагрузки: тяжёлые модели для сложных задач, быстрые и дешёвые — для массового параллельного исследования.
