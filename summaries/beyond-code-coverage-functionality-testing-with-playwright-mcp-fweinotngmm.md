## 1. О чём доклад
Доклад о том, как совместить TDD и AI-агентов для функционального тестирования веб-приложений через Playwright MCP. Автор показывает, почему покрытие кода юнит-тестами обманчиво, и предлагает workflow с поведенческими e2e-тестами, генерируемыми и запускаемыми с помощью Copilot и Playwright.

## 2. Главные идеи
- Рост коммитов на GitHub взрывной (1 млрд в 2025, прогноз ~14 млрд в 2026), и доля коммитов от AI-агентов растёт.
- Исследование Stanford по 120 тыс. разработчиков: AI ускоряет продуктивность только в чистых кодовых базах; в запутанных – усиливает энтропию.
- TDD обретает новый смысл: AI быстро пишет падающие тесты и код для их прохождения, а основное время разработчик тратит на рефакторинг.
- Чрезмерный фокус на покрытии юнит-тестами ведёт к тестам, привязанным к реализации, и к «самоподтверждающим» тестам от AI, не проверяющим реальное поведение.
- Playwright как e2e‑инструмент симулирует действия пользователя в браузере и позволяет тестировать именно функциональность.

## 3. Практические выводы
- Внедрять стандарты чистого кода: тестовое покрытие, типизация, документация, модульность.
- Писать тесты, отталкиваясь от feature request, а не от новых методов (триггер – фича, а не класс).
- Использовать Playwright MCP server или Playwright Agents (planner, generator, healer) для генерации, прогона и починки тестов.
- Коммитить код до того, как агент начнёт править тесты, чтобы сохранить историю.
- Добавлять скриншоты пройденных тестов в pull request.
- Генерировать один тест на одну фичу; при сложном управлении состоянием предпочитать Playwright Agents.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

## 5. Что полезно разработчику
- Понимание, что чистая кодовая база – мультипликатор выгоды от AI.
- Практический пайплайн: импорт требований через Work IQ → Copilot генерирует Playwright‑тесты → AI пишет код до зелёных тестов → разработчик рефакторит.
- Совет добавлять скриншоты тестов в PR для наглядности.
- Рекомендация всегда коммитить перед автоматическими правками агента.
- Возможность запускать тесты в headless-режиме.

## 6. Что можно применить в агентных продуктах
- **Playwright MCP server** для написания и запуска функциональных тестов прямо из агента.
- **Playwright Agents** с преднастроенными agent.md для лучшей работы с состоянием.
- Направлять агента генерировать тесты сразу при поступлении фича-реквеста, затем быстро кодить прохождение, оставляя рефакторинг человеку.
- При невозможности использовать Playwright – тестировать API напрямую.

## 7. Стоит ли смотреть полностью: maybe
Доклад даёт хороший обзор проблемы и демо Copilot + Playwright, но значительная часть времени уходит на технические настройки демо. Полезно тем, кто хочет внедрить e2e‑тестирование в агентный цикл разработки.
