## 1. О чём доклад
Инженер из Intercom рассказывает, как компания поставила цель удвоить пропускную способность (throughput) инженерной команды за год с помощью AI. Выбрав Claude Code как единую платформу, обучив его внутренним правилам и создав библиотеку навыков, они достигли удвоения количества pull requests быстрее запланированного срока.

## 2. Главные идеи
- Решительный переход на одну AI-платформу (Claude Code) и отказ от распыления на несколько инструментов.
- Видение: агент должен уметь всё, что умеет инженер, включая дебаггинг, тестирование, планирование и доступ в production.
- Создание «маховика навыков» (skills) и руководств, которые постоянно улучшаются всей командой.
- Принцип «давай агенту проблему, а не набор пошаговых задач».
- Культурный сдвиг: AI обязателен в должностных инструкциях, успехи празднуются публично, выделена full-time команда «2x».

## 3. Практические выводы
- Выберите одну платформу и оптимизируйте её, вместо использования множества агентов.
- Инвестируйте в небольшие, тестируемые и долговечные навыки, проверяя их качество на исторических данных (backtesting).
- Настройте автоматическое одобрение простых PR с высокой уверенностью, пройдя аудит для compliance.
- Подключайте агента ко всем системам, но обеспечьте контроль, аудит и разрешения.
- Постоянно повторяйте сообщение об urgency и поощряйте инженеров, делящихся успешными кейсами.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Инструменты: Claude Code (основная платформа), GitHub Copilot, Cursor, Augment, Codex (code review), Honeycomb (observability), DX (опросы разработчиков), S3 (транскрипты сессий), Slack.
- Продукты и компании: Finn (AI-агент поддержки Intercom), Anthropic, Snowflake, Linear, Glean, LaunchDarkly.
- Подходы: измерение продуктивности через code changes per R&D person, flywheel навыков, обратная связь из транскриптов, maturity rating инженеров по уровням, backtesting качества навыков.

## 5. Что полезно разработчику
- Используйте агента для любой технической работы, а не только для генерации кода: отладка, написание тестов, планирование задач.
- Автоматизируйте повторяющиеся операции и оформляйте их как навыки (skills).
- Переходите от конкретных инструкций к описанию проблемы, позволяя агенту самому выбирать нужные навыки.
- Доверяйте агенту рутинные code review – это снижает риски по сравнению с человеческим ревью при хорошей настройке.

## 6. Что можно применить в агентных продуктах
- Концепцию платформенных навыков (skills), которые инкапсулируют лучшие практики и постоянно дообучаются.
- Автоматическое ревью кода с высокой уверенностью, проверенное на исторических данных и принятое аудиторами.
- Модель взаимодействия «проблема, а не задача» – агент сам выбирает стратегию, опираясь на доступные навыки.
- Сбор транскриптов сессий для анализа эффективности навыков и улучшения промптов.
