## 1. О чём доклад

Элизабет Фуэнтес (AWS) представляет 5 техник борьбы с галлюцинациями AI-агентов. Каждая — изменение в коде, а не в промпте, и сопровождается сравнительным демо на примере туристического агента на Strands (open-source фреймворк AWS).

## 2. Главные идеи

- **Семантический отбор инструментов**: фильтрация релевантных инструментов через векторный поиск перед отправкой в контекст — токены падают с ~3000 до <300 на запрос.
- **GraphRAG (графовый RAG)**: замена текстового поиска структурированным Cypher-запросом в Neo4j — агент получает вычислимый, а не выборочный ответ для агрегаций и подсчётов.
- **Мульти-агентная валидация**: цепочка «исполнитель → валидатор → критик» через Swarm‑агента выявляет скрытые ошибки и предотвращает «успешный» ответ при сбоях.
- **Нейро-символьные стражи**: правила пишутся на Python в хуках, срабатывающих перед вызовом инструмента; модель не может их обойти, в отличие от текста в промпте.
- **Рантайм-гайденс (steering)**: мягкие правила не блокируют агента, а направляют его к самокоррекции и завершению задачи. Правила меняются на лету через локальный сервер (Agent Control).

## 3. Практические выводы

- Каждую технику можно запустить локально и затем перенести в продакшен на Amazon Bedrock Agent Core.
- Agent Core Gateway автоматически строит семантический индекс инструментов, а правила steering живут в DynamoDB без переразвёртывания.
- Для жёстких ограничений используйте хуки, для мягких — Agent Control; GraphRAG заменяет сэмплирование запросом ко всем данным.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Библиотеки/среды**: Strands Agent, OpenAI API, sentence-transformers, Faiss (векторное хранилище), Neo4j (Cypher), Swarm Agent, Agent Control SDK.
- **Облачные сервисы AWS**: Amazon Bedrock, Bedrock Agent Core, Agent Core Gateway, Lambda, DynamoDB, CloudWatch, AWS CDK.
- Amazon Bedrock Agent Core в докладе описан как специализированный сервис для продакшен-агентов: инструменты регистрируются один раз, а система подбирает нужный инструмент под конкретный запрос.

## 5. Что полезно разработчику

- Семантический отбор инструментов снижает шум и стоимость токенов — критично для агентов, работающих в песочницах или офисных плагинах.
- Мульти-агентная валидация и хуки — готовые паттерны для наблюдаемости и надёжной исполнимости инструментов.
- Рантайм-гайденс позволяет динамически менять политики без перезапуска harness’а, что удобно для экспериментов и evals.

## 6. Что можно применить в агентных продуктах

- Внедрить семантическую маршрутизацию инструментов, чтобы агент видел только релевантные сценарии.
- Подключить цепочку проверки: один агент выполняет, другой валидирует, третий утверждает — для ответственных операций (публикация постов, генерация медиа).
- Добавить хуки-стражей на Python для критичных бизнес-правил (например, лимиты на длину поста или использование определённых ИИ-моделей).
