## 1. О чём доклад
Вводный воркшоп по GraphRAG: построение ассистента для управления талантами (поиск по навыкам, формирование команд, замена сотрудников) с использованием графа навыков на Neo4j.

## 2. Главные идеи
- Граф промежуточной прослойкой между агентом и данными: декомпозиция в knowledge graph даёт больше контроля над логикой извлечения и объяснимость.
- Свойственный графу символьный (точные совпадения) и субсимвольный (векторный поиск по эмбеддингам) подходы. Комбинирование обоих через взвешенные скоринги «точное перекрытие + семантическая близость».
- Аналитика на графе (алгоритм Leiden) поверх созданных связей: детерминированная кластеризация навыков/людей в сообщества для последующего использования агентом.
- Экспертные инструменты: в агенте лучше определить узкие функции с готовыми графовыми запросами, чем полностью полагаться на динамическую text-to-cypher генерацию.

## 3. Практические выводы
- Моделируйте данные так, чтобы имена отношений читались как естественный язык (напр. «Person KNOWS Skill»), это улучшает понимае схемы моделью.
- Обязательно ставьте uniqueness constraints на узлы (email, skill name), иначе графовая БД резко замедляется.
- Если имена сущностей короткие, эмбеддинги создавайте на описаниях – они получаются информативнее.
- Материализация связей (Similar Skill Set) может ускорить повторяющиеся обходы, но при живых данных это требует поддержки.
- Визуализируйте результат векторизации: отображение графа помогает быстро оценить качество semantic similarity группировок.

## 5. Что полезно разработчику
- Графовые БД не медленные «из коробки» – главное не забывать про индексы/constraints.
- Возможность делать variable length queries (`*0..2`) для поиска пути между узлами через семантически похожие связи.
- Использование graph analytics (GDS) для автоматического обогащения данных: добавили Community ID и получили готовые кластеры для агента.

## 6. Что можно применить в агентных продуктах
- Разбивка RAG на специализированные инструменты: find_skills, find_similar_skills, find_persons_by_skills и т. д.
- Взвешенный гибридный поиск (Cypher overlap score + Neo4j Vector Index) одной функцией Custom Score.
- Простая модель Person-KNOWS-Skill естественным образом ложится на аннотированную схему для текстовой генерации запросов.
