## 1. О чём доклад
Доклад посвящён масштабированию агентов на Kubernetes с помощью протокола ACP и утилиты acpx. Автор — Onur Solmaz, контрибьютор OpenClaw — рассказывает, как стандартизировать взаимодействие клиент–агент, автоматизировать рутинные процессы ревью PR через workflow-движок и организовывать одноразовых агентов в подах Kubernetes.

## 2. Главные идеи
- ACP (Agent Client Protocol) унифицирует диалог человека с агентом, в отличие от MCP, который даёт инструменты модели.
- acpx — CLI-инструмент и «швейцарский нож» для ACP, содержащий лёгкий workflow-движок (похож на A10) для автоматизации циклов ревью, разрешения конфликтов и поверхностного рефакторинга.
- Одноразовые агенты «по требованию»: под каждый таск поднимается отдельный Kubernetes-под с полным окружением — расточительно, но даёт агенту настоящий компьютер и делает его мощнее.
- Массовый поток PR в OpenClaw требует автоматической «сортировки»: выяснение意图, оценка реализации, пропуск CI — рутину должен забирать агент, человек оставляет только фундаментальные решения.
- Личные агенты и корпоративные будут различаться: в enterprise потребление инференса и денег гораздо выше.

## 3. Практические выводы
- Механическую работу по ревью PR (проверка конфликтов, CI, поверхностные баги) можно и нужно полностью переложить на агента, запрограммировав workflow один раз.
- Для multi-agent сценариев нужна возможность динамически создавать агентов в мессенджерах, но платформы (Slack, Discord) пока не поддерживают provisioning; автор обходит это отдельным веб-UI.
- Полный Kubernetes-под в качестве песочницы агента — рабочая абстракция, несмотря на ресурсоёмкость.
- Синхронизация состояния между агентами может строиться по образцу rsync или алгоритмов Dropbox.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- OpenClaw (open source agent framework), Codex, Cloud Code, Zed.
- ACP (Agent Client Protocol), acpx, MCP.
- Kubernetes, Helm, Firecracker (упомянут в контексте OpenHands).
- TextCortex — open-source оркестратор, который использует автора на основной работе.
- Devin, rsync, алгоритм синхронизации Dropbox.

## 5. Что полезно разработчику
- acpx позволяет превратить сессию Codex в воспроизводимый пайплайн: поиск бага, оценка рефакторинга, цикл ревью с выдачей структурированного JSON.
- Workflow-движок даёт возможность описывать стандартные операционные процедуры (SOP) для агентов — это способ *«автоматизировать автоматора»*.
- Пример реального использования: обработка ошибок в продакшене через Slack → диспатч агента в Kubernetes-под для отладки.

## 6. Что можно применить в агентных продуктах
- Архитектуру с одноразовыми подами для изолированных задач (один под = одна сессия агента), поднимаемыми по helm-чартам.
- ACP-совместимые адаптеры для разных IDE и harness’ов, чтобы не дублировать интеграции.
- Концепцию «консьерж-агента» в Slack, который при необходимости порождает отдельных агентов под конкретные запросы.

## 7. Стоит ли смотреть полностью
maybe — доклад узкоспециализированный, но даёт конкретный срез решений для тех, кто строит оркестрацию агентов на Kubernetes и интересуется протоколом ACP.
