## 1. О чём доклад

Доклад о новой архитектуре AI для GTM, которая позволяет узнать покупателя и его потребности ещё до первого контакта. Спикер объясняет, почему «болтить» AI поверх старых процессов — тупиковый путь, и показывает трёхслойную архитектуру (сигналы → интеллект → действие), где AI становится ядром, объединяющим идентификацию, обогащение и персонализированные коммуникации.

## 2. Главные идеи

- Традиционные AI-чат-боты вредят, задавая вопросы «Чем я могу помочь?» — покупатель уже изучил рынок.
- *«К моменту, когда покупатель обращается к вам, решение в основном уже принято.»*
- *«80% сделок достается продавцам, которые были в списке до первого контакта с покупателем.»*
- Три слоя архитектуры: **Signals** (CRM, enrichment, LinkedIn-сигналы), **Buyer Intelligence** (база знаний, ICP scoring, контекстный граф) и **Action** (персонализированный чат, алерты, CRM-апдейты, триггеры последовательностей).
- Контекстный граф связывает каждое действие с конкретным контактом, аккаунтом и сделкой, позволяя видеть настоящий интент.
- Агенты (identification → enrichment → ICP filter → match → action) работают как конвейер в реальном времени.

## 3. Практические выводы

- Начинайте с идентификации: без устойчивого деанонимизатора GTM-проект обречён.
- Разделяйте fit-скоринг и intent-скоринг, иначе коммуникация будет нерелевантной.
- Полётный механизм: каждая отправка, ответ и закрытая сделка должны улучшать модель.
- Нужен аудируемый policy engine, чтобы GTM-лидеры могли чинить логику без разработчиков.
- При warning fatigue или долгой редактуре писем (более 30 секунд) инициатива умирает.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

HubSpot, Slack, LinkedIn как ключевой источник социальных сигналов; концепции Knowledge Base, ICP scoring, Context Graph, Alert / Routing Logic; несколько анонимайзеров для повышения точности (company ~70%, individual ~15-20%); собственное решение Position Squared — Intelligence Dashboard с разделами Anonymous Visitors, LinkedIn Intelligence, Hot Signals.

## 5. Что полезно разработчику

Подход с конвейером агентов (деанонимизация → фильтрация → поиск совпадений → действие) напрямую применим к браузерным и офисным AI-агентам. Особенно ценен наблюдаемый dashboard для observability всего конвейера: поломку отдельного агента видно и можно аудировать. Многосоставная enrichment-логика (несколько поставщиков одновременно) — хорошая схема для tool execution.

## 6. Что можно применить в агентных продуктах

Можно аналогично построить конвейер: несколько параллельных вызовов для обогащения данных → Knowledge Base для оценки ICP → контекстный граф связей → action-агент генерирует персонализированные действия. Особенно ценна идея персонального контекстного графа (persona → account → deal), который позволит AI-студии лучше таргетировать контент и избегать шаблонных диалогов.
