## 1. О чём доклад

Доклад о построении экспертных AI-систем, которые не просто извлекают данные (RAG), а умеют рассуждать и давать советы, опираясь на «мудрость» предметной области. Спикер делится опытом внедрения Knowledge Augmented Generation (KAG) на основе графов знаний и мультиагентной архитектуры, где центральный агент-координатор управляет знаниями, опытом, инсайтами и принимает решения. Приведён реальный кейс — чат-бот для конкурентного анализа на базе такого графа.

## 2. Главные идеи

- Мудрость (wisdom) — не статичный справочник, а активный центр, соединяющий знания, опыт и инсайты через обратные связи. Именно она направляет принятие решений.
- Граф знаний фиксирует таксономию мышления эксперта: знания = «что», опыт = «что работало», инсайты = «что попробовать дальше».
- KAG превосходит векторный RAG в задачах, где нужны сложные взаимосвязи, точные числовые сравнения и многошаговые запросы. Граф обеспечивает доказательное принятие решений, а не просто выдачу релевантных отрывков.
- Практичная архитектура: wisdom agent как супервизор управляет специализированными агентами, каждый из которых обновляет свою часть единого графа знаний.
- Извлечение графа лучше делать гибридно: автоматически через LLM-трансформер, затем эксперт вручную «прореживает» лишние связи.

## 3. Практические выводы

- Для систем, которые должны давать стратегические советы (например, «как обойти конкурента на текущем рынке»), одного RAG недостаточно — граф знаний даёт структуру и точность.
- Точность ответов на числовые вопросы (сравнение выручки по годам) достигается за счёт структурированных данных в графе и вызова функций, а не векторного поиска.
- Прототипирование сложных графовых пайплайнов удобно начинать в Node-RED (no-code), а затем переносить на production-стек (LangChain, Neo4j, Chroma).
- Качество растёт, если taxonomy графа построена совместно с экспертом предметной области.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- Node-RED (IBM) — визуальный инструмент для прототипирования агентных пайплайнов.
- Neo4j — графовая база данных для хранения и запросов (Cypher).
- LangChain, Chroma DB — для построения гибридных RAG+KAG систем.
- LLM-модели: OpenAI, Anthropic, on-prem модели.
- LLM Graph Transformer — автоматическое извлечение графа из текста.
- Стек LLM Graph RAG на GitHub (спонсор — Neo4j), запускаемый через Docker.

## 5. Что полезно разработчику

- Чёткое разделение: когда хватает простого RAG (Chroma + агент), а когда нужен граф знаний (сложные многозвенные запросы, количественная аналитика).
- Шаблон гибридного извлечения графа: автоизвлечение LLM → экспертная валидация и pruning.
- Пример архитектуры с супервизорным wisdom-агентом и отдельными агентами под каждый тип данных (опыт, инсайты, ситуация).
- Быстрый старт: готовый docker-стек для конвертации текста в граф и ручного прунинга.

## 6. Что можно применить в агентных продуктах

- Wisdom agent как оркестратор, распределяющий задачи между агентами-сборщиками и обновляющий центральный граф.
- Единый граф знаний как основа для принятия решений агентом-советником.
- Связка агента-стратега с агентами-аналитиками (сбор рыночных данных, анализ прошлых кампаний, мониторинг соцсетей).
- Механизм непрерывного улучшения: агенты обновляют граф, таксономия остаётся стабильной, качество ответов растёт.
