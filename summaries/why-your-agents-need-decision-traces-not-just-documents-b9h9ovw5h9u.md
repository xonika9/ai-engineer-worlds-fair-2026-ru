## 1. О чём доклад  
Доклад объясняет, почему агентам нужны не только документы и базы знаний, а контекстные графы с треками решений (decision traces). Спикер из Neo4j показывает, как такие графы помогают агентам не просто отвечать на вопросы, но принимать более точные решения с опорой на прецеденты, причинно-следственные связи и исторический контекст.

## 2. Главные идеи  
- Knowledge base помогает отвечать на вопросы, но контекстный граф добавляет следы решений, прецеденты и динамическую информацию о том, *почему* решения были приняты.  
- Контекстный граф содержит сущности, события (решения, транзакции), политики и контекст рассуждений — это позволяет агенту действовать с экспертизой.  
- Графовые эмбеддинги позволяют искать похожие треки решений не только по семантике, но и по структурному сходству графа.  
- Гибридный поиск (семантический + структурный) находит прецеденты, на основе которых агент выдаёт рекомендации.  
- Полноценная память агента требует трёх компонентов: кратковременная память (история диалога), долговременная (извлечённые сущности) и рассуждения (decision traces).

## 3. Практические выводы  
- Для улучшения качества решений агентам стоит сохранять и использовать треки предыдущих решений в виде графа.  
- Графовые эмбеддинги (через Neo4j GDS) способны вытаскивать структурно похожие прецеденты, которые сложно найти только по тексту.  
- Инструмент «create context graph» позволяет одной командой создать полный стек приложения с контекстным графом и интерфейсом, что резко ускоряет прототипирование.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- **Организации и LLM:** Neo4j, Claude, OpenAI (эмбеддинги), spaCy, gliner.  
- **Фреймворки и агентные платформы:** Pydantic AI, LangGraph, Crew, Strands, Google ADK, Microsoft Agent Framework.  
- **Технологии и продукты Neo4j:** Graph Data Science (GDS, графовые эмбеддинги), Cypher (язык запросов), Neo4j Agent Memory (пакет памяти), MCP-сервер.  
- **Коннекторы данных:** GitHub, Notion, Jira, Slack.  
- **Подходы:** контекстные графы, гибридный поиск (семантический + структурный), извлечение сущностей с многоэтапной стратегией (spaCy → gliner → LLM fallback) и дедупликация.

## 5. Что полезно разработчику  
- Скаффолдинг приложения одной командой: `uvx create-context-graph` генерирует бэкенд, фронтенд, фикстуры и графовую схему под выбранный домен (здравоохранение, FinServ или кастомный).  
- Пакет Neo4j Agent Memory даёт готовую реализацию кратковременной и долговременной памяти с извлечением сущностей и сохранением треков рассуждений.  
- Возможность импорта реальных данных из SaaS-инструментов (Slack, GitHub, Jira) вместо демо-данных.  
- Код демо и шаблоны доступны в открытом репозитории.

## 6. Что можно применить в агентных продуктах  
- Добавить слой контекстного графа поверх существующих баз знаний, чтобы агент видел не только факты, но и прошлые решения с их причинами.  
- Использовать гибридный поиск по трекам решений для автоматического подбора прецедентов и выдачи рекомендаций вида «принять / отклонить / почему».  
- Внедрить графовую память (short‑term + long‑term + reasoning) для сохранения цепочек рассуждений, что особенно полезно в системах поддержки принятия решений и финансовом анализе.
