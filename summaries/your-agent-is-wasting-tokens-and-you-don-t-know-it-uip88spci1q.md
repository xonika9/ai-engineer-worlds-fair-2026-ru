## 1. О чём доклад
Короткое выступление о пяти конкретных приёмах, позволяющих существенно снизить расход токенов при разработке агентов на больших языковых моделях.

## 2. Главные идеи
- Кэшировать системный промпт (а по возможности — промпты инструментов и сообщения), чтобы не передавать их целиком при каждом вызове.
- Маршрутизировать запросы по сложности: для простых задач задействовать дешёвую модель (например, Claude Haiku), для сложных — более мощную (Claude Sonnet); можно добавить отдельную сверхдёшевую модель-роутер.
- Выгружать большие результаты инструментов: сохранять их локально или в облаке и возвращать агенту краткое саммари, чтобы не нагружать контекст.
- Ограничивать число циклов вызова инструментов (max iterations), иначе агент рискует зациклиться и сжечь токены.
- Обрезать историю в многошаговых диалогах: скользящее окно (например, последние 10 сообщений) плюс суммаризация ранней истории.

## 3. Практические выводы
- Использовать параметр `cache_prompt=default` для системного промпта.
- Реализовать условную логику выбора модели в зависимости от сложности задачи.
- Для объёмных результатов инструментов ввести промежуточное хранение и суммаризацию.
- Явно задавать максимальное количество итераций в агенте, предварительно анализируя длительность и частоту вызовов через observability‑инструменты.
- Применять «скользящее окно» диалога (sliding window conversation manager) и периодически суммаризировать сообщения за пределами окна.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- AWS Strands agents (основной фреймворк примеров)
- Claude Haiku, Claude Sonnet
- Подходы: кэширование системных промптов, маршрутизация по сложности, offloading tool results, capping tool loops, скользящее окно истории.

## 5. Что полезно разработчику
Применимо к любым AI-агентам (браузерным, офисным): кэширование инструкций, выбор модели под задачу, контроль циклов вызова инструментов, управление контекстом. Observability напрямую упоминается как способ оценки эффективности инструментов перед деплоем. Методы можно встроить в developer tools, песочницы и системы запуска инструментов для улучшения метрик (evals) и мониторинга. Для агентном продукте — прямая экономия токенов в браузерных/офисных агентах.

## 6. Что можно применить в агентных продуктах
- Автоматическое кэширование системных инструкций.
- Роутинг с настраиваемым выбором модели (лёгкая/тяжёлая) для разных команд.
- Ограничение циклов инструментов по умолчанию.
- Скользящее окно истории с автосуммаризацией для длинных сессий.
- Встроенная панель observability: показ использованных токенов и числа итераций по каждому инструменту.
