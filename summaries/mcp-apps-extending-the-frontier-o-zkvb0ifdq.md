## 1. О чём доклад
Доклад посвящён MCP Apps — официальному расширению протокола MCP для передачи интерактивного UI от сервисов прямо в чат-интерфейсы агентов. Вместо безликих текстовых ответов компании могут отправлять собственные визуальные компоненты, сохраняя бренд и UX. Показано, как это меняет веб и открывает новый способ дистрибуции приложений через ассистентов.

## 2. Главные идеи
- Текст в чате «обезличивает» сервисы; MCP Apps даёт сервисам (Shopify, Hugging Face и др.) отправлять свой интерактивный UI.
- В основе — возврат HTML как ресурса MCP, хост рендерит его в песочнице, а UI шлёт сообщения обратно хосту (инструменты, промпты), сохраняя полный контекст.
- Это не просто технология, а новая философия: «веб без сайтов» — личный ассистент собирает атомарные UI-чунки от разных провайдеров в едином потоке.
- MCP Apps не привязан к способу генерации UI: поддерживаются предопределённое (чёрный ящик), декларативное (JSON-рендер) и генеративное (на лету моделью) — Claude использует MCP Apps под капотом для генеративного UI.
- Спектр сообщений от UI: уведомление (минимальный контроль), вызов инструмента, промпт (полный контроль хоста).
- Спецификация развивается открыто при поддержке Anthropic и OpenAI; планируются переиспользуемые представления, взаимодействие модели с UI.

## 3. Практические выводы
- Создать MCP App можно с официального SDK XApps, используя встроенные «скилы»; агент может сгенерировать приложение без ручного кода.
- Для хостов достаточно подключить React-компонент из SDK — поддержка сотен приложений из коробки.
- Единая кодовая база работает везде: Claude, ChatGPT, VS Code, Cursor, Copilot и других клиентах.
- Сообщения от UI возвращаются хосту, а не напрямую бэкенду сервиса, что сохраняет целостность пользовательского пути.
- Сообщество и рабочие группы (триweekly встречи) позволяют влиять на стандарт.

## 5. Что полезно разработчику
- XApps SDK с гарантированной совместимостью со спецификацией.
- Встроенные «скилы» позволяют создавать приложение через облачного агента без программирования.
- Простая интеграция в хост через `<MCPApp>` компонент.
- Потенциальная аудитория — миллиарды пользователей чат-платформ, что на порядки больше аудитории запуска App Store.
- Возможность добавлять интерактивность (клики, формы) с обратной связью через хост, а не напрямую к сервису.

## 6. Что можно применить в агентных продуктах
- Встраивать брендированные виджеты (календари, карты, витрины) как атомарные UI-чунки по запросу ассистента.
- Использовать спектр сообщений: уведомления для фоновых действий, вызов инструментов для бизнес-логики, промпты для делегирования контекста модели.
- Переиспользуемые представления (reusable views) для тяжелых приложений вроде Autodesk — избегать перерендера.
- Модель, взаимодействующая с UI (клики, заполнение форм) по стандартизированному протоколу (work in progress).
