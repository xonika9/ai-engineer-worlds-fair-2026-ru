## 1. О чём доклад
Доклад о построении фреймворк-фри гибридной RAG-системы для чат-бота на примере FAQ-ассистента по HR-справочнику. Рассматривается, как обойти «мультимодальный налог» — затраты токенов и потерю контроля при прямой загрузке PDF в LLM — с помощью структурированной предобработки документов через Docling, собственных стратегий чанкинга и комбинации семантического поиска с ключевыми словами. Демонстрируется полный конвейер: от загрузки файлов разных форматов (включая скриншоты) до наблюдаемости через LangFuse и гвардрейлов на уровне кода.

## 2. Главные идеи
- Предварительная конвертация документов в markdown (Docling) и управляемый чанкинг (по заголовкам, параграфам, фиксированный 512 символов с перекрытием 64, по предложениям) дают чистые данные для RAG и снижают галлюцинации.
- Гибридный поиск: векторное сходство + BM25 для точных ключевых слов, слияние через Reciprocal Rank Fusion.
- Агенты реализованы как детерминированные Python-функции, а не вызовы LLM, — быстрее и без галлюцинаций.
- Маленькая модель (Qwen 2.5 0.5B) достаточна для качественных ответов, если данные хорошо подготовлены: *«if you can vet your data before sending it to your LLM, a smaller model can sort you out»*.
- Защитные проверки (медицинские эскалации, инъекции) выполняются кодом до передачи в LLM.
- Телеметрия через LangFuse (бесплатно) даёт полную видимость цепочек и сессий.

## 3. Практические выводы
- Всегда очищайте и структурируйте документы перед индексацией; выбор стратегии чанкинга критически влияет на точность.
- Для точных ответов (SKU, лекарства) обязательно используйте гибридный поиск с BM25.
- Заменяйте агентов на простые функции, если не требуется генерация; это ускоряет ответ и исключает неопределённость.
- Гвардрейлы в коде надёжнее длинных промптов — легко тестировать и контролировать.
- Развертывание в Docker + GitHub Codespaces и локальные модели (Ollama) позволяют запустить систему без GPU и внешних зависимостей.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Docling, PostgreSQL (pgvector), Ollama, модель Qwen 2.5 (0.5B), BGE-эмбеддинги, LangFuse, FastAPI, React, Docker, GitHub Codespaces.
- BM25, Reciprocal Rank Fusion (RRF), гибридный поиск, чанкинг по заголовкам/параграфам/512 символов/предложениям.
- Подходы: «markdown first», агенты как Python-функции, ограничения через intent rejects и term dictionaries.

## 5. Что полезно разработчику: AI-агенты, браузерные и офисные агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте
- Модель агентов на Python-функциях применима для исполнения инструментов в песочницах — убирает задержку и галлюцинации.
- Схема observability с LangFuse подходит для evals и мониторинга агентных цепочек.
- Гибридный поиск и RRF можно использовать в офисных/браузерных агентах для точного извлечения из баз знаний.
- Контейнеризация всей системы облегчает развёртывание developer tools.

## 6. Что можно применить в агентных продуктах
- Добавить конвертацию в markdown и настраиваемый чанкинг для загружаемых документов.
- Внедрить гибридный поиск (вектор + BM25) с RRF для повышения релевантности.
- Использовать Python-функции вместо LLM-агентов для критичных расчётов и фильтрации.
- Реализовать ограничения на уровне кода до вызова модели.
- Подключить LangFuse-подобную телеметрию для видимости сессий и отладки.
