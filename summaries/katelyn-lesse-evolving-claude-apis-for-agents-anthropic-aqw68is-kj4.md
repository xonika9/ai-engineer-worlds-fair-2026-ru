## 1. О чём доклад
Как Anthropic развивает платформу и API Claude для создания мощных агентных систем. Доклад на примере Claude Code показывает три ключевых направления: использование возможностей модели, управление контекстным окном и предоставление модели собственного компьютера.

## 2. Главные идеи
- **Использование способностей Claude** – через API-функции: расширенные размышления с бюджетом токенов и надёжный вызов инструментов.
- **Управление контекстным окном** – критично для максимальной производительности. Включает MCP (подключение внешних систем), инструмент памяти (сохранение/возврат контекста по требованию), редактирование контекста (удаление устаревших результатов инструментов).
- **Предоставить Claude компьютер** – позволить модели писать и выполнять код в песочнице, комбинируя это с агентными навыками (agent skills) – папками скриптов и инструкций.

## 3. Практические выводы
- Настройка расширенных размышлений (token budget) улучшает решение сложных задач; для быстрых ответов лимит можно снижать.
- Контекстное окно – главный ресурс. Сочетание памяти и редактирования контекста дало +39% производительности в собственных тестах (internal evals).
- Инструмент выполнения кода снимает с разработчика заботы о песочнице и безопасности – контейнеры управляются платформой.
- Agent skills добавляют модели предметную экспертизу: Claude сам решает, когда применить навык, опираясь на запрос и описание скилла.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Claude Code (кодинг-агент)
- MCP (Model Context Protocol)
- Memory tool (клиентская файловая система для хранения контекста)
- Context editing (очистка старых результатов tool calls)
- Code execution tool (песочница для запуска кода)
- Agent skills (папки со скриптами/инструкциями)
- Расширенные размышления (extended thinking с бюджетом)
- Встроенные и кастомные инструменты (web search tool, пользовательские функции)
- Anthropic, GitHub/Sentry (примеры внешних систем через MCP)

## 5. Что полезно разработчику
- Использовать API-бюджет размышлений для контроля глубины анализа.
- Подключать любые внешние инструменты через MCP, а не только через промпт.
- Внедрять память для сохранения предпочтений, паттернов кодовой базы и git-процессов; модель сама возвращает их в нужный момент.
- Применять context editing для вытеснения нерелевантных данных из окна (особенно старых результатов инструментов) и пользоваться увеличенными контекстными окнами (до 1 млн токенов).
- Песочница выполнения кода избавляет от ручной настройки контейнеров и безопасности.

## 6. Что можно применить в агентных продуктах
- Связка MCP + agent skills: MCP даёт доступ к инструментам и контексту, навыки – знание, как их применять (например, вёрстка лендингов по дизайн-системе).
- Память агента – хранить и автоматически извлекать контекст сессий и пользовательские предпочтения.
- Автономный запуск кода в песочнице, чтобы агент сразу проверял написанное, без участия разработчика.
- Инфраструктура для сохранения сессий и оркестрации контейнеров (как в веб- и мобильной версиях Claude Code) – модель может работать асинхронно, результат будет ждать пользователя.

## 7. Стоит ли смотреть полностью: yes / maybe / no
**yes** – доклад даёт чёткую картину новейших возможностей платформы Claude для агентов, подкреплён примерами и внутренними метриками.
