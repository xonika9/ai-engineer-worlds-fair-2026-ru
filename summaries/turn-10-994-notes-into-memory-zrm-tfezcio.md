## 1. О чём доклад
Создание персональной AI Research OS, превращающей второй мозг (тысячи заметок в Obsidian, Readwise, Notion и др.) в живую вики, доступную агентам. Система позволяет не терять исследовательские материалы, а накапливать и переиспользовать знания в новых проектах.

## 2. Главные идеи
- Глобальный второй мозг + deep research loop → проектная вики как «живая память» агента.
- Три версии: статичный research.md (V1) → подключение личных источников (V2) → файловая вики с индексом без векторных БД (V3).
- Ядро V3: неизменяемые raw‑файлы, каталог `index.yaml` (метаданные и краткие резюме источников) и слой wiki‑производных (concepts, entities, comparisons, notes).
- Иерархический доступ агента ради экономии токенов: index → source wiki page (executive summary) → wiki derivatives → raw source.
- Вики эволюционирует с каждым вопросом: агент дописывает новые концепты, сравнения и заметки прямо во время диалога.
- Метод PARA (Tiago Forte) для организации заметок отделяет личную базу от автоматических изменений LLM.

## 3. Практические выводы
- Для единичных вопросов достаточно ChatGPT/поиска; для повторяющихся сложных задач нужна персональная исследовательская система.
- Файловая структура с индексом часто практичнее векторных БД для личного использования — проще инспектировать и расширять.
- Deep research лучше запускать в режимах light/fast (1–2 раунда по 3 запроса) из‑за высокого расхода токенов; deep‑режим оставлять для массового пересмотра заметок.
- Система легко дополняется новыми коннекторами одной командой агенту (Codex, Cloud Code).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
Obsidian, Readwise, Notion, Google Drive, Granola, Apple Notes, Cloud Code (Claude Code), Codex, Bright Data, Gemini (grounded in Google), NotebookLM, open‑source harnesses: open code, Pi, Hermes, метод PARA (Tiago Forte), multi‑agent deep research, index.yaml, AI Research OS (репозиторий).

## 5. Что полезно разработчику: AI-агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте
- AI-агенты и harness: подход с файловой вики и index.yaml прямо применим для лёгкой памяти агентов; анализ архитектур (open code, Pi, Hermes) дал готовые шаблоны песочниц и permission flow.
- Sandboxes & tool execution: система умеет клонировать репозитории и исследовать, как устроены песочницы, — можно повторно использовать эти «сырые» разборы.
- Developer tools: плагины для Cloud Code легко адаптируются под любой харнес; вся логика — цепочка промптов и файловых операций.
- Evals & observability: прямых упоминаний нет, но прослеживание источников и лог вопросов дают основу для аудита решений агента.
- агентном продукте: в докладе не упоминалась, однако идея легковесной индексируемой вики может заменить тяжёлые хранилища внутри студии.

## 6. Что можно применить в агентных продуктах
- Внедрить файловую вики с `index.yaml` для управления знаниями проектов — агенты получат быстрый доступ к релевантным заметкам и снизят расход контекстного окна.
- Добавить deep research loop, чтобы по теме проекта автоматически собирать и структурировать релевантные источники из личных архивов и веба.
- Использовать иерархию executive summary → wiki-деривативы → raw‑данные для пошагового углубления без перегрузки контекста.
