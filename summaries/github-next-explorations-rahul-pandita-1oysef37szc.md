## 1. О чём доклад  
Рахул Пандита (GitHub Next) рассказывает о миссии команды — исследовать будущее разработки через быстрые эксперименты. Показаны два текущих проекта: Copilot Next Edit Suggestions (умные многофайловые подсказки) и Copilot Workspace (среда, охватывающая весь внутренний цикл работы над задачей).

## 2. Главные идеи  
- Исследования и эксперименты — единственный путь к реальному внедрению AI, как в истории с электричеством (40 лет проб и ошибок).  
- Next Edit Suggestions расширяет «призрачный текст»: подсказки появляются не только у курсора, но и в других местах, где нужны согласованные правки.  
- Copilot Workspace закрывает цикл «задача → понимание → спецификация → план → код» и выступает мыслящим партнёром, а не заменой разработчика.  
- Ключ: помощь в старте, быстрая проверка (встроенный рантайм), итеративность и полный контроль человека.

## 3. Практические выводы  
- Многофайловый ghost text реален, но требует решений для невидимых правок и незакрытых файлов.  
- Инструменты должны встраиваться в естественный ход работы, давать «второй мозг» и не отбирать управление.  
- Доверие растёт, когда код можно сразу запустить (терминал, live preview) и итеративно исправить.  
- Большинство экспериментов отсеивается — это нормальная часть пути от прототипа к продукту.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- GitHub Next, Copilot, Copilot Next Edit Suggestions, Copilot Workspace  
- Copilot for CLI, Copilot Voice, GitHub Blocks, Speckle Lang  
- Шрифты Monospace  
- Внутренний цикл разработки (inner loop)  
- Методы: генерация спецификации/плана, fine‑tuning моделей, терминал на естественном языке, live preview.

## 5. Что полезно разработчику  
- Автоматические подсказки, предлагающие связанные правки при изменении одного участка (например, добавление параметра → обновление метода и документации).  
- Среда, которая по issue генерирует понятную спецификацию, план и реализацию; позволяет тут же редактировать уточнения и проверять код без выхода из инструмента.  
- Возможность начать задачу в один клик и итеративно дорабатывать, оставаясь главным.

## 6. Что можно применить в агентных продуктах  
- Многофайловая реакция на единичное действие.  
- Автоматическое построение спецификации и плана по описанию задачи с последующей реализацией и верификацией.  
- Встроенная «песочница» для проверки сгенерированного кода (терминал, предпросмотр).  
- Дизайн «агент-партнёр»: предлагает, финальное слово за человеком.
