## 1. О чём доклад
Практический GraphRAG: как с помощью графов знаний сделать большие языковые модели умнее, точнее и объяснимее. Докладчики из Neo4j разбирают, почему векторный RAG не решает проблему, показывают этапы построения графа знаний и его использования для поиска, делятся готовыми инструментами и паттернами.

## 2. Главные идеи
- Стандартные LLM не владеют доменными знаниями, галлюцинируют и не объясняют ответы.
- Векторный RAG ограничен векторным сходством, не гарантирует релевантность и плохо объясним.
- GraphRAG объединяет креативность LLM с логической структурой графа знаний (nodes, relationships, properties) — как левое и правое полушария мозга.
- Построение графа: сначала лексический граф (документы, чанки, связи), затем извлечение сущностей и отношений через LLM по заданной схеме, после — обогащение графовыми алгоритмами (PageRank, кластеризация, сообщества).
- Retrieval не сводится к векторному поиску: сначала индексный поиск (векторный/полнотекстовый) находит точки входа, затем обход связей собирает контекст; в LLM попадают не просто текстовые фрагменты, а подграф с сущностями и отношениями.
- Встроенные механизмы объяснимости: видны источники, использованные чанки и извлечённые сущности.
- Исследования подтверждают: снижение затрат токенов (Microsoft), трёхкратное повышение точности (data.world), сокращение времени решения на 28,6% (LinkedIn).

## 3. Практические выводы
- Качественный GraphRAG требует предварительной подготовки графа: структурирование документов, извлечение сущностей с явной схемой, обогащение алгоритмами.
- Использование графовой схемы при извлечении сущностей резко повышает связность и релевантность данных.
- Обход связей в графе во время retrieval даёт более полный и точный контекст, чем одиночный векторный поиск.
- Агентический подход разбивает запрос на подзадачи и вызывает специализированные ретриверы-инструменты, написанные на Cypher.
- Открытые инструменты Neo4j (Knowledge Graph Builder и GraphRAG Python package) позволяют быстро запускать пайплайны GraphRAG.

## 5. Что полезно разработчику
- Пошаговая методика построения графа знаний из PDF, веб-страниц, видео и транскриптов.
- Создание лексического графа: чанки, ссылки «следующий/предыдущий», граф сходства (KNN) для расширения контекста.
- Как задавать схему для LLM-извлечения, чтобы получать сущности и связи предсказуемого качества.
- Использование готовых open-source пакетов (graphrag, Knowledge Graph Builder) для ускорения разработки.
- Проектирование ретриверов: начинать с индексного поиска, затем обходить релевантные связи для сбора объяснимого контекста.
- Конфигурирование агентных циклов с несколькими инструментами, реализующими доменные запросы на Cypher.

## 6. Что можно применить в агентных продуктах
Агентический подход с разбиением вопроса на задачи и вызовом отдельных ретриверов‑инструментов, каждый из которых выполняет свой Cypher‑запрос и возвращает контекст. Инструменты конфигурируются с описанием и входными параметрами; агент в цикле выбирает и вызывает их, уточняя результат. Такой дизайн позволяет строить сложные диалоговые системы, извлекающие данные непосредственно из графа знаний.
