## 1. О чём доклад
Продуктовый инженер из Granola делится опытом внедрения AI-фич в десктопное приложение для заметок. Доклад не про устройство LLM, а про инженерные практики: как перестать «надеяться на one-shot» и выстроить цикл обратной связи, чтобы поведение моделей стало предсказуемым, а итерации — быстрыми.

## 2. Главные идеи
- Сделать AI-фичу «просто добавив чат» недостаточно: пользователи начинают применять её не по сценарию, а стоимость и качество быстро уходят из-под контроля.
- LLM — чёрный ящик, который требует собственных инструментов наблюдаемости. Готовые SaaS-решения не всегда дают нужную глубину, но теперь построить своё трассировочное средство стало реально.
- Один промпт не может одинаково хорошо обслуживать продажи, инженеров и HR — нужно адаптировать вывод под роль, а для этого надо видеть, где агент ошибается.
- Скорость проверки гипотез критична: превращение Electron-приложения в веб-песочницу с превью-ссылками на каждый PR и автоматическими скриншотами от AI-агентов радикально ускоряет тестирование вариантов.
- Ключевой вывод: магия продукта рождается не из «улучшения one-shot», а из быстрой обратной связи — спикер сравнивает это с игрой в теннис, где вы постоянно отвечаете на действия модели.

## 3. Практические выводы
- Стройте собственные tracing-инструменты поверх AI SDK с сохранением в БД и удобным UI для всех ролей (инженеры, продукт, поддержка). Это даёт полную видимость цепочки tool calls, причин решений и затрат.
- Абстрагируйте IPC Electron-приложения до веб-стандартов, чтобы запускать фронтенд как веб-приложение и получать preview-ссылки на каждый PR. Это снижает трение при тестировании десктопных фич.
- Настройте CI так, чтобы AI-помощник (например, Cursor) мог автоматически проверять PR и прикреплять скриншоты — это ускоряет приёмку изменений.
- Не доверяйте «одной строке кода» для веб-поиска в LLM: контролируйте выбор провайдера, следите за токенами и стоимостью.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Продукты:** Granola (десктопное приложение для заметок с транскрипцией), Cursor (AI-среда для тестирования PR), CloudWatch.
- **Фреймворки и технологии:** Electron, AI SDK (универсальная обёртка для LLM), OpenTelemetry, React, веб-стандарты (IPC abstraction).
- **Подходы:** собственный tracing UI для LLM-пайплайнов, веб-шелл для десктопного приложения, автоматические скриншоты в PR через AI.

## 5. Что полезно разработчику
- Методика перевода десктопного Electron-приложения в гибридный режим «веб-песочница» для параллельного тестирования вариантов.
- Схема построения внутреннего tracing-инструмента без внешних SaaS: обёртка над AI SDK, хранение вызовов в собственной БД и кастомный UI.
- Понимание ограничений «магических» one-shot решений: внедрение веб-поиска, чата или суммаризации требует постоянного мониторинга и итераций.

## 6. Что можно применить в агентных продуктах
- Инструменты полной наблюдаемости за агентным циклом: визуализация вызовов инструментов, причины решений, затраты — доступные не только инженерам, но и команде продукта.
- Адаптация вывода агента под роли пользователей через итеративное тестирование множества вариантов, а не через один универсальный промпт.
- Инфраструктура для быстрой обратной связи: preview-окружения, в которых агент может самопроверяться и отчитываться скриншотами.
- Подход «игра с LLM в теннис» — быстрые ответы на поведение модели вместо попыток запрограммировать идеальный one-shot.

## 7. Стоит ли смотреть полностью: yes / maybe / no
**yes** — короткий доклад с конкретными инженерными приёмами, особенно полезен тем, кто встраивает AI в десктопные приложения или строит агентов.

## 8. Самые важные таймкоды
- [0:01:52–0:02:48] Пример generic AI-чата: пользователи уводят сценарий, проявляются проблемы масштабирования и контроля.
- [0:02:49–0:03:59] Почему веб-поиск в LLM — не одна строка кода, а токены, стоимость и нестабильность провайдера.
- [0:04:39–0:05:59] Создание собственного tracing-инструмента: полная видимость tool calls, UI для всей команды.
- [0:06:45–0:08:21] Превращение Electron-приложения в веб-шелл и автоматические скриншоты от AI на PR.
- [0:08:44–0:09:05] Главный тезис: one-shot — не ответ, важна быстрая обратная связь как игра в теннис.
