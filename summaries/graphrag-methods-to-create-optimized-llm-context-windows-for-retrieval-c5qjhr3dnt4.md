## 1. О чём доклад

Джонатан Ларсон (Microsoft Research) показывает, как GraphRAG применяется к анализу кода (GraphRAG for Code), анонсирует новый открытый бенчмарк Benchmark QED и представляет технологию lazy graph, улучшающую retrieval даже на коротких контекстных окнах.

## 2. Главные идеи

- LLM-память со структурой — ключевой фактор создания эффективных AI-приложений.
- Агенты в сочетании с такими структурами дают значительно более мощные результаты.

## 3. Практические выводы

- GraphRAG for Code даёт семантическое понимание репозитория: точное описание логики игры, успешный перевод с Python на Rust и генерация документации для кодовой базы объёмом 100 000 строк.
- В задаче многолфайловой модификации (добавление прыжка в Doom) агент GitHub Copilot, подключённый к GraphRAG, справился успешно — обычные агенты с этой задачей не справлялись.
- Lazy graph доминирует над векторным RAG на контекстных окнах 8k, 120k и 1 млн токенов как по локальным, так и по глобальным вопросам, и при этом в 10 раз дешевле последнего.
- Benchmark QED позволяет генерировать вопросы по всему спектру «локальные–глобальные» и «data-driven–activity-driven», а затем оценивать системы по метрикам comprehensiveness, diversity, empowerment и relevance.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **GraphRAG** (paper, GitHub), **GraphRAG for Code**
- **GitHub Copilot coding agent** (анонсирован на Build)
- **Benchmark QED** (autoQ, autoE, autoD) — открытый исходный код
- **Lazy graph** — технология улучшенного retrieval
- **Azure Local** – платформа, в которую встроят lazy graph
- **Microsoft Discovery** – платформа научного со-рассуждения на графах
- **Microsoft Research** – компания-разработчик

## 5. Что полезно разработчику

- Использование GraphRAG радикально повышает качество понимания кода, перевода между языками и генерации документации.
- Готовый открытый бенчмарк QED для оценки retrieval-систем в своём проекте.
- Интеграция с Copilot открывает путь к надёжным многофайловым рефакторингам.

## 6. Что можно применить в агентных продуктах

Агент, оснащённый GraphRAG, выполняет сложные мультифайловые изменения, которые не удавались обычным агентам. Lazy graph может быть встроен как дешёвый и качественный retrieval-слой для агентов, требующих глобального понимания данных.
