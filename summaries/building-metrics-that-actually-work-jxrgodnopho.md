## 1. О чём доклад
Доклад посвящён построению метрик и систем оценки (evals) для AI-приложений, которые действительно работают. Авторы делятся методологией, вынесенной из многолетнего опыта в Google (поиск), и демонстрируют практический подход через hands-on workshop. Основной фокус — как перейти от ad-hoc проверок к масштабируемой, декомпозированной скоринговой системе, способной жить в production и коррелировать с реальными пользовательскими сигналами.

## 2. Главные идеи
- Эвалюация — не просто тестирование, а главное место, где живут доменные знания. Хорошие метрики радикально упрощают остальной стек (промпты, файн-тюнинг, RL).
- Не существует одной «правильной» метрики. Лучший путь — построение **scoring system**: множество простых, объективных сигналов, объединяемых в единую оценку (подобно сотням сигналов в Google Search).
- Итеративность: начинать с vibe testing, добавлять код-метрики, затем LLM-as-judge, но в итоге разбивать субъективные оценки на конкретные измеримые вопросы.
- Калибровка критична: метрика должна коррелировать с реальными данными (thumbs up/down), иначе она бесполезна.
- Специализированные модели (bidirectional attention, regression head) дают низкую вариативность, высокую точность и скорость (до 50 мс на десятки измерений), что позволяет использовать скоринг онлайн.

## 3. Практические выводы
- Создавайте scoring spec с явными вопросами и весами (critical/major/minor) вместо одного «размытого» промпта к LLM.
- Используйте co-pilot для генерации начального набора измерений и итеративного улучшения через синтетические примеры.
- Интегрируйте скоринг в Google Sheets для быстрой проверки на реальных датасетах и анализа confusion matrix.
- Применяйте скоринг для сравнения моделей, промптов и для выбора лучшего ответа среди нескольких семплов (best‑of‑N) в онлайне.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Подход: scoring system (декомпозиция на сигналы, калибровка, математическая комбинация).
- Продукт: Pi Labs (scoring models, copilot, интеграция с Google Sheets, Python SDK).
- Технологии: bidirectional‑attention модели с regression‑головой, generalized additive model для объединения сигналов.
- Платформы/инструменты: Google Sheets, Google Colab, Hugging Face (датасеты).
- Модели: Gemini 1.5/2.5, Claude Haiku.
- DSPy (оптимизатор промптов, упомянут в контексте автоматического улучшения).
- Компания: Google (упоминается как источник опыта).

## 5. Что полезно разработчику
- Старт с чёткого system prompt и vibe‑тестирования, затем постепенное наслаивание метрик.
- Разбиение оценки на конкретные вопросы («содержит ли вывод все action items?») даёт прозрачность и снижает вариативность.
- Возможность быстро протестировать метрики в spreadsheet‑среде без развёртывания кода.
- Скоринг как инструмент мониторинга регрессий при смене промптов или моделей.

## 6. Что можно применить в агентных продуктах
- Онлайн‑ранжирование: при высокой температуре генерируется несколько ответов агента, скоринг выбирает лучший — рост качества без изменения модели.
- Быстрые (<50 мс) проверки позволяют встраивать оценку в критические пути агента (например, валидация выходного JSON, проверка полноты ответа).
- Единая скоринговая система становится источником сигнала для автоматической оптимизации (meta‑prompts, RL) и защиты от деградаций в многошаговых сценариях.
