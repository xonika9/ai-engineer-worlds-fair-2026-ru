## 1. О чём доклад
Доклад о том, как с помощью MCP и инфраструктуры Bright Data строить масштабируемые пайплайны сбора данных. Вместо прямого парсинга каждой страницы через LLM предлагается поручать агенту создавать скрапер, который затем дёшево извлекает данные, экономит токены и способен самовосстанавливаться при изменениях на сайте.

## 2. Главные идеи
- Эффективнее не парсить HTML напрямую LLM, а попросить агента *написать скрапер*, используя навыки и MCP; скрапер многократно дешевле в токенах.
- MCP Bright Data даёт агенту 66 инструментов: curl с обходом блокировок (CAPTCHA, Cloudflare), извлечение HTML или упрощённого markdown, готовые API для 500+ доменов, инфраструктуру удалённых браузеров.
- Самоисцеляющиеся пайплайны: агент может по расписанию проверять данные и чинить сломавшийся скрапер за минуты.
- Инфраструктура из 150+ миллионов IP имитирует поведение реального человека (движения мыши, набор текста), обеспечивая доступ к самым защищённым сайтам.
- Публичные данные остаются публичными; Bright Data выиграла суды против Meta и X (Twitter).

## 3. Практические выводы
- Не нужно направлять LLM на разбор сотен страниц; проще один раз создать скрапер и переиспользовать его скрипт.
- Подход применим не только для масштабных сборов, но и для личных задач: мониторинг цен, поиск квартиры, бронирование столиков.
- Агент способен работать с геоограниченными сайтами (выбор IP нужной страны) и выполнять действия: заполнение форм, клики.
- Следует проверять условия сайтов; Bright Data работает строго с публичными данными, не принимая пользовательские соглашения и не входя в аккаунты.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Bright Data**: MCP (66 инструментов), scraping skills на GitHub, API Web Unlocker, Scraper as Markdown, инфраструктура удалённых браузеров, pre-built APIs, AI-решение капчи.
- **Claude Code** и **Claude Haiku** (достаточно для browsing-агентов).
- **MCP** (Model Context Protocol).
- Подходы: VibeCode, самоисцеляющиеся пайплайны, имитация человеческого поведения (предзаписанные движения мыши и клавиатурный ввод).

## 5. Что полезно разработчику
- Автоматизация создания скраперов, которые раньше отнимали часы или дни.
- Через MCP можно отправлять curl-запросы, получать чистый markdown (экономия токенов) и не решать проблемы блокировок.
- Готовые API для популярных доменов (например, Amazon) избавляют от ручного парсинга.
- Удалённые браузеры работают на серверах Bright Data, снимая нагрузку с локальной машины.
- Пайплайн самостоятельно проверяет данные и исправляет скрапер при сбоях.

## 6. Что можно применить в агентных продуктах
- Агент с Bright Data MCP исследует страницы, извлекает markdown, строит парсеры и выполняет действия (включая клики и заполнение форм).
- Для browsing-агентов хватает лёгких моделей уровня Claude Haiku, поскольку инфраструктура имитирует человека.
- Можно создавать «слушателей» изменений на сайтах и запускать автоматические действия (бронирование, уведомления).
- Периодическая самодиагностика: агент обнаруживает пропажу данных и чинит пайплайн за несколько минут.
