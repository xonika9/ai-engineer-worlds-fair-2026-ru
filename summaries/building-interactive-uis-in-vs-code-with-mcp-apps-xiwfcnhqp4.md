## 1. О чём доклад
Доклад знакомит с концепцией MCP Apps — расширением протокола MCP для встраивания интерактивных UI прямо в чат (на примере VS Code с GitHub Copilot). Спикеры объясняют, зачем это нужно, показывают архитектуру и проводят живое демо профилирования Go-кода с визуализацией flame graph в окне Copilot Chat.

## 2. Главные идеи
- MCP (Model Context Protocol) — открытый протокол от Anthropic, стандартизирующий доступ LLM к инструментам, ресурсам и подсказкам.
- Ранние версии MCP возвращали только текст/ASCII, что ограничивало визуализацию и интерактивность.
- MCP Apps позволяют серверу возвращать не просто данные, а ссылку на HTML-ресурс, который хост (VS Code) рендерит в изолированном iframe внутри чата.
- Компонент остаётся живым: может обновляться через callback к серверу, поддерживает клики, ввод, редактирование.
- Безопасность обеспечивается sandbox-iframe: приложение не имеет доступа к API VS Code или внешним ресурсам.

## 3. Практические выводы
- Переход от текстовых ответов к интерактивным UI элементам улучшает опыт работы с агентами в чате.
- Архитектура: пользователь → агент (LLM) → вызов тула MCP-сервера → JSON-результат со ссылкой на HTML → хост подгружает HTML и показывает в iframe.
- Можно строить UI на React, Vue, Svelte или ванильном JS — сервер лишь предоставляет ссылку на скомпилированный ресурс.
- Важно брать MCP-серверы из проверенных источников (например, список в VS Code), чтобы избежать вредоносного кода.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- VS Code, GitHub Copilot, Copilot Chat
- MCP (Model Context Protocol), MCP Apps
- Excalidraw MCP server (диаграммы)
- Shopify (интерактивный e-commerce в чате)
- Figma (генерация компонентов)
- Go pprof (профилирование), TypeScript, React (хуки), Vue, Svelte, ванильный JS
- Подход: sandbox-iframe для изоляции UI

## 5. Что полезно разработчику
- Умение создавать MCP-серверы, которые возвращают UI-ресурсы, а не только текст.
- Понимание трёхчастной структуры MCP App: тул (LLM-вызов), ресурс (HTML-интерфейс) и связь между ними.
- Использование готовых MCP-серверов из каталога VS Code с учётом безопасности.
- Профилирование кода и визуализация результатов прямо в среде разработки без переключения контекста.

## 6. Что можно применить в агентных продуктах
- Встраивание интерактивных компонентов в чат-интерфейсы: диаграммы (Excalidraw), data-дашборды, полный checkout в e-commerce.
- Сохранение брендированного UI внутри чата (пример Shopify).
- Снижение объёма текстовых подсказок: пользователь кликает, а не печатает следующий запрос.
- Живое взаимодействие с результатами работы агента (например, редактирование диаграммы или навигация по flame graph).
