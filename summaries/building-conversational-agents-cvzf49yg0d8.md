## 1. О чём доклад
Практический воркшоп по созданию разговорных агентов с помощью Gemini API. Первая часть посвящена новому Interactions API и построению текстового кодинг-агента, вторая — нативной аудиомодели Life API и голосовым агентам в реальном времени.

## 2. Главные идеи
- **Interactions API** унифицирует работу с моделями и агентами: серверное состояние, асинхронное выполнение, улучшенное неявное кэширование и привычный интерфейс в стиле Responses API.
- **Agent skills** (навыки) позволяют встроить в IDE готовые инструкции и ссылки на документацию, благодаря чему даже Gemini 3 Flash корректно использует API, незнакомое на момент обучения модели.
- **Life API** даёт низколатентное нативное аудио, мультиязычность, управление голосом через системные инструкции, видеопоток (например, screen share) и вызов инструментов, включая Google Search.
- Для продакшен-голосовых агентов пока лучше подходят каскадные pipeline (ASR → LLM → TTS) из-за большей наблюдаемости; нативная аудиомодель — взгляд в будущее.

## 3. Практические выводы
- Для новых проектов стоит переходить на Interactions API — он ближе к индустриальному стандарту и упрощает многошаговые цепочки с инструментами.
- Установите coding agent skill (`npx install`), чтобы агент знал актуальные модели и мог сам добрать недостающую информацию из документации.
- Голосовому агенту критичны проработанные системные инструкции и защитные ограничения: без них модель склонна игнорировать вызов инструментов и галлюцинировать.
- В production с Life API лучше использовать партнёрские интеграции (LiveKit, PipeCast) для WebRTC, хранения аудио/транскриптов и observability.
- Ephemeral tokens позволяют клиенту подключаться к Life API напрямую, минуя лишний серверный хоп и сокращая задержку.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **API/модели:** Gemini API, Interactions API (beta), Life API, Gemini 3.1 Flash Live, Gemini 3 Flash, Lyra 3 (генерация музыки).
- **Среда/навыки:** Google AI Studio, ai.dev, dot agents skills (Gemini interactions API skill, Gemini life API skill).
- **Партнёры:** LiveKit, PipeCast, Software Mansion (Fish Jam), Vision Agents, Vox Implants.
- **Примеры внедрений:** Shopify Sidekick, Waymo, Stitch, Hey Otto.
- **Технологии:** WebSocket, ephemeral tokens, MCP, Google Search grounding, context window compression, SSE (для Interactions API).

## 5. Что полезно разработчику
- Серверное состояние через `previous interaction ID` убирает ручное управление историей и повышает hit rate кэша в два-три раза.
- Быстрый старт с голосом: сервер генерирует короткоживущий token, браузер или телефон напрямую общаются с Life API.
- Голос агента легко менять системной инструкцией, не ограничиваясь предустановленными голосами.
- Цикл обработки tool calls в Interactions API: проверка `requires_action`, выполнение функции, рекурсивный вызов `run`.
- Видеопоток (1 кадр/с) можно использовать для screen share, как это делает Sidekick.

## 6. Что можно применить в агентных продуктах
- Длительные сессии с ветвлением истории: храните interaction ID клиента и возвращайтесь к любой точке диалога.
- Комбинация Google Search и кастомных функций — разрешена недавним обновлением.
- Асинхронное выполнение для агентов, работающих дольше нескольких минут (deep research).
- Агент-ассистент с видеоподдержкой: модель видит экран и голосом проводит пользователя по настройкам.
- Мультиязычные голосовые интерфейсы для сервисов, ориентированных на пожилых или мультикультурную аудиторию.

## 7. Стоит ли смотреть полностью: yes / maybe / no
**maybe** — демо Life API страдало ошибками (не работал Google grounding, путаница с custom tools), много времени заняла выдача ключей, но сегмент с Interactions API и концепция skills поданы наглядно, а сессия вопросов добавляет реалистичный контекст о границах применимости.
