## 1. О чём доклад  
Обзор подходов Anthropic к внедрению ИИ в корпоративных продуктах: как строить оценки, управлять компромиссами качества/стоимости/задержки и избегать типичных ошибок при работе с Claude.

## 2. Главные идеи  
- Оценки (evals) — главный актив; их нужно проектировать до архитектуры, а не после.  
- Fine‑tuning не панацея: сначала исчерпайте prompt engineering, кэширование, контекстный retrieval и только потом прибегайте к тонкой настройке.  
- Для каждого сценария важно заранее определить приоритеты в треугольнике «интеллект – стоимость – задержка» (например, саппорт требует ответа <10 с, аналитика может работать минуты).  
- Интерпретируемость (понимание активаций нейронов) — долгосрочное направление для повышения безопасности и управляемости.  
- Лучшие результаты достигаются при глубоком погружении в домен клиента и итеративных спринтах по оптимизации промптов.

## 3. Практические выводы  
- Начинайте с телеметрии и репрезентативных тестовых кейсов, включая «глупые» запросы (вроде «как убить зомби в Minecraft»), чтобы проверить обработку выбросов.  
- Используйте prompt caching: он даёт до 90% снижения стоимости и до 50% прироста скорости без потери качества.  
- Определите допустимое время ответа под бизнес‑задачу и проектируйте UX‑отвлечения (например, анимированный индикатор), чтобы уложиться в бюджет задержки.  
- Переход к fine‑tuning оправдан только после того, как исчерпаны более лёгкие методы и чётко измерен выигрыш по метрикам.  
- Оценки позволяют «нащупать» оптимальную точку в латентном пространстве модели; без них оптимизация вслепую.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- **Модель:** Claude Sonnet 3.5 (new), бенчмарк sbench (агентное кодирование).  
- **Продукты:** API Anthropic, Claude for Work, AWS Bedrock, GCP Vertex.  
- **Клиентский кейс:** Intercom – агент Finn 2; достигнуто 86% автоматического разрешения обращений.  
- **Протокол:** Model Context Protocol (MCP) – open‑source для взаимодействия моделей с источниками данных.  
- **Методы:** prompt engineering, prompt caching, contextual retrieval, citations, agentic architectures, интерпретируемость (feature activation, Golden Gate Claude).  

## 5. Что полезно разработчику  
- Создавать evals на старте, включая edge‑cases; использовать логирование и AB‑тестирование для непрерывного улучшения.  
- Применять prompt caching для повторяющихся префиксов – резко ускоряет и удешевляет вызовы.  
- Не переходить к тонкой настройке, пока модель с хорошо спроектированными промптами не достигнет потолка по целевым метрикам.  
- В саппорт‑сценариях ориентироваться на время ответа ≤10 с; иначе пользователь уходит.

## 6. Что можно применить в агентных продуктах  
- Агентные архитектуры (подробнее – в выступлении коллеги Барри) позволяют разбивать сложные задачи на цепочки шагов.  
- Пример Finn 2: агентная логика, интегрированная с доменными политиками (возвраты, тональность), даёт высокий процент автономных решений.  
- Оценки и персональные метрики так же критичны для агентов, как и для простых вызовов.

## 7. Стоит ли смотреть полностью: maybe  
Доклад даёт качественные ориентиры, но не содержит глубоких технических деталей; полезен тем, кто начинает внедрять LLM в продукт или хочет систематизировать подход к оценкам.
