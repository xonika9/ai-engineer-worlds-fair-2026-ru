## 1. О чём доклад  
Доклад посвящён роли Model Context Protocol (MCP) в построении агентных систем, текущему состоянию экосистемы, архитектурным паттернам подключения инструментов и будущему развитию протокола. Спикер показывает, как MCP позволяет агенту отдавать собственный интерфейс, объединять инструменты и UI, и аргументирует, что в 2026‑м лучшие агенты будут комбинировать MCP, навыки, CLI и компьютерное управление для полной связности.

## 2. Главные идеи  
- MCP — общий стандарт, который поддерживают OpenAI, Google, LangChain и тысячи фреймворков; 110 млн ежемесячных загрузок.  
- Для продуктивных агентов 2026 года нужна не одна технология связности, а комбинация: навыки, MCP, CLI/computer use.  
- Ключевая задача клиентской стороны — *progressive discovery* (отложенная загрузка инструментов по требованию) и *programmatic tool calling* («режим кода») для композиции вызовов без лишних шагов инференса.  
- Серверы MCP должны проектироваться под агента, а не механически оборачивать REST API один-в-один: нужно задействовать богатую семантику (приложения, задачи, элиситации).  
- Протоколу предстоит улучшить масштабируемость (stateless transport), асинхронное меж‑агентное взаимодействие, SDK и добавить server discovery, cross‑app‑доступ, навыки поверх MCP.

## 3. Практические выводы  
- Внедрять progressive discovery: инструменты подгружаются, только когда модель их запрашивает — резкое снижение расхода контекста.  
- Предоставлять агенту исполняемую среду (V8 isolate, Lua) и позволять писать код, который компонует вызовы MCP; использовать structured output для типов.  
- Не плодить универсальные REST‑to‑MCP‑конвертеры; проектировать сервер так, как человек хотел бы взаимодействовать с системой.  
- Уже сейчас можно экспериментировать с «‑загрузкой навыков» через инструмент load‑skills, не дожидаясь официальной спецификации.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Anthropic API, Claude Code, MCP (Model Context Protocol).  
- OpenAI Agent SDK, Google ADK, LangChain, fast MCP, Cloudflare MCP server.  
- Stateless transport (предложение Google), streamable HTTP, structured output, V8 isolate, Lua.  
- Ключевые примитивы MCP: tools, elicitation, tasks, MCP applications, skills over MCP.

## 5. Что полезно разработчику  
- Реализация progressive discovery на клиенте: либо встроенными средствами API, либо через инструмент «поиск инструментов» с динамической подгрузкой.  
- Programmatic tool calling: дать модели REPL‑окружение, чтобы она генерировала скрипты, а не выполняла множество последовательных вызовов.  
- Использовать structured output сервера или дёшево извлекать типы через дополнительную модель для надёжной композиции.  
- Готовиться к переходу на stateless transport и новым SDK (TypeScript v2, Python v2) для упрощённого масштабирования.

## 6. Что можно применить в агентных продуктах  
- Комбинация MCP + CLI + навыки + computer use как стек связности для knowledge‑worker‑агентов.  
- Доставка UI и инструментов через один MCP‑сервер (MCP‑приложения) — единая точка для человека и модели.  
- Планируемая семантика server discovery (well‑known URL) позволит агентам автоматически находить MCP‑серверы сайтов.  
- Cross‑app access упростит бесшовную аутентификацию в корпоративных средах, а навыки поверх MCP заменят реестры плагинов.
