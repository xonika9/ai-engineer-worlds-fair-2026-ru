## 1. О чём доклад
Дэн Мейсон (Stride) делится опытом построения агентного workflow для телемедицинской поддержки (клиент Aila Science). Рассказывает, как заменили ручное нажатие кнопок операторами на виртуального ассистента, общающегося с пациентами по SMS. Решение построено на LangGraph / MCP и LLM (Claude), с обязательным человеческим контролем при низкой уверенности.

## 2. Главные идеи
- Гибридная система: LLM управляет диалогом и состоянием пациента, но человек остаётся в цикле для сложных или неоднозначных ситуаций.
- Вся бизнес‑логика лечения вынесена из кода в самоссылающиеся Markdown‑документы («blueprints»), что позволяет добавлять новые протоколы без программирования.
- Второй LLM‑агент (evaluator) оценивает не только корректность ответа, но и сложность изменений; при падении интегральной оценки ниже порога запрос уходит на утверждение человеку.
- Вместо RAG выбрана простая иерархия документов: каждый документ явно ссылается на следующий нужный фрагмент, а модель читает целые файлы, а не вырванные куски.

## 3. Практические выводы
- Достигнуто примерно 10‑кратное увеличение пропускной способности операторов без снижения безопасности.
- Визуализация графа в LangGraph сильно упростила объяснение системы заказчику и отладку.
- LLM‑оценщик плохо распознаёт собственные ошибки, поэтому в формулу оценки добавлен фактор сложности изменений (установка якорей, правка данных и др.).
- Отказ от RAG в пользу явной загрузки документов оправдан, когда объём справочных материалов невелик и важна полнота контекста.
- Промпты и руководства занимают примерно столько же объёма, сколько код на Python.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **LangGraph / LangChain / LangSmith** – оркестрация агентов и наблюдаемость.
- **MCP** – для доступа к файлам и внешней базе данных.
- **Claude** (основной движок) и GPT‑4o (в роли судьи для офлайн‑оценок).
- **PromptFu** – внешний инструмент для LLM‑as‑a‑judge‑оценок.
- **Node, React, MongoDB, Twilio, AWS, Terraform, GitHub Actions** – остальная инфраструктура.
- **Клиент Aila Science**, консультанты **Stride**.
- **Использование «vibecoding»** с помощью Cline/Klein для написания Python‑части.

## 5. Что полезно разработчику
- Разделение агентов: виртуальный оператор (генерирует ответ и меняет состояние) и оценщик (проверяет и выставляет уверенность).
- При ошибке вызова инструмента или «молчании» модели можно добавлять retry‑узлы, удаляющие неудачное сообщение и заставляющие агента повторить попытку.
- Кэширование системного промпта и длинных документов критично для снижения стоимости; при смене системного промпта кэш сбрасывается.
- Состояние сжимается после каждого взаимодействия: в «долгую» память попадают только итоговые сообщения и оценки, а не внутренние рассуждения моделей.
- Использование Command‑объектов в LangGraph позволяет гибко маршрутизировать переходы между узлами без жёсткой привязки в графе.

## 6. Что можно применить в агентных продуктах
- Модель «LLM предлагает – LLM‑оценщик проверяет – человек утверждает» для ответственных доменов.
- Самоссылающиеся Markdown‑документы как альтернатива RAG, когда документы невелики, а точность важнее масштаба.
- Динамическое изменение порога уверенности: чем ниже порог, тем больше автоматических отправок, но выше риск.
- Принцип «всегда отвечай на вопрос пациента, а потом мягко возвращай на план лечения».
- Управление несколькими лечениями через смену набора документов, а не переписывание кода.

*«you can do new treatments in new workflows without writing more code»*
