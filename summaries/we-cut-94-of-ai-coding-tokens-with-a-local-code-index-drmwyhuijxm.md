# We Cut 94% of AI Coding Tokens With a Local Code Index

## 1. О чём доклад

Докладчик рассказывает, как в его проекте внезапно вырос счёт за AI-инструменты. Анализ показал: основные деньги уходили не на генерацию кода, а на отправку лишнего контекста — 45 000 токенов запросов при реальной необходимости лишь в 5 000. Они создали локальный поисковый слой, который сокращает входные токены на 94% без значительной потери точности.

## 2. Главные идеи

- 90% затрат составляет входной контекст и лишь 10% — выходной. Сокращать надо именно вход.
- Двойной поиск (по смыслу и по точным словам) компенсирует слабости друг друга, снижая долю пропущенных результатов с 25% до 10%.
- Простые формулы скоринга (50% семантика, 30% ключевые слова, 20% свежесть) обгоняют сложные модели.
- Общий локальный индекс для всех AI-инструментов с памятью предотвращает переобъяснение кодовой базы.

## 3. Практические выводы

- *«Fix the input, the model choice matters less than you think»*.
- Сжатие вывода экономит лишь около 8%, реальная экономия — в уменьшении входных токенов.
- Стоит комбинировать семантический и буквальный поиск, а не полагаться на один.
- Индекс должен работать локально — это быстро, дёшево и не передаёт код в облако.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- Инструменты: Cloud Code, Cursor, Copilot, Code X, Opus, Sonnet.
- Фреймворки/продукты: FastAPI (тестовый проект), CCE (опенсорсная команда для проверки).
- Подходы: локальный слой поиска, разбиение кода на функции/классы, гибридный поиск, сжатие через заголовки функций, граф вызовов, динамический скоринг.

## 5. Что полезно разработчику

Идея локального семантического индекса с двойным поиском прямо применима в developer tools и observability: можно встроить быстрый поисковый слой для AI-агентов, сократив латентность и стоимость вызовов. Метод скоринга (0,4 мс) подходит для evals и harness, где важна скорость без лишних AI-обращений.

## 6. Что можно применить в агентных продуктах

- Добавить локальный кодо-индекс с гибридным поиском для агентов, чтобы они посылали только релевантные функции, а не целые файлы.
- Реализовать общую память между сессиями, чтобы агенты не начинали контекст «с нуля».
- Использовать простые формулы релевантности вместо дорогих эвалов.

Доклад практичный: конкретные цифры, метрики, публичный тест. Полезен для всех, кто платит за AI-кодинг.
