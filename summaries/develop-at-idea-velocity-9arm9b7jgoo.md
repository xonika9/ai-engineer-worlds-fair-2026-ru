1. О чём доклад  
Разработчик из Snapchat показывает персональный сетап для ускорения идей до кода: использование OpenClaw, tmux, оркестраторов и стадийного тестирования. Цель — заменить рутинные ответы и ручное тестирование автономными агентами, добиваясь «скорости мысли».

2. Главные идеи  
- Frictionless communication: общаться с агентом кратко через Slack, чтобы контекст и история задач запоминались.  
- Специализация контекста: OpenClaw оперирует спецификациями и целями, а не перегружен деталями реализации, в отличие от прямого использования Claude Code.  
- Центральная ось контроля: спектр от «ковбойского» доступа до личного контроля через tmux-терминалы.  
- Менеджер вместо кодера: агент-менеджер unbiased оценивает результаты рабочих агентов, отсеивает слабые PR и даёт трезвую критику.  
- Постоянное улучшение: браузерные тесты, которые полгода назад требовали человека, теперь агенты проходят без проблем.

3. Практические выводы  
- Выстраивать стек: общение (Slack) → оркестратор → управляемые агенты (Claude Code с sub-агентами).  
- Использовать work trees и параллельные терминалы (tmux/Cmux) для распараллеливания задач.  
- Стадийная разработка: локальный агент + staging-агент для интеграционных тестов перед продом.  
- Заменять ручное тестирование слоями: юнит-тесты → MCP/HTTP → браузерные тесты → визуальная проверка.  
- Переключать модели по стоимости/токенам: основная Codex 5.3, при перерасходе токенов — MiniMax.

4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- OpenClaw, Claude Code, tmux  
- Cmux (терминальный мультиплексор, интеграция с Clockwork Teams, SSH)  
- Форк «азиатского оркестратора» для workers  
- Модели: Codex 5.3, GPT-5.4, MiniMax  
- Песочницы (Docker, изолированное окружение), staging-инструкция  
- Multi-AI analysis website (анализ несколькими моделями одновременно)  
- AI RPG с кастомным миром и D&D-системой  
- Подход: «скептик-агент» для код-ревью

5. Что полезно разработчику: AI-агенты, браузерные и офисные агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте  
- Разделение агентов на менеджера и рабочих с непредвзятой оценкой результатов (пример с PR).  
- Схема слоёв тестирования агентов: от юнит-тестов до визуальной проверки — применима для evals и observability.  
- Staging-песочница для безопасного обкатывания агентов без двойного расхода токенов.  
- Cmux как инструмент параллельного управления сессиями и уведомлениями для повышения эффективности.  
- Подход frictionless-коммуникации — можно адаптировать для браузерных/офисных агентов внутри агентном продукте.

6. Что можно применить в агентных продуктах  
- Реализовать менеджера агентов, который сравнивает выводы нескольких рабочих агентов и выдаёт сводное решение (как в multi-AI analysis).  
- Внедрить вертикальные табы с уведомлениями для отслеживания параллельных задач агентов.  
- Добавить staging-окружение для агентов, чтобы не рисковать основным процессом и экономно тестировать.  
- Использовать спектр контроля: позволить пользователю переходить от «ковбойского» режима к пошаговому через tmux-подобный интерфейс.
