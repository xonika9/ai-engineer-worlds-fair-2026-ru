## О чём доклад
Доклад Пьетро Сулло (сооснователь Manufact) о MCP Apps — расширении Model Context Protocol для возврата интерактивных UI-виджетов прямо в чат. Рассматриваются примитивы MCP Apps, механики дистрибуции через магазины ChatGPT, Claude и Cursor, а также динамическое обнаружение серверов моделью. Основной посыл: AI-приложения становятся новыми браузерами, а MCP Apps — новыми веб-сайтами, поэтому публикация в сторах критически важна для охвата пользователей с высоким намерением.

## Главные идеи
- MCP Apps позволяют серверу возвращать не только JSON, но и полноценный UI в sandboxed iframe с двусторонней коммуникацией (сообщения, обновление состояния модели, вызов новых инструментов).
- Ключевые примитивы: model context (setState), отправка follow-up-сообщений из виджета, real-time streaming аргументов в UI, вызов других инструментов из UI, разделение приватного вывода (UI видит всё, модель — только разрешённое), полноэкранный режим и синхронизация темы.
- Динамическое обнаружение: Claude уже ищет подходящий коннектор в реестре, если у задачи нет явно заданного инструмента; ChatGPT вскоре запустит то же самое.
- Магазины ChatGPT, Claude и Cursor открыты для самостоятельной отправки серверов, а публикация там даёт огромный органический трафик от пользователей, которые ищут решение прямо в чате.

## Практические выводы
- Стройте MCP серверы с UI: они значительно улучшают UX по сравнению со стеной текста.
- Отправляйте серверы в сторы — это главный канал дистрибуции для продукта, ориентированного на AI-агентов.
- Используйте раздельные outputs для конфиденциальности: виджет может показывать полные данные пользователю, а модель получать лишь текстовое описание.
- Для быстрого старта берите шаблон `npx create-mcp-app` и открытый SDK MCPUs, а для проверки перед сабмитом — cloud-сервис Manufact.

## Что полезно разработчику: AI-агенты, браузерные и офисные агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте
В докладе напрямую не затрагиваются evals/observability, но есть детали, полезные для агентных систем: выполнение инструментов через виджеты, изолированное sandbox-окружение (iframe), возможность вызова дополнительных инструментов из UI, разделение приватного контекста и состояния модели. Подход с магазинами и динамическим обнаружением может служить моделью дистрибуции для браузерных или офисных агентов.

## Что можно применить в агентных продуктах
- Встроить генерацию MCP Apps с UI-widget’ами, чтобы агенты могли возвращать интерактивные карточки, формы или диаграммы прямо в интерфейсе студии.
- Использовать примитив setState для синхронизации состояния между UI и моделью, а send follow‑up message — для инициирования новых цепочек действий из виджета.
- Интегрировать проверку на поддержку MCP Apps со стороны клиента и соответственно переключать выводы, повышая совместимость.
- Применить практику раздельных outputs для случаев, когда агент должен показать пользователю приватные данные, но не отсылать их в LLM.

## Возможная идея для поста
«MCP Apps — это новые сайты, а ChatGPT и Claude — новые браузеры. Как сторы меняют дистрибуцию и почему публикация MCP-сервера с UI даёт доступ к аудитории с максимально высоким намерением, а динамический discovery модели сам найдёт ваш коннектор, когда он понадобится пользователю.»
