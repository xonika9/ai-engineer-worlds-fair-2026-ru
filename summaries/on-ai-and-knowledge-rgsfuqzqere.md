## 1. О чём доклад
Пабло Кастро из Microsoft раскладывает знание для агентов на три вида: встроенное в модель, привнесённое извне и накопленное в процессе работы. Он показывает, как встроенное знание запустило нынешний экспоненциальный рост, как привнесённое знание эволюционировало от простого RAG до сложных систем поиска и общекорпоративного заземления, и как накопленное знание превращается в петлю обучения, где агенты автоматически улучшаются на собственных трассах. Всё это иллюстрируется продуктами Microsoft Foundry и Microsoft IQ с живыми демонстрациями.

## 2. Главные идеи
- Знание бывает трёх видов: встроенное (то, на чём обучена модель, её параметрическая память), привнесённое извне (заземление на данные организации) и накопленное (результат работы, из которого система учится).
- Именно встроенное в модель знание запустило экспоненту: от IntelliSense (1996) к ранжированию подсказок машинным обучением, затем к GitHub Copilot ещё до ChatGPT, к Cursor и далее.
- Для участия агента в жизни компании одного встроенного знания мало — так возник RAG, который из простого приёма вырос в развитую систему инженерии контекста.
- Заземление усложнилось по двум осям: от изолированных наборов данных к общекорпоративному охвату и от простого векторного поиска к сложным системам извлечения.
- Комбинация методов извлечения устойчиво превосходит отдельные методы, особенно на реальных клиентских сценариях; задача платформы — дать комбинировать блоки, не вываливая сложность на пользователя.
- Накопленное знание материализуется в петлю обучения: наблюдать процессы, рефлексировать и автоматически донастраивать агентов, захватывая то уникальное, что отличает конкретную компанию.

## 3. Практические выводы
- Платформа заземления Microsoft IQ даёт единую точку входа к «фоновым» данным организации: Work IQ (документы, почта, календарь, чаты SharePoint), Fabric IQ (аналитические активы, отчёты Power BI), Foundry IQ (свои данные агента) и Web IQ (публичные данные из веба).
- Система извлечения в Foundry IQ послойная: сверху можно просто отдать PDF и картинки и получить готовое заземление, снизу — управлять векторными индексами, квантизацией и лексическим поиском, двигаясь вверх-вниз по мере роста требований.
- Поверх обычного извлечения есть агентный слой, который для сложных случаев рефлексирует над данными и решает, удовлетворена ли информационная потребность, прежде чем вернуть результат; по метрикам полноты и покрытия он стабильно лучше одиночного извлечения.
- Каждая база знаний доступна как MCP-сервер, поэтому её можно подключить к своему харнессу без связующего кода; система оптимизирована по токенам, чтобы отдавать плотный ответ минимальным их числом.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Microsoft Foundry как платформа для агентов: хостинг, наблюдаемость, управление и каталог из тысяч моделей; объявлена общая доступность Claude в Microsoft Foundry.
- Microsoft IQ и его составляющие: Work IQ, Fabric IQ, Foundry IQ, Web IQ.
- Технология поиска за Foundry IQ (в расшифровке — «AI search»), векторные индексы, лексический и гибридный поиск, агентное извлечение.
- Agent optimizer и Foundry toolkit в VS Code; шаги eval generate, optimize и optimize apply с внешней конфигурацией агента (инструкции, инструменты, навыки).
- Историческая линия: IntelliSense, GitHub Copilot, GitHub Copilot X, Cursor, Opus 4.5; портал ai.azure.com; упомянута заметка Сатьи о совместном обучении людей и агентов.

## 5. Что полезно разработчику
Доклад даёт рабочую рамку для проектирования заземления: разделять встроенное, привнесённое и накопленное знание и подбирать под каждое свой механизм. Практично то, что базы знаний отдаются как MCP-серверы (подключение без связующего кода), система извлечения послойная (можно начать просто и углубляться), а agent optimizer позволяет автоматически улучшать агента по оценке, вынеся его конфигурацию наружу.

## 6. Что можно применить в агентных продуктах
- Комбинировать методы извлечения вместо ставки на один (например, только косинусную близость векторов) — на реальных данных это заметно улучшает результат.
- Добавлять агентный слой извлечения для сложных запросов: рефлексия над данными и проверка достаточности перед возвратом ответа.
- Подключать источники знаний через MCP, чтобы переиспользовать их в разных харнессах без склеивающего кода.
- Строить петлю обучения: собирать трассы работающего агента, генерировать и оценивать кандидатов конфигурации, а затем разворачивать лучший вариант.

## 7. Стоит ли смотреть полностью: yes — ясная трёхчастная модель знания для агентов и живые демонстрации заземления и автоматической оптимизации в Microsoft Foundry.
