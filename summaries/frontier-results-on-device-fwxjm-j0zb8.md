## О чём доклад

Доклад о том, как перестать платить за большие облачные LLM (GPT, Claude), перенося инференс на устройство пользователя с помощью малых языковых моделей (SLM). На примере фичи для Mima (саммари тредов в соцсетях) автор показывает полный цикл: от прототипа на Claude до выбора и донастройки локальной модели Llama 3.2, которая сравнялась по качеству, но работает быстрее, бесплатно и без рисков утечки данных.

## Главные идеи

- *Prototype big, deploy small.* Сначала доказываем возможность задачи на самой большой модели, потом переносим на самую маленькую из допустимых по качеству — SAGE (small and good enough).
- Четыре шага: 1) доказать возможность (большая модель), 2) задать критерии успеха и собрать золотой датасет, 3) тестировать модели от маленьких к большим, 4) выбрать SAGE-модель.
- Закрыть разрыв в качестве помогает промпт-инжиниринг и пост-обработка в harness.
- Локальный инференс решает проблемы безопасности (данные не покидают устройство), задержки (нет round-trip), стоимости инференса и работы офлайн.

## Практические выводы

1. Соберите «золотой датасет» из входных данных и желаемых ответов (можно взять вывод самой умной модели).
2. Определите метрики: валидность JSON, структурная достоверность, фактическая согласованность, длина, P50/P95 задержка.
3. Прогоните на Phoenix (Arize) возможность eval для нескольких SLM. Смотрите сырые ответы — LLM-судья может быть предвзят.
4. *Few-shot prompting дал наибольший прирост качества* у Llama 3.2, в отличие от строгих правил и цепочки мыслей.
5. Часть ошибок (длина, ссылки) проще исправить пост-обработкой в коде.

## Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Arize Phoenix** — open-source платформа для evals и observability.
- **Goose** — open-source agent harness.
- **Модели:** Claude Sonnet, Qwen 2.5/3, Llama 3.2, Gemma 4 E2B, Gemini, Gemini Nano, MobileNet, YOLO, MediaPipe, Whisper, Wave2Vec2.
- **Браузерные API:** Chrome Prompt API (Gemini Nano on-device).
- **Mima** — PWA-клиент соцсетей.
- **Веб-стандарты:** web.dev, Mozilla, W3C.

## Что полезно разработчику: AI-агенты, браузерные и офисные агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте

- **Harness:** Goose показан как пример агентской обвязки для локальных моделей.
- **Evals и observability:** Phoenix — готовый инструмент для возможность- и regression-эвалов; можно встроить CI/CD-подобные проверки, чтобы смена промпта или модели не ломала агентное поведение.
- **Sandbox/tool execution:** Идея пост-обработки ответов модели в harness (обрезка длины, валидация ссылок) применима к любым песочницам исполнения инструментов.

## Что можно применить в агентных продуктах

- Добавить в агентном продукте слой **локального SAGE-инференса** через Chrome Prompt API (Gemini Nano) или модель в WebAssembly, чтобы офисные/браузерные агенты работали без облачных запросов.
- Встроить **автоматические возможность evals** через Phoenix для каждого нового промпта или модели, аналогично CI-тестам.
- Использовать **few-shot промпты** при проектировании агентских инструкций.
- Внедрить пост-обработку ответов SLM в самом harness, закрывая типичные провалы малых моделей.
