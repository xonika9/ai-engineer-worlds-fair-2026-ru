## 1. О чём доклад

Эрик Ханчетт (AWS) объясняет, как spec-driven development помогает писать код быстрее и качественнее с помощью AI-ассистентов. Подход требует создать структурированные спецификации (markdown‑файлы с требованиями, дизайном и списком задач) до начала кодинга, что даёт языковым моделям чёткий контекст и направление.

## 2. Главные идеи

- Большие языковые модели ведут себя как нетерпеливые стажёры: без подробных инструкций легко уходят в сторону. Spec‑документы направляют их в нужное русло.
- Процесс: сначала требования в формате EARS и уточняющие вопросы, затем дизайн‑документ (Mermaid‑диаграммы), далее детальный список задач. На каждом этапе человек проверяет и корректирует результат.
- Важен баланс контекста: не перегружать `agents.md` / `claw.md`, использовать skills для активации правил «на лету», никогда не доверять агенту полностью.
- Подход годится для новых проектов, легаси и даже багфиксов. Рекомендуется запрашивать MVP (первые 4–5 задач) для быстрой проверки.
- Property‑based тесты (fast‑check) автоматически проверяют соответствие требованиям.
- Через Model Context Protocol (MCP) можно подключать Jira / Asana и затягивать готовые требования в спецификации.

## 3. Практические выводы

- Создавайте спецификации до кода — это снижает число ошибок и ускоряет разработку.
- Используйте инструменты, задающие уточняющие вопросы, чтобы AI собрал максимум контекста.
- Всегда ревьюйте сгенерированные документы на галлюцинации.
- Для быстрого результата просите сначала MVP, а затем остальные задачи.
- Интегрируйте MCP с таск‑трекерами, чтобы автоматически втягивать требования продукт‑менеджера.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- AWS Kiro (IDE и CLI) со встроенным spec‑режимом.
- Spec Kit (от GitHub) — open‑source реализация.
- Ручное управление через инструкции любому AI-ассистенту.
- Model Context Protocol (MCP) для подключения внешних данных.
- Property‑based testing (fast‑check в TypeScript).
- Форматы: EARS, Mermaid, markdown‑документы.

## 5. Что полезно разработчику

Прямых упоминаний указанных направлений нет, однако подход spec‑driven‑разработки применим к созданию любых AI-агентов: чёткие спецификации дают платформам (включая «harness»‑решения) структуру для исполнения инструментов и повышения качества. Можно использовать как методологию для агентов агентном продукте.

## 6. Что можно применить в агентных продуктах

- Внедрить создание markdown‑спецификаций (требования → дизайн → задачи) перед генерацией любого кода.
- Настроить MCP‑подключение к Jira / Asana студии, чтобы автоматически собирать и актуализировать требования.
- Добавить обязательное ревью спецификаций человеком и автогенерацию property‑based тестов для ответственных модулей.
