## 1. О чём доклад
Gabe De Mesa из OpenGov рассказывает, как команда построила и масштабировала агента OG Assist для ERP-решений в госсекторе, — от перехода на собственный цикл до интеграции обратной связи, песочниц и генеративного UI.

## 2. Главные идеи
- Отказ от LangGraph в пользу «Effect Native Agent Loop» даёт полный контроль, трассировку, structured concurrency и горячую замену моделей.
- Большая ставка на библиотеку Effect (open-source TypeScript) и на инструменты/навыки как главные строительные блоки.
- Протокол A2A (Google) используется как контракт между фронтендом и бэкендом, упрощая разработку.
- *«Shipping is the start, not the finish»* — эвалы (автоматические в CI) и сбор обратной связи через «палец вверх/вниз» ускоряют итерации.
- Безопасность: человек в цикле утверждает мутирующие действия; код выполняется в изолированных эфемерных песочницах.
- Длинный контекст решается скользящим суммированием с памятью — агент помнит ключевые моменты без перегрузки промпта.
- Генеративный UI: агент в рантайме строит формы и варианты, делая взаимодействие персонализированным.
- *«You can’t scale what you can’t see»* — трассировка на Effect даёт сквозную наблюдаемость вызовов из коробки.
- Внутреннее ускорение: команда использует Claude и Cursor для чтения, написания и ревью кода, а также строит инструменты для собственных рабочих процессов.

## 3. Практические выводы
- Строить агента на собственном цикле (Effect) выгодно для полного контроля и observability.
- Инструменты и навыки — минимально необходимый фундамент.
- A2A‑протокол помогает чётко описать контракт агента.
- Обязательные уровни безопасности: подтверждение человеком и песочницы для выполнения кода.
- Скользящее суммирование вместо впихивания всей истории решает проблему длинных диалогов.
- Автоматические эвалы в CI + пользовательская обратная связь — ключ к быстрым улучшениям.
- Генеративный UI делает опыт «живым» без жёсткой зашивки элементов.

## 4. Упомянутые инструменты, фреймворки, компании и подходы
OpenGov, OG Assist, Effect (Effect‑TS / Effect AI), TypeScript, LangGraph (предыдущее решение), A2A протокол (Google), Zod (аналогия схем), Claude, Cursor, CI‑эвалы, песочницы, скользящее суммирование, генеративный UI, трассировка Effect.

## 5. Что полезно разработчику
- **Песочницы:** безопасное выполнение кода агентов в изолированной среде.
- **Tool execution:** модель «инструмент → набор инструментов → регистрация в языковой модели» как стандартный паттерн.
- **Evals & observability:** автоматические эвалы в CI и трассировка Effect для отладки агентных систем.
- **Developer tools:** связка Claude + Cursor для ускорения разработки.
- Эти практики напрямую применимы при проектировании агентов в агентном продукте.

## 6. Что можно применить в агентных продуктах
- Использовать Effect для написания собственного агентного цикла с полной трассировкой и dependency injection.
- Внедрить A2A-контракт как интерфейс между студийными агентами и UI.
- Реализовать песочницы для выполнения сгенерированного кода.
- Добавить механизм «человек в цикле» для подтверждения критических действий.
- Применить скользящее суммирование для длинных сессий.
- Встроить эвалы и сбор обратной связи через thumbs‑up/down.
