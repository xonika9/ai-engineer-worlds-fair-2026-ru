## 1. О чём доклад  
О возможностях on‑device AI на Android: как использовать Gemini Nano через ML Kit GenAI API, гибридный и облачный инференс, и как система управляет моделью на устройстве. Формат – сессия вопросов и ответов на AI Engineer World’s Fair 2026.

## 2. Главные идеи  
- On‑device инференс (Gemini Nano) обеспечивает приватность, офлайн‑работу и отсутствие затрат на облако для разработчика.  
- Gemini Nano – это единая модель, поставляемая через системный сервис AI Core; она разделяется всеми приложениями, а система управляет приоритетами и очередями.  
- Для большинства задач достаточно Prompt API (текст + изображение на вход, текст на выход).  
- Гибридный подход через Firebase AI Logic расширяет охват: если Nano недоступна на устройстве, инференс выполняется в облаке (Gemini Flash).  
- Фоновые задачи могут расходовать батарею; для длительной обработки лучше запускать при зарядке.  
- Кастомные модели – через LiteRT, но требуют больше ручного тестирования.

## 3. Практические выводы  
- Используйте ML Kit GenAI API для быстрого доступа к Gemini Nano без настройки модели.  
- При недоступности устройства – гибридный режим с облаком через Firebase AI Logic.  
- Система сама управляет очередями: foreground‑приложения получают приоритет, фон может ожидать.  
- Дискретное использование (10–20 запросов в день) практически не влияет на батарею.  
- Скоро появится embedding API для построения RAG‑решений прямо на устройстве.  
- Для старых устройств классические ML‑модели (OCR, Vision) работают на миллиардах устройств; GenAI‑модели требуют флагманов последних двух лет.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- **Модели и API**: Gemini Nano, ML Kit GenAI APIs (Prompt API, summarization, proofreading, rewrite), Gemini Flash, Gemma 4, embedding API (скоро).  
- **Системные сервисы**: AI Core, Firebase AI Logic.  
- **Облачные провайдеры**: Gemini API, Vertex AI, Gemini Developer API.  
- **Инструменты для кастомизации**: LiteRT (Light RT LM), AI Edge Gallery.  
- **Устройства**: Pixel 9 / 10 и другие OEM‑флагманы.  
- **Примеры упомянутых продуктов**: Pocket Claw, Open Claw.

## 5. Что полезно разработчику  
- Единый Prompt API для текста и изображений – не нужно заботиться о модели.  
- AI Core автоматически подбирает оптимальные ускорения (TPU) на устройстве.  
- Для RAG‑подобных решений можно уже сейчас комбинировать Prompt API с векторным поиском на клиенте, а скоро – встроенный embedding API.  
- Гибридный инференс позволяет одним кодом покрыть как устройства с Nano, так и без неё.  
- Система сама регулирует нагрузку: разработчику не нужно управлять одновременным доступом сотен приложений.

## 6. Что можно применить в агентных продуктах  
- Локальный запуск агентов с чувствительными данными (банкинг, персонализация) без отправки на сервер.  
- Prompt API может служить низкоуровневым движком для систем, подобных Pocket Claw / Open Claw, которые комбинируют «скиллы» и промпты.  
- Гибридный подход позволяет агенту деградировать на облако при отсутствии on‑device модели, сохраняя единый интерфейс.  
- Планируемый embedding API упростит локальные RAG‑пайплайны в автономных агентах.
