## 1. О чём доклад
Опыт построения «фабрики программного обеспечения» на основе Cursor, где инженер переходит от написания кода к управлению агентами. Рассматриваются уровни автономности, необходимые примитивы и паттерны, смена роли разработчика, запуск и масштабирование такой фабрики.

## 2. Главные идеи
- Разработчик становится менеджером агентов: меньше пишет код, больше ставит intent, скоупит задачи и проверяет результаты.
- Фабрика строится из примитивов (структура кода, паттерны использования), guardrails (хуки, правила, тесты) и enablers (skills, MCP, изолированное окружение).
- Правила должны возникать динамически, только когда агенты сходят с рельсов, а не загружаться массово «на всякий случай».
- Агентам нужна возможность самим верифицировать свою работу (тесты, скриншоты, computer use).
- Масштабирование требует автоматизации повторяющихся действий и извлечения уроков из транскриптов для continual learning.

## 3. Практические выводы
- Чётко формулируйте намерение, но не отдавайте агентам критически важные решения (безопасность, платежи).
- Переходите от синхронной работы к асинхронной: планируйте синхронно, выполняйте асинхронно через cloud agents.
- Запускайте агентов в изолированных средах (разные VM) для чистоты изменений.
- Стройте инструменты для самоавтоматизации: анализ повторяющихся задач, автоматическое извлечение правил из транскриптов.
- Дайте агентам «свободу» – например, канал для жалоб, чтобы выявлять узкие места инфраструктуры.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Инструменты и продукты:** Cursor (включая Cursor 3, Cloud Agents, Workers, Automations, Bugbot), Playwright, Linear, Slack, Figma, Ableton (пример проекта).
- **Концепции и подходы:** Уровни автономности Dan Shapiro, Agent skills, MCP-серверы, Continual learning, Computer use, Agentic code owners, Feature flags.

## 5. Что полезно разработчику
- Переключение мышления с «пишу код» на «управляю фабрикой агентов».
- Приём: правила вносить только когда агент ошибается, а не заранее.
- Использование cloud agents для длительных задач в фоне и просмотр видеозаписей их проверок.
- Автоматизация собственных рутинных действий (daily review, обновление правил) через самого Cursor.

## 6. Что можно применить в агентных продуктах
- Автоматический запуск агентов по событиям из Linear или Slack.
- Agentic code owners для умного ревью PR: оценка риска, привлечение нужных людей.
- Continual learning плагин: автоматическое создание правил из истории сессий.
- Инструмент «пожаловаться» для агентов, чтобы выявлять проблемы в harness.
