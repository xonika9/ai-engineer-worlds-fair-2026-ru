# Combine Skills and MCP to Close the Context Gap

- Спикеры: Pedro Rodrigues, Supabase
- Трек: Skills @ AI Engineer
- Видео: [YouTube](https://www.youtube.com/watch?v=JT3OzDKrucU)
- Официальный источник: [расписание AI Engineer World's Fair](https://www.ai.engineer/worldsfair/schedule)
- Статус обработки: summarized

> Это русскоязычное саммари. Полные расшифровки и медиафайлы здесь не публикуются.

## 1. О чём доклад
Доклад о создании навыков (skills) для агентов на примере Supabase. Педро Родригес рассказывает, как команда написала skill, чтобы агенты работали с продуктом безопасно и актуально, не полагаясь на устаревшие данные. Основной вопрос: как «закрыть разрыв контекста» с помощью инструкций.

## 2. Главные идеи
- Агентам нужны правильные инструкции (guidance), чтобы избегать устаревших знаний и типичных ошибок (например, пропуск RLS в PostgreSQL).
- Навык — это папка с front matter, основным файлом `skill.md` и опциональными ресурсами/скриптами.
- Три принципа при написании skill:
  1. *Не дублировать информацию*: указывать на живую документацию (вплоть до предоставления доков через SSH как файловую систему, чтобы агенты навигировали её bash-инструментами).
  2. *Если можно пропустить — будет пропущено*: агент ленив к загрузке референсных файлов, поэтому критически важное (чек-листы безопасности) должно быть прямо в `skill.md`.
  3. *Быть opinionated*: задавать проверенные workflow (например, управление схемой БД: менять в dev, запускать advisor, только затем создавать миграцию).
- Eval-тестирование на Braintrust показало, что MCP + skill превосходит «голый» MCP на всех моделях (Claude Opus/Sonnet 4.6, GPT 5.4, Codex), потому что навык даёт нужное руководство.

## 3. Практические выводы
- Указывайте на единый источник правды — документацию, не дублируйте.
- Начинайте с минимального `skill.md`, итеративно расширяйте.
- Критический контент (безопасность) не прячьте в референсных файлах — агент их проигнорирует.
- Будьте opinionated: знаете продукт — задавайте оптимальные процессы.
- Тестируйте навык как документацию с помощью evals.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Supabase, MCP (Model Context Protocol), Braintrust.
- Claude Sonnet 4.6, Opus 4.6, GPT 5.4, GPT 5.4 mini, Codex.
- Подход: документация через SSH как файловая система для агентов.
- PostgreSQL Row Level Security (RLS).
- Упомянут Vercel skills package (в ответе на вопрос из зала).

## 5. Что полезно разработчику
Практические правила структурирования навыков: не копировать доку, а направлять к ней; держать самое важное в основном файле; задавать рекомендованные процессы; проверять поведение агента через evals.

## 6. Что можно применить в агентных продуктах
- Комбинация «инструменты (MCP) + инструкции (skills)» значительно улучшает выполнение задач.
- Модель подачи документации как удалённой файловой системы (SSH) для навигации агентом.
- Eval-тестирование поведения агента как часть CI документа-навыка.
- Принцип итеративной доработки навыка от минимальной версии.

## 7. Стоит ли смотреть полностью: maybe
Доклад даёт конкретные уроки по написанию навыков и результаты замеров, но часть материала может быть знакома тем, кто уже работал со skills. Полезен для старта, меньше — для глубоко погружённых.
