## 1. О чём доклад

Доклад о создании и развёртывании голосовых агентов с открытым исходным кодом. Рассматривается фреймворк Pipecat (Python, vendor‑neutral), его архитектура пайплайнов, проблемы быстродействия (целевая задержка 800 мс от голоса до голоса), определение окончания речи, холодные старты и масштабирование. Представлен Pipecat Cloud – тонкий слой поверх глобальной real‑time инфраструктуры Daily, решающий задачи деплоя и автоскейлинга.

## 2. Главные идеи

- Голосовой агент требует трёх частей: кода, развёртывания и подключения пользователей (сеть/телефония).
- Pipecat – open source, vendor‑neutral фреймворк со встроенными компонентами для turn detection, обработки прерываний, управления контекстом и function calling.
- Pipecat Cloud упрощает деплой: оптимизирован для real‑time, решает холодные старты, автоскейлинг, глобальное размещение и приватность данных.
- В экосистеме: открытая модель Smart Turn (бесплатно в облаке) и коммерческая Crisp для подавления фонового шума.
- Speech‑to‑speech модели (Moshi, Sesame, Ultravox) – будущее, но пока уступают цепочке STT→LLM(text)→TTS в надёжности инструкций и вызовах функций.
- Выбор между OpenAI и Gemini: оба сильны в текстовом режиме, Gemini значительно дешевле и лучше в native audio input.

## 3. Практические выводы

- Используйте готовые решения фреймворка для сложных голосовых задач, не переписывайте turn detection и обработку прерываний.
- Добивайтесь полного времени ответа до 800 мс; при географии, далёкой от inference‑серверов, размещайте агента рядом с ними или задействуйте локальные open‑weight модели.
- Для enterprise‑сценариев с жёстким следованием инструкциям и function calling пока предпочтительнее связка STT → LLM (text) → TTS. Speech‑to‑speech применяйте для нарративных, игровых или особо естественных диалогов.
- Сравнивайте модели на собственных евалах; тестируйте Gemini как более экономичный, но проверяйте его поведение.
- Для холодных стартов и непредсказуемой нагрузки используйте готовый облачный хостинг с автоскейлингом (Pipecat Cloud).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- Pipecat (open source, pipeline‑архитектура на Python)
- Daily (глобальная real‑time инфраструктура)
- Pipecat Cloud (обёртка вокруг Docker/Kubernetes)
- Smart Turn (открытая модель определения окончания речи, хостинг через FAL)
- Crisp (коммерческая модель шумоподавления, бесплатна в Pipecat Cloud)
- OpenAI (GPT‑4o, real‑time API, GPT‑4 audio preview)
- Google Gemini (2.0 Flash, Multimodal Live API, native audio)
- Twilio, Pivo (телефония)
- Moshi (Kyutai), Sesame, Ultravox (speech‑to‑speech модели)
- Подходы: vendor neutrality, цепочка STT→LLM→TTS, speech‑to‑speech, function calling, turn detection.

## 5. Что полезно разработчику

- Готовые battle‑tested компоненты: turn detection, прерывания, контекст, function calling.
- Простая смена провайдеров (TTS, STT, LLM, телефония) – 60+ сервисов.
- Переключение между «chained» и «real‑time» режимом OpenAI парой строк кода.
- Открытая модель Smart Turn и встроенная поддержка Crisp для шума.
- Pipecat Cloud убирает возню с Kubernetes: fast start, автоскейлинг, глобальный деплой.
- Клиентские SDK (JavaScript, React, iOS, Android) для мультимодальных приложений.

## 6. Что можно применить в агентных продуктах

- Пайплайн‑архитектура для маршрутизации аудио‑фреймов и параллельной работы нескольких LLM‑агентов (пример: игра с LLM‑судьёй).
- Интеграция с телефонией через Twilio, Pivo для массовых голосовых ботов.
- Быстрое прототипирование с заменой моделей без переписывания логики.
- Решения холодных стартов и автоскейлинга для production‑агентов с непредсказуемой нагрузкой.

## 7. Стоит ли смотреть полностью: maybe

Доклад полезен для инженеров голосовых агентов, но основная информация сжато передана в саммари. Смотреть целиком стоит ради Q&A с тонкостями выбора моделей и географической задержки.
