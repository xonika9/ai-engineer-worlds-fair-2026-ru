## 1. О чём доклад

Видео — запись второго дня AI Engineer World’s Fair 2026. Основная тема — переход от чат-моделей к автономным агентам, работающим в длинных циклах. Ключевые доклады охватывают:
- новый класс моделей Anthropic (Mythos/Fable) и практики взаимодействия с ними;
- верификацию кода и агентных систем (Sonar);
- perceptual-агентов для браузерных и десктопных приложений (Amazon AGI);
- эволюцию разработки и появление «бесплатного кода» (Google DeepMind);
- продакшен-оценки, observability и memory harness для длительных агентных задач.

## 2. Главные идеи

- **Модели выросли, а «сбруя» осталась старой.** Чтобы раскрыть способности нового класса моделей (Fable), нужно пересмотреть системные промпты — они стали меньше, без жёстких примеров, чтобы не ограничивать воображение модели. Модели становятся «биологией» — их поведение эмпирично, и нужен новый подход к поиску неизвестных (`blind spot pass`, интервью, реализационные заметки).
- **«Цикл — это продукт».** Автономные агенты ценны не разовым ответом, а способностью устойчиво выполнять рабочие процессы. Петли «наблюдение-планирование-действие» и «генерация-верификация-улучшение» становятся основной архитектурой.
- **Верификация должна быть встроена в процесс,** а не выполняться постфактум. Многоуровневая верификация (zero-trust, algorithmic + agentic) даёт до 92% сокращения проблем в агентом коде.
- **Perception-агентам нужен разделяемый контекст.** Агенты, работающие с интерфейсами, должны не просто кликать, а непрерывно воспринимать экран и реагировать в реальном времени, как человек.
- **Оценка (eval) смещается с модели на систему.** Важны сценарно-ориентированные тесты, метрики надёжности, а не только accuracy. Agent-as-a-judge дополняет LLM-as-a-judge для ловли сложных паттернов отказов.
- **Память агента — это управляемый цикл «запись-управление-чтение».** Для длительных задач простого RAG недостаточно; решающее значение имеет политика recall (ранжирование, учёт важности и источника фактов), а не только объём хранимого.
- **«Исследование автоматизируется, человек движется вверх по стеку».** Авто-исследования (auto research) уже побеждают людей в ML-соревнованиях, но прорывные идеи всё ещё исходят от людей; задача инженера — проектировать хорошие абстракции и функции оценки.

## 3. Практические выводы

- **Системный промпт для мощных моделей:** убирайте избыточные примеры, давайте контекст, а не ограничения. Используйте `blind spot pass` для поиска неучтённых проблем в кодовой базе.
- **Встраивайте верификацию в цикл разработки.** Определите контекст и ограничения до начала генерации кода агентом; комбинируйте алгоритмические проверки (data flow, control flow) и агентическую верификацию (логика, неизвестные).
- **Для UI-агентов применяйте perceptual loop:** дайте агенту видеть не DOM, а реальный рендеринг (скриншоты/пиксели), и замкните цикл «восприятие → действие → подтверждение».
- **Разделяйте задачи между кодом и агентом:** если задача имеет точный ответ — используйте детерминированный код; если нужна оценка или интерпретация — делегируйте агенту; человек оставляет за собой финальное решение.
- **Стройте агентов с управляемой памятью.** Введите политику recall: ведите лог принятых решений, присваивайте важность воспоминаниям, различайте факты и слухи, сохраняйте источник информации.
- **Оценивайте агентов через сценарии и продакшен-телеметрию.** Переходите от оценки ответов к оценке рабочих процессов. Собирайте все traces, внедряйте continuous monitoring, используйте продакшен-трафик как оценочные данные.
- **Применяйте «рецепты» (recipes) как воспроизводимую упаковку агентных систем:** версионируйте промпты, оценки, навыки, инструменты и профили обвязки — это становится новым «исходным кодом».

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Anthropic:** Claude Code, Claude Fable (Mythos-класс), Opus 4.x, серия статей по unhobling.
- **Sonar:** Sonar Vortex, подход AC/DC (Agent-Centric Development Cycle), multi-layered verification.
- **Amazon AGI:** perception agent harness (open-source), annotation tool (Chrome extension), verification tool.
- **Google DeepMind:** Gemini (Omni, Nano Banana), AI-ассистент Pitchfork, идея self-play для кода.
- **Arise:** платформа для evals, продукт Signal (агент-судья для traces).
- **Arena:** Arena.ai, Agent Arena, Chatbot Arena, causal inference для сравнения агентов.
- **Weights & Biases:** Arya (AI research agent), Weave (observability и eval), Launch (оркестрация GPU-задач).
- **OpenGV:** OG Assist, Effect-based agent loop, A2A-протокол, sandboxing.
- **Sakana AI:** memory harness с политиками recall (ранжированный ledger), локальные модели (Qwen, DeepSeek).
- **Meta:** фреймворк продакшен evals: сценарии, продакшен telemetry, continuous evaluation.
- **Elastic:** test-time compute для retrieval, multi-pass algebra над эмбеддингами, инструменты Data Room, Search Box, Knowledge Graph.
- **Resonate:** durable execution platform, подход «prompt is a platform», генерация серверов из спецификаций.
- **Prime:** авто-исследовательские агенты (Codex vs Claude) на speedrun-бенчмарках.
- **Recursive:** концепция Eureka Machine, RSI, auto research на nanoGPT и CUDA kernel.
- **Cognition, Cursor, Copathy, Polygraph:** упомянуты в контексте продуктивных агентных систем.

## 5. Что полезно разработчику

- **Perception agents (Amazon AGI):** подход с разделяемым контекстом через скриншоты и непрерывную петлю «восприятие-действие» прямо применим к браузерным и офисным агентам. Annotation tool с Chrome extension — готовый пример для повторения.
- **Harness:** Effect-based agent loop (OpenGV) — пример миграции с LangGraph на собственный лёгкий цикл с полным контролем, трейсингом и structured concurrency. Polygraph — мета-обвязка для многорепозиторных агентов.
- **Sandboxes:** cloud sandboxes для мобильной разработки (загрузка VM за 30 сек), GPU sandboxes (Prime) — критичны для безопасного выполнения кода агентами.
- **Memory harness (Sakana AI):** дизайн памяти как лэддер политик (no recall → RAG → decision ledger → oracle) и вывод, что качество recall-политики влияет на стоимость и успех длинных задач.
- **Evals & observability:** Agent-as-a-judge (Arise Signal) для поиска скрытых паттернов отказов; Weave для непрерывного мониторинга агентных сессий; Arena как источник живых метрик для бенчмаркинга.
- **Tool execution:** A2A-протокол для взаимодействия агентов, концепция «skills» (Agents SDK) и «recipes» (версионируемые папки с промптами, оценками и обвязкой).
- **Developer tools:** Claude Code с новыми возможностями (сокращённый системный промпт, ask-user-question), Codex, Cursor.

## 6. Что можно применить в агентных продуктах

- **Рецепты (recipes) как формат агентных продуктов.** Упаковывать обвязку, промпты, оценки и профили инструментов в версионируемые git-репозитории, чтобы агенты студии были воспроизводимыми и могли улучшаться со временем.
- **Механизм memory harness с ранжированным recall.** Для длительных сессий внедрить ledger принятых решений и политику «ранжированного возврата», отделяя факты от слухов и сохраняя источники.
- **HTML как универсальный медиум для генерации.** Использовать HTML/CSS для создания слайдов, отчётов и даже видео — модель мыслит в терминах структуры, а не пикселей, и результат легко кастомизировать.
- **Подход «blind spot pass» для итеративного улучшения промптов.** Периодически просить модель просканировать кодовую базу или документацию и выявить неизвестные, которые мы упустили в спецификации.
- **Верификация внутри цикла.** Не проверять результат работы агента только в конце, а встроить algorithmic (статический анализ, тесты) и agentic (оценка бизнес-логики) проверки в каждый шаг рабочего процесса.
