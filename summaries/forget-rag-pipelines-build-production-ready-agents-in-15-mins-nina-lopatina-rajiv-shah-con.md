## 1. О чём доклад  
Как построить production-ready RAG-агента за 15 минут с помощью платформы Contextual AI: извлечение из сложных документов, гибридный поиск с rerank, генерация на основе grounded-модели и проверка фактичности. Демонстрация через ноутбук и веб-интерфейс, а также оценка качества и интеграция через MCP.

## 2. Главные идеи  
- RAG можно использовать как управляемый сервис, не собирая пайплайны из отдельных компонентов.  
- Сквозная платформа для production: парсинг PDF (таблицы, изображения, заголовки), чанкинг, гибридный поиск (BM25+векторы), instruction-following reranker и собственная grounded LLM.  
- Все части (parsing, rerank, генерация, оценка LMUnit) доступны как отдельные API и могут встраиваться в существующие системы.  
- Grounded LLM обучена отвечать строго по контексту и отличать факты от комментариев.  
- Встроенная проверка утверждений (groundness check) подсвечивает неподтверждённые предложения и даёт атрибуцию с bounding boxes.

## 3. Практические выводы  
- Не нужно изобретать RAG с нуля – мнения по умолчанию и гибкие настройки закрывают потребности и бизнес-пользователей, и разработчиков.  
- Оценка через LMUnit (набор юнит-тестов на естественном языке) позволяет быстро найти слабые места агента и итеративно их исправлять.  
- Готовый MCP-сервер делает агентов доступными в сторонних клиентах (Claude Desktop, Cursor).  
- Стартовый кредит $25 даёт возможность протестировать платформу без риска.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Contextual AI (платформа и SDK для Python/JavaScript).  
- Компоненты: parsing (table extraction mode, image captioning), instruction-following reranker, grounded generation, LMUnit (natural language unit tests).  
- Клиент: Qualcomm (public‑facing поиск по документации).  
- Интеграция: MCP-сервер для подключения к Claude Desktop.  
- Партнёрство со Snowflake.  
- Упомянуты OpenAI/Gemini как контраст для grounded‑модели.

## 5. Что полезно разработчику  
- Можно стартовать с одного API‑ключа, без оркестрации векторных баз и десятка моделей.  
- Компоненты используются независимо: если не устраивает свой reranker или extraction, их можно заменить по отдельности.  
- Гибкие настройки агента: реформулировка запросов, query decomposition, фильтрация по метаданным, кастомный промпт для reranker, управление контекстом.  
- Атрибуция ответов с указанием на исходные фрагменты и визуальными bounding boxes.  
- MCP‑сервер позволяет легко превратить RAG‑агента в инструмент для AI‑ассистентов.

## 6. Что можно применить в агентных продуктах  
- Использовать grounded‑генерацию, чтобы исключить выдумки при ответах на основе корпоративных документов.  
- Внедрить RAG‑агента в десктопные или IDE‑ассистенты через MCP (показан пример с Claude Desktop).  
- Автоматически оценивать качество ответов с помощью LMUnit по критериям вроде точности, умения отличать корреляцию от причинности, избегания лишней информации.  
- Настраивать entitlements‑слой для управления правами доступа к документам (в дорожной карте).  
- Автоматически подгружать обновляемые документы через ingestion pipeline.
