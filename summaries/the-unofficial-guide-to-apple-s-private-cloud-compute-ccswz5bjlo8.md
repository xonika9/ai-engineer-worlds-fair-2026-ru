## 1. О чём доклад
Неофициальный разбор Apple Private Cloud Compute (PCC) — системы конфиденциального инференса ИИ на удалённых серверах, доступной на iPhone. Спикер, не сотрудник Apple, на основе публичных источников объясняет архитектуру, шесть ключевых технических компонентов, ограничения и показывает, какие из этих подходов можно применить вне экосистемы Apple.

## 2. Главные идеи
- Удалённый инференс превращает сервер в чёрный ящик; Apple отвечает на вопрос *«how do you get remote compute while remaining private»* и экономически доступным.
- Пять требований: вычисления без сохранения состояния, принудительные гарантии (не политика, а отсутствие возможностей — *«no SSH on the instance. You can't SSH to it»*), невозможность таргетирования конкретного пользователя, отсутствие привилегированного доступа во время исполнения и проверяемая прозрачность.
- Архитектура ставит между iPhone и AI-движком анонимайзер (Oblivious HTTP через Cloudflare), слепые подписи для отделения аутентификации от личности, Secure Enclave для ключей и удалённую аттестацию: сервер предъявляет подписанные утверждения о загрузочной цепочке и коде, клиент проверяет их по публичному журналу прозрачности и только потом шифрует данные открытым ключом, привязанным к этой аттестации. Расшифровать можно только при неизменном окружении.
- Нет диска, нет SSH, нет логирования — всё завязано на аппаратно-криптографические гарантии.

## 3. Практические выводы
- Полное доверие к производителю железа и оператору ЦОД сохраняется (Apple сама управляет поставками и переподписывает узлы), однако атака на всю систему требует компрометации всех узлов одновременно.
- Платная цена приватности: выше задержки, дороже вычисления, невозможны кастомные модели и точный учёт использования, усложнённая клиентская логика.
- Журналы прозрачности (SIGStore/SIGSUM), воспроизводимые сборки и открытый код позволяют независимо связать исходный код с бинарниками — это сильнее, чем просто верить внутреннему логу Apple.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Oblivious HTTP (совместная спецификация Cloudflare и Apple), слепые подписи (открытые библиотеки).
- Secure Enclave, TPM, виртуальные TPM в облаках.
- Журналы прозрачности: SIGSUM, SIGStore.
- Конфиденциальные вычисления с GPU: NVIDIA H100/H200 (шифрование памяти).
- Apple PCC, Azure AI private inferencing (закрытый бета‑тест, сентябрь 2024), Meta WhatsApp private processing (копирует архитектуру Apple).
- Подходы: удалённая аттестация, безопасная загрузка, изолированный рантайм без SSH и постоянного хранилища.

## 5. Что полезно разработчику
- Oblivious HTTP и слепые подписи доступны как библиотеки — можно анонимизировать API‑запросы.
- TPM (физический или виртуальный) для привязки приватных ключей к «железу» вместо софтового хранения.
- Журналы прозрачности (SIGSUM/SIGStore) для публикации и независимой верификации релизов.
- Конфиденциальные виртуальные машины с H100/H200 — база для инференса с шифрованием оперативной памяти GPU.
- Комбинировать TPM‑замеры с проверкой по логу прозрачности для примитивной, но рабочей удалённой аттестации.

## 6. Что можно применить в агентных продуктах
- Анонимизация через Oblivious HTTP + слепые подписи скрывает, какой именно пользователь инициировал вызов к инструментам агента.
- Конфиденциальное выполнение без логирования и диска гарантирует, что переданные агентом данные не накапливаются и не могут быть использованы для обучения или аудита на стороне сервера.
- Публичный журнал прозрачности позволяет агенту (или стороннему аудитору) перед отправкой чувствительных данных убедиться, что код сервера не подменён.
