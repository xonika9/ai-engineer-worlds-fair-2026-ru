## 1. О чём доклад  
Доклад о концепции контекстных графов (context graphs) — как объединить разрозненные корпоративные данные, память агентов и следы их рассуждений, чтобы ИИ-системы принимали обоснованные и объяснимые решения. Стивен Чин (Neo4j) показывает, как выйти из «матрицы» фрагментированного знания и построить сквозной контекст для агентных приложений.

## 2. Главные идеи  
- Без единого контекста агенты дают слабые ответы; контекстный граф связывает сущности, события и причины.  
- Три слоя памяти: краткосрочная (текущий пайплайн и диалог), долгосрочная (организованная история) и следы рассуждений (reasoning traces) для аудита и отладки.  
- Комбинация LLM (язык, креативность) с графами знаний (контекст, обогащения) даёт объяснимые результаты.  
- Контекстный граф фиксирует не только «что», но и «почему» было решено, помогая комплаенсу и повторяемости.

## 3. Практические выводы  
- GraphRAG с извлечением по связям даёт более полные и релевантные ответы, чем классический RAG.  
- Открытый пакет Neo4j agent memory упрощает добавление памяти в агентные архитектуры.  
- Демонстрация для финансовой сферы: система сама видит историю клиента, предыдущие отказы, риски и выдаёт подкреплённое заключение с объяснением причин.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Neo4j (графовая БД), Neo4j Aura (облачный сервис).  
- Neo4j agent memory (open-source).  
- Cypher (язык запросов к графам).  
- GraphRAG, knowledge graph, context graph.  
- Gartner (включил context graphs в хайп-цикл).  
- Foundation Capital ($3T возможность).  
- Next.js, MCP tools, OpenAI embeddings, алгоритмы FastRP и Louvain.  
- GraphAcademy (бесплатный курс с экземпляром Aura).

## 5. Что полезно разработчику  
- Готовые шаблоны интеграции памяти в агентов: связка Cypher + векторный поиск + алгоритмы сообществ.  
- Открытый код и бесплатная песочница Aura для экспериментов.  
- Подходы к построению объяснимых и аудируемых агентных решений.

## 6. Что можно применить в агентных продуктах  
- Единый слой контекста для кросс-доменных сценариев (тикеты, CRM, внутренние системы) через MCP‑инструменты.  
- Накопление следов рассуждений для улучшения будущих решений и отладки.  
- Генерация детальных обоснований (факторы риска, предыдущие решения) напрямую в интерфейсе пользователя.

## 7. Стоит ли смотреть полностью: maybe  
Доклад вводный и насыщен демонстрациями. Хорош для первого знакомства с контекстными графами, но опытным инженерам может не хватить глубины. Полезные ссылки на ресурсы.
