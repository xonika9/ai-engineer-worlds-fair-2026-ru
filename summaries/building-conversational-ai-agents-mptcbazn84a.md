## 1. О чём доклад
Доклад-воркшоп от разработчика ElevenLabs о создании мультиязычных разговорных AI‑агентов. Разбирается пайплайн «речь → текст → LLM → синтез речи», встроенная система определения и переключения языков, интеграция инструментов, а также демонстрируется настройка агента через дашборд и SDK.

## 2. Главные идеи
- Конвейер строится на связке ASR → LLM → TTS с передачей по тексту, что обеспечивает прозрачность и упрощает мониторинг.
- Встроенный system tool для детекции языка позволяет агенту автоматически переключаться между 31 языком (в разработке до 99).
- Каждому языку можно назначить отдельный голос (вплоть до регионального акцента) из библиотеки >5000 голосов, за использование которых авторы получают роялти.
- Логика агента расширяется function calling; поддерживаются как публичные LLM (GPT‑4o, Gemini 2.5, Flash), так и кастомные модели через OpenAI‑совместимый эндпоинт.
- Безопасность обеспечивается водяными знаками, модерацией и возможностью блокировать нежелательные слова для конкретного голоса.

## 3. Практические выводы
- Быструю конфигурацию агентов удобно начинать с дашборда, а при массовом создании или маркетплейсах — полностью управлять через API.
- Для снижения задержки стоит выбирать лёгкие LLM (Gemini Flash) и следить за timeout’ами инструментов; встроенные филлеры сглаживают ожидание.
- Сложные сценарии с несколькими задачами решаются agent‑to‑agent transfers, незаметно для пользователя.
- Кастомизация произношения достигается словарями произношения в TTS; для улучшения распознавания аббревиатур в ASR пока помогает нормализация через system prompt LLM.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- ElevenLabs: ASR‑модель Scribe, TTS‑модели, Sound Effects, Voice Library, Marketplace, MCP‑сервер, SDK (JavaScript/Python).
- LLM-провайдеры: OpenAI GPT‑4o, Google Gemini 2.5/Flash, возможность подключать Vertex AI или собственные модели.
- Интеграции: cal.com (планирование встреч), Nvidia Tokio (аватар‑стек, lip sync), партнёры Hedra/HeyGen (аватары), Supernova (изучение английского).
- Подходы: function calling, RAG, агент‑агентные трансферы.

## 5. Что полезно разработчику
- Пайплайн можно встроить в приложения через React/Next.js или Python (вплоть до Raspberry Pi). Конфигурация и развёртывание — через API или MCP‑сервер на естественном языке.
- Широкая кастомизация: выбор LLM, подключение внешних инструментов, управление голосами и языками через веб‑интерфейс или код.
- Инструменты безопасности (вотермаркинг, верификация клонирования голоса) делают платформу пригодной для продуктовых проектов.

## 6. Что можно применить в агентных продуктах
- Мультиязычные голосовые боты (службы поддержки, медицинские приёмные, языковое обучение) с плавным переключением языков и региональных акцентов.
- Оркестрация через трансферы между специализированными агентами (отдельные команды могут независимо развивать части сценария).
- Агенты с доступом к корпоративным API (CRM, календари) через server‑side tools.
- Аватар‑решения с озвучкой ElevenLabs (совместно с партнёрами по визуализации).
