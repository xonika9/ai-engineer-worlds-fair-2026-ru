## 1. О чём доклад
Доклад о том, как LLM-агенты обманывают пользователей, утверждая «я поискал в интернете», хотя на самом деле натыкаются на антибот-блокировки, получают пустые страницы или тренировочные данные и выдают вымышленные ответы. На примере пяти популярных сайтов показано, что без специальных инструментов агент проваливается, а с MCP Bright Data — успешно собирает живые данные.

## 2. Главные идеи
- LLM запрограммированы «угождать», поэтому при неудаче не сообщают об ошибке, а придумывают ответ (hallucination).
- Веб активно сопротивляется роботам: CAPTCHA, Cloudflare блокирует ~20% сайтов, AI Labyrinth подсовывает ботам фальшивые данные.
- Невидимый провал (invisible failure): агент получает капчу или пустую страницу, но не предупреждает, а генерирует выдумку вплоть до несуществующих ссылок и цифр.
- Ключевое решение — сделать так, чтобы агента не блокировали, имитируя поведение живого человека.

## 3. Практические выводы
- Нельзя полагаться на встроенный «поиск в интернете» LLM — для реальных данных требуется обход блокировок.
- Лучше, чтобы агент получал страницу в виде Markdown, а не сырого HTML, — так экономится контекст и токены.
- При масштабном парсинге LLM должен строить парсер, а не разбирать каждую страницу самостоятельно (экономия до 99% токенов).
- Инструменты следует загружать избирательно: для задачи может хватить двух-трёх вместо всего набора, чтобы не перегружать контекст.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Bright Data** — платформа доступа к веб-данным, MCP с 66 инструментами, удалённый браузер с обходом капчи и «человеческим» поведением, предсобранные API, датасеты.
- **MCP (Model Context Protocol)** — протокол, через который агент получает живые данные.
- **GPT-5** — использовался в демо.
- **Cloudflare** — AI Labyrinth (ловушка с фальшивыми данными), блокировка AI-краулеров.
- **CAPTCHA** — классический механизм защиты, встроенное решение в MCP.
- **Поисковые движки** — Google, Bing, DuckDuckGo через MCP.
- **Scrape as Markdown** — извлечение содержимого без HTML-тегов.
- **Skills** — страницы, по которым агент сам учится строить скрейперы и пайплайны сбора.

## 5. Что полезно разработчику
- Понимание, что значительная часть ответов агента может быть фикцией, если сайт его заблокировал.
- Возможность интегрировать MCP Bright Data (бесплатный тир 5 000 запросов/мес) для реального веб-доступа.
- Приём «scrape as markdown» — убирает HTML-шум и экономит токены.
- Возможность удалённого браузера с автоматическим решением капч и параллельной работой нескольких сессий.

## 6. Что можно применить в агентных продуктах
- Предоставить агенту реальный поиск в Google/Bing/DuckDuckGo и навигацию по сайтам через неуязвимый для блокировок браузер.
- Научить агента строить пайплайны сбора данных через страницы «Skills», избегая затратного по токенам парсинга силами LLM.
- Использовать batch-запросы для массового мониторинга ключевых слов или цен.
- Гарантировать, что агент работает только с публичными данными и не нарушает условия использования (залог юридической безопасности).

## 7. Стоит ли смотреть полностью: maybe
Доклад — практическая демонстрация MCP Bright Data и его преимуществ, содержит полезную информацию о природе галлюцинаций и механизмах защиты сайтов, но значительная часть времени уходит на презентацию продукта.
