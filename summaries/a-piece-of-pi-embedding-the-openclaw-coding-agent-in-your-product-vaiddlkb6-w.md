## 1. О чём доклад  
Доклад о том, как встраивать кодингового агента (Pi/OpenClaw) в собственные продукты. Рассмотрена архитектура агентов от ядра до многоканальных систем, практический пример автоматизации обработки почтовых запросов с генерацией черновиков ответов.

## 2. Главные идеи  
- Кодинговые агенты становятся фундаментальным строительным блоком для ПО.  
- Агент — это LLM, в цикле вызывающая инструменты; при добавлении runtime и shell (Bash) получается кодинговый агент.  
- Pi — минимальный опенсорсный агент, идеальный для экспериментов и «разборки на части».  
- OpenClaw использует ядро Pi для многоканальной маршрутизации, сессий и суб-агентов.  
- Архитектурный паттерн: *«make it easy for coding agents»* – оборачивайте доступ к системам (CRM/ERP) в CLI-инструменты.  
- Безопасность (sandboxing) – перспективное направление, например, Nvidia Open Shell.

## 3. Практические выводы  
- Начинайте с ядра Pi, воссоздайте логику агента с вызовами инструментов в цикле.  
- Используйте хуки `beforeToolCall` для ролевого доступа и события для аудита действий.  
- CLI – удобный интерфейс для взаимодействия агента с бэкенд-системами.  
- Сессии позволяют сохранять контекст длительных процессов (agents per customer).  
- Конечный результат можно встроить прямо в почту (черновик), не требуя сложного UI.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Pi (минимальный кодинговый агент, TypeScript)  
- OpenClaw (многоканальная платформа на Pi)  
- Cohere Claude desktop (пример с Excel-скиллом)  
- FFmpeg, Pandas, OpenPyXL, LibreOffice  
- Bash  
- CRM, ERP  
- Nvidia Open Shell  
- Архитектурный подход: make it easy for agents, CLI-обёртки, сессии

## 5. Что полезно разработчику  
- Агент предельно прост: LLM, цели, вызовы инструментов в цикле; легко воспроизвести.  
- Хуки beforeToolCall дают возможность встраивать enterprise-политики (role-based access).  
- События (stream) облегчают мониторинг и отладку.  
- Расширения Pi позволяют встраивать интерактивный UI (select, дропдауны) даже в терминале.  
- Pi разделён на пакеты: `agent-core`, `coding-agent`, `pi-ai` (LLM-абстракция), `terminal-ui`.  
- OpenClaw добавляет плагины для маршрутизации, оркестрации провайдеров и суб-агентов.

## 6. Что можно применить в агентных продуктах  
- Мульти-агентная архитектура: отдельный агент на клиента, gateway для разбора входящих писем.  
- Сохранение контекста через сессии (Pi sessions) для многократных взаимодействий.  
- Генерация черновиков email как финальный артефакт; пользователь остаётся в привычной почте.  
- Интеграция с почтовым ящиком и автоматическая классификация запросов.  
- Единый слой абстракции LLM (`pi-ai`) для гибкой смены моделей.
