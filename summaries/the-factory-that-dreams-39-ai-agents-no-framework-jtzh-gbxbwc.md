## 1. О чём доклад
Рассказ Рашаба Доши (Machinecraft) о том, как их фабрика из 100 человек без data science-команды построила систему из 36 специализированных AI-агентов, которая управляет всем процессом привлечения клиентов (go-to-market). «Мозг» компании собирается из исторических данных без обучения моделей, использует память, «сны» и принципы семейного бизнеса, а затем оформлен в открытый продукт Brain OS.

## 2. Главные идеи
- Знания компании не должны храниться только в головах нескольких людей — иначе они утекают с каждым уходом сотрудника.
- Вместо одного всезнающего промпта нужен «пантеон» узких агентов, каждый с одной чёткой ролью (Афина управляет, Прометей продаёт и т.д.).
- Архитектура без обучения: данные нарезаются на куски, превращаются в векторы и отношения, а готовые LLM извлекают факты.
- Многослойная память с фильтром значимости (salience gate), ночными циклами «сновидений» для консолидации опыта и исправления противоречий.
- «Файл души» — набор инженерных правил на основе джайнской философии: перепроверять, ссылаться на документы, не работать в одиночку, честно докладывать правду.
- Человек остаётся в контуре: агенты готовят черновики, отправляет человек — *Eira drafts, human sends*.

## 3. Практические выводы
- Построить собственную систему агентов можно без ML-инженеров и огромных бюджетов: авторское решение обошлось в ~$30k вместо предложенных агентством $230k.
- Память важнее «умной» модели: несколько слоёв (рабочая, факты, эпизоды, отношения) с автоматической чисткой дают связное поведение и предотвращают галлюцинации.
- Разделение на 36 ролей с перекрёстными совещаниями делает систему надёжнее одного мега-агента.
- Ночной цикл консолидации превращает сырой опыт в переиспользуемые навыки — утром приходит отчёт о «снах».
- Золотое правило «черновик от ИИ, отправляет человек» снижает риски и сохраняет контроль.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Машины термоформовки (продукт компании, контекст).
- Brain OS — форкаемая архитектура «пустая нервная система» (агенты, память, цикл сна, файл души).
- `forkmybrain.org` — сайт проекта.
- Google Workspace, системы коммуникаций, CRM, векторные и графовые базы данных.
- Три разных провайдера LLM, подобранных под конкретные задачи.
- Мониторинг для отслеживания «мыслей» системы.
- 213 инструментов, доступных через один протокол.

## 5. Что полезно разработчику
- **Многоагентная архитектура**: чёткое разделение ролей (один агент — одна задача) и их координация через внутренние «совещания».
- **Memory layers**: salience gate, эпизодическая память и цикл сна как способ борьбы с забыванием контекста — прямое применение в браузерных/офисных агентах, где длинные сессии быстро размывают контекст.
- **Tool execution**: 213 инструментов по одному протоколу — урок для построения extensible tool harness; все инструменты сперва создают черновик, санкционирует человек (принцип «draft, human sends»).
- **Observability**: постоянный мониторинг мышления агентов позволяет проводить evals и отладку без спекуляций.
- **Guardrails как файл души**: формализованные принципы вместо общих «будь полезен» — применимо к песочницам, где агентам нужны чёткие границы.

## 6. Что можно применить в агентных продуктах
- Внедрить многослойную память с ночным циклом консолидации для агентов, работающих с историей проектов или документами.
- Использовать ролевую модель агентов (аналитик, критик, сборщик) вместо одного универсального промпта, реализовав их «совещания».
- Все опасные операции (рассылка, коммит) реализовывать по шаблону *черновик → подтверждение человеком*.
- Файл души как конфиг поведенческих ограничений: проверять факты, ссылаться на источники, эскалировать неопределённость.
