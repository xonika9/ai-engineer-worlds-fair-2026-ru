## 1. О чём доклад
Фуад Матин из OpenAI делится опытом защиты агентов, способных писать и выполнять код. На примере Codex CLI и Codex в ChatGPT разбираются ключевые риски (промпт-инъекции, утечка данных, вредоносные пакеты, выход из песочницы) и три главных принципа безопасности: строгая изоляция, ограничение доступа в интернет и обязательный человеческий контроль.

## 2. Главные идеи
- *Every agent will become a code‑executing agent* – модели вроде O3/O4-mini используют код не только для разработки, но и для OCR, обрезки изображений без явных указаний.
- Сложные цепочки агентов заменяются одной reasoning‑моделью, которая сама выбирает инструменты и пишет код.
- Основные угрозы: эксфильтрация через промпт-инъекции, непреднамеренная установка вредоносных пакетов, генерация уязвимого кода, повышение привилегий.
- Детерминированная защита на системном уровне (песочница, сетевые политики, human‑in‑the‑loop) надёжнее, чем только модельные проверки.

## 3. Практические выводы
- Запускать агента в изолированной среде: отдельный контейнер, OS‑песочница (macOS Seatbelt, Linux seccomp/landlock) или режим Codex CLI с ограничением файловой системы и сети.
- Интернет‑доступ либо полностью отключать, либо использовать конфигурируемый allow‑list HTTP‑методов и доменов, иначе агент может стать жертвой инъекций через внешний контент.
- Все изменения обязательно пропускать через человеческий обзор (GitHub PR), не полагаясь только на LM‑мониторинг.
- Перед установкой зависимостей проверять их безопасность через MCP‑серверы вроде Socket.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Codex CLI (open source), Codex в ChatGPT, Responses API, Agents SDK, контейнерный сервис OpenAI.
- macOS sandboxing (Seatbelt/seatbelt), Linux sandboxing (seccomp, landlock), контейнеризация.
- Chromium как пример применения Seatbelt.
- MCP‑сервер Socket для проверки уязвимостей зависимостей.
- API‑инструменты: local shell, apply patch.
- Preparedness Framework (внутренний стандарт OpenAI).

## 5. Что полезно разработчику
- Готовые референсные реализации безопасного агента: Codex CLI с полной изоляцией на уровне ОС, allow‑листом сети и режимом автоматического одобрения команд в песочнице.
- Конкретные техники OS‑sandboxing: политики Seatbelt для macOS, seccomp‑фильтры и landlock для Linux.
- Стратегия отключения интернета по умолчанию и точечного разрешения HTTP‑методов с явными предупреждениями.
- Рекомендация выполнять код агента в remote‑контейнере, а не на локальной машине.

## 6. Что можно применить в агентных продуктах
- Многоуровневую изоляцию: системная песочница + детерминированные сетевые ограничения поверх модельных сигналов.
- Конфигурируемый сетевой allow‑list с предупреждениями о рисках (как в Codex).
- Watch mode (как у Operator) для мониторинга критических операций и принудительного участия человека.
- Интеграцию MCP‑серверов безопасности в цикл работы агента (автоматическая проверка зависимостей перед установкой).
- Принцип минимальных привилегий: агент не должен иметь доступ к чувствительным данным или сети, если задача этого не требует.
