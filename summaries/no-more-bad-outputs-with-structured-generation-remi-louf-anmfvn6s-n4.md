## 1. О чём доклад

Доклад о том, как structured generation (управление выводом языковых моделей по заданной структуре) решает фундаментальную проблему LLM — неспособность гарантированно возвращать валидные форматы (JSON, код, точные ответы). Спикер Remi Louf представляет open-source библиотеку outlines и на примерах показывает, что добавление структуры не только устраняет ошибки, но и ускоряет инференс, повышает точность открытых моделей и делает их конкурентоспособными с GPT-4.

## 2. Главные идеи

- Большинство текстовых задач по своей природе структурированы, поэтому ограничение вывода модели допустимыми токенами (regex, JSON Schema, контекстно-свободные грамматики) даёт гарантированно валидный результат.
- Structured generation не замедляет инференс (оверхед близок к нулю), а зачастую ускоряет его благодаря сокращению числа генерируемых токенов.
- Добавление структуры позволяет небольшим открытым моделям (например, Mistral 3 medium) достигать точности на уровне и выше коммерческих аналогов — 96.5% на бенчмарке function calling против 93.5% у GPT-4.
- Структурирование вывода перераспределяет роль few-shot примеров: модель учится не столько задаче, сколько формату, поэтому одного примера часто достаточно для высокой точности.

## 3. Практические выводы

- Используйте outlines как легковесную библиотеку, а не фреймворк: её можно встроить в существующие пайплайны без перестройки.
- Определяйте структуру через Pydantic-модели, JSON Schema, регулярные выражения или грамматики — и модель не сгенерирует ничего лишнего.
- Генерация структурированного вывода сокращает «болтовню» модели (yapping), экономя токены и время.
- Открытые модели с structured generation способны заменить проприетарные API в задачах извлечения данных, генерации кода и вызовов инструментов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **outlines** — Python-библиотека для structured generation.
- **dottxt** — компания автора, разрабатывающая эффективную индексацию токенов под структуру.
- **vLLM** и **TGI** — используют outlines внутри function calling.
- **Transformers**, **llama.cpp**, **MLX**, **OpenAI** (последняя для сравнения) — поддерживаемые провайдеры моделей.
- **Guidance**, **LMQL** — альтернативные библиотеки.
- **JSON Schema**, **Pydantic**, **regex**, **context-free grammars**, **text-to-SQL**, **vision models**.

## 5. Что полезно разработчику

- Простая установка (`pip install outlines`) и интеграция в любой этап пайплайна.
- Возможность задавать кастомные типы (например, код аэропорта из трёх заглавных букв) прямо в схеме — модель не галлюцинирует формат.
- Поддержка генерации с изображениями: модель возвращает строго описанный JSON, а не свободный текст.
- Незаметный для инференса оверхед и сокращение до 50% объёма генерируемых токенов.

## 6. Что можно применить в агентных продуктах

- Надёжные вызовы инструментов (function calling) даже на небольших моделях — точность до 96.5%, что делает агентов практически пригодными для production.
- Генерация SQL-запросов без галлюцинаций в именах таблиц и колонок: запрос гарантированно выполнится, даже если не вернёт ожидаемого результата.
- Извлечение полётной информации из писем, структурирование ответов от vision-моделей, любые сценарии, где раньше требовалась постобработка регулярными выражениями.
