## О чём доклад

Доклад рассказывает о том, почему TypeScript становится основным языком для прикладного AI, тогда как Python сохраняет позиции в обучении моделей и исследованиях. Спикер прослеживает сдвиг от Python к TypeScript с 2022 по 2025 год, связывая его с появлением coding agents и переходом AI на уровень приложений.

## Главные идеи

- AI перемещается из инфраструктурного слоя (модели, тренировка) в слой приложений, где уже давно доминирует TypeScript.
- Рост coding agents (Cursor, Codex, Lava Cloud Code) в 2024–2025 сделал TypeScript языком по умолчанию для генерации кода, что привело к его лидерству на GitHub.
- Преимущества TypeScript для AI-приложений: *«один язык для всего — агентный цикл, инструменты, бэкенд, UI»*, общая типовая система через Zod, доступ к крупнейшему пакетному менеджеру npm.
- Python остаётся для тренировки и инференса моделей, но агенты и приложения будут собираться на TypeScript.
- Прогноз: *«any application, even the gigantic ones, will be written in TypeScript»*.

## Практические выводы

- Строить AI-агенты на TypeScript, чтобы использовать единый стек и тип-контракты без синхронизации между Python-бэкендом и React-фронтом.
- Продолжать тренировать модели на Python, так как эта ниша сохранится.
- Применять экосистему npm (аутентификация, платежи, UI, инфраструктура) для ускорения интеграции AI-функций.
- Использовать TypesScript как язык, улучшаемый coding-агентами, что повысит качество генерируемого кода.

## Что полезно разработчику

Доклад напрямую не затрагивает браузерные/офисные агенты, sandboxes, harness, evals или observability. Однако тренд на массовое строительство агентов на TypeScript подтверждает, что инструменты и платформы агентов (включая среды исполнения и developer tools) будут появляться именно в этой экосистеме. Это значит, что для агентного продукта может быть стратегически выгодно ориентироваться на TypeScript-агентов, их tool execution и sandbox-среды, а также учитывать единую типовую безопасность.

## Что можно применить в агентных продуктах

- При проектировании агентных возможностей рассмотреть TypeScript как основной язык, чтобы получить единую типовую систему и интеграцию с npm-экосистемой.
- Использовать Zod для описания контрактов инструментов и данных агента, обеспечивая сквозную типизацию от UI до выполнения инструментов.
- Учитывать, что coding-агенты будут генерировать более качественный TypeScript-код, что упрощает интеграцию AI-помощников в саму студию.

## Возможная идея для поста

*«Любое приложение, которое можно написать на JavaScript, в итоге напишут на TypeScript. Даже AI-агенты».* Пост о том, почему агентный слой переезжает на TypeScript и как это меняет правила игры для разработчиков AI-приложений.
