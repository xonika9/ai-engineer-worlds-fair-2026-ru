## 1. О чём доклад
Выступление посвящено **WebMCP** (Web Model Context Protocol) — предложенному веб-стандарту от команды Google Chrome. Доклад объясняет, как дать ИИ-агентам структурированный «набор инструментов» для выполнения действий на сайте, заменяя хрупкие и ресурсоёмкие методы вроде анализа DOM, скриншотов и расчёта координат кликов.

## 2. Главные идеи
- Текущие способы взаимодействия агентов с сайтами (передача всего DOM, скриншоты, координаты) **неэффективны, расходуют много токенов и ломаются** при изменении вёрстки.
- Прежде чем внедрять WebMCP, необходимо **улучшить веб-основы**: семантический HTML, высокую доступность, быструю загрузку и продуманные пользовательские сценарии.
- WebMCP позволяет сайту объявить **меню из структурированных инструментов**, которые агент вызывает напрямую, что резко повышает надёжность и производительность.
- Пользователь может в любой момент **переключиться между ручным управлением и агентом**, а ИИ-агент понимает разнообразные формулировки одной и той же команды.

## 3. Практические выводы
- Начинать внедрение агентной готовности сайта стоит с **доступности, семантики и производительности**, и только потом добавлять WebMCP.
- Для тестирования нужен **Chrome Canary** (версия 146+), включённый флаг `chrome://flags/#web-mcp` и расширение **Model Context Tool Inspector** из Chrome Web Store.
- Обратная связь приветствуется: API на стадии раннего превью, детали реализации могут меняться.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Google Chrome** (команда DevRel) и **DeepMind** (упомянуты как партнёры по стенду).
- **Model Context Tool Inspector** — браузерное расширение для отладки WebMCP-инструментов.
- **Gemini** (модели 1.5 и 3.1) — использовалась в демонстрациях в качестве ИИ-агента.
- **WebMCP** и базовый **MCP** (Model Context Protocol) — объяснено их различие: WebMCP реализует клиентскую часть инструментов для внутрибраузерных агентов.
- Подходы: улучшение **Core Web Vitals**, **семантический HTML**, **accessibility tree**.

## 5. Что полезно разработчику
- Конкретные примеры двух API: **декларативный** (добавление атрибутов к HTML-формам — автоматическая генерация JSON-схемы) и **императивный** (регистрация кастомных инструментов через JavaScript с функцией `execute`, схемой, названием и описанием).
- Рекомендация делать описания инструментов **максимально подробными**, чтобы агент точно знал, когда их вызывать.
- Важно **синхронизировать UI** с вызовами инструментов, чтобы пользователь видел выполняемые действия.

## 6. Что можно применить в агентных продуктах
- Упростить многошаговые сценарии для конечного пользователя: **бронирование билетов, фильтрация товаров, заполнение сложных медицинских или финансовых форм**.
- Обеспечить **смешанный режим работы**, где пользователь часть пути проходит сам, а затем передаёт контроль агенту (и наоборот).
- Дать возможность агенту «продолжать вызывать один и тот же инструмент», пока задача не будет выполнена, обрабатывая разнообразные пользовательские формулировки.
