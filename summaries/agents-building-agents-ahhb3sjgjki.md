## 1. О чём доклад

Доклад Альфонсо Грациано (NearForm) о подходе, в котором AI-агенты (coding agents) помогают итеративно улучшать других AI-агентов. Рассматриваются два основных цикла: оптимизация на основе eval-сьютов (AutoAgent) и анализ «живых» данных от пользователей для поиска и устранения проблем в продакшене.

## 2. Главные идеи

- AI-агенты страдают от недетерминизма, галлюцинаций, задержек и высокой стоимости.
- Золотой датасет и скорреры дают воспроизводимую оценку качества, несмотря на недетерминированность.
- AutoAgent — автономный цикл, в котором coding agent (Claude Code) выдвигает гипотезы, меняет системный промпт, инструменты и код агента, прогоняет eval-сьют и откатывает изменения при регрессиях. На простом агенте точность выросла с 18% до 83% за ~10 итераций; на продакшен-агенте, уже оптимизированном человеком, получено +10% на внутренних бенчмарках.
- Второй контур: сбор трассировок и обратной связи от пользователей (thumbs up/down, комментарии), кластеризация failure modes, анализ корневых причин, приоритизация с экспертами и исправление силами coding agent. Отчёт генерируется раз в спринт.
- Оба контура опираются на Harness Engineering — создание для coding agent окружения с чёткими ограничениями, качественными воротами (линтеры, тесты, evals), observability и управлением контекстом.

## 3. Практические выводы

- Начинайте с золотого датасета и скорреров — это основа для любых улучшений агента.
- Coding agent способен найти оптимизации, которые человек пропустил; важно держать human-in-the-loop для валидации гипотез и приоритизации.
- Трассировки с обратной связью эффективнее обрабатывать пакетно, кластеризуя однотипные сбои, а не разбирая каждый инцидент вручную.
- Снабжайте coding agent возможностью самостоятельно проверять свои изменения (регрессионные тесты на трассировках).

## 5. Что полезно разработчику

Подходы AutoAgent и анализа live-данных напрямую применимы к разработке AI-агентов любых типов — браузерных, офисных, инструментальных. Harness Engineering даёт шаблон для построения sandbox-окружений и циклического исполнения инструментов с observability и evals. Для observability и developer tools подсвечена важность трассировок и автоматической кластеризации failure modes. Агентный продукт может использовать эти методики для итеративной шлифовки собственных агентов.

## 6. Что можно применить в агентных продуктах

Внедрить цикл AutoAgent с coding agent’ом, который по расписанию прогоняет eval-сьют и предлагает PR с улучшениями промптов и логики инструментов. Добавить сбор пользовательской обратной связи в трейсы, периодическую кластеризацию проблем и генерацию отчёта для приоритизации фиксов. Сделать качественные ворота обязательной частью конвейера.
