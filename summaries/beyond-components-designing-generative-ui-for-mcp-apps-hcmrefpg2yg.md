## 1. О чём доклад
Доклад об эволюции пользовательских интерфейсов в эпоху AI – от статичных компонентов к генеративному UI – и о ключевой роли протокола MCP как механизма доставки и изоляции такого UI. Автор (Ruben Casas, Postman) рассматривает три уровня генерации UI и размышляет о будущем коллаборации человека и агента.

## 2. Главные идеи
- Мы прошли путь от копирования кода из ChatGPT до моделей (GPT‑5.2, Opus 4.5), способных генерировать фронтенд‑код, порой превосходящий человеческий.
- *«Взаимодействие с этим новым компьютером похоже на разговор с терминалом… а GUI ещё не изобретён»*. Три стадии UI: статический (агент лишь наполняет готовые компоненты данными), декларативный (агент создаёт JSON/YAML‑дескриптор, который отображается на дизайн‑систему) и генеративный (модель сама пишет HTML/CSS/JS на лету).
- Генеративный UI требует песочницы – без sandbox запускать LLM‑код на клиенте небезопасно. MCP apps предоставляют двойной iFrame и встроенную изоляцию, что делает их идеальным транспортом для generative UI; даже Anthropic выбрала MCP для своего визуализатора.

## 3. Практические выводы
- Декларативный подход (статические компоненты + генерируемый дескриптор) сегодня даёт наилучший баланс между гибкостью, консистентностью, скоростью и экономией токенов.
- Без sandbox код, сгенерированный LLM, нельзя предъявлять пользователю напрямую.
- MCP apps пригодны не только для third‑party UI, но и для first‑party компонентов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- Модели: GPT‑5.2, Opus 4.5.
- Протоколы и SDK: AGUI‑протокол, Goose (Auto Visualizer), JSON Render от Vercel (поддержка JSON и YAML), Excalidraw MCP app.
- Компании: Postman, Anthropic (Claude visualizer, выбор MCP для first‑party UI).
- Техники: reverse sampling, двойной iFrame, отображение дескрипторов (JSON/YAML/Python) на компонентную систему.

## 5. Что полезно разработчику
- Внедрять декларативный UI с дескрипторами, сохраняя собственную дизайн‑систему.
- Использовать MCP apps как безопасную «песочницу» и транспорт для доставки любых UI‑артефактов.
- Не исключать генеративные компоненты, но обязательно добавлять изоляцию при выполнении кода, полученного от модели.

## 6. Что можно применить в агентных продуктах
- Разворачивать MCP apps для изолированного и аутентифицированного выполнения сгенерированного UI.
- Реализовывать коллаборативный опыт «человек + агент» по образцу Excalidraw MCP app – общий холст, где стороны могут менять артефакт одновременно.
- Использовать генерацию на лету для максимально персонализированных интерфейсов под конкретный запрос пользователя.
