## 1. О чём доклад
Сэм Морроу, лид разработки MCP-сервера GitHub, рассказывает о проблемах и решениях при создании и масштабировании сервера: борьба с раздуванием контекста, снижение отказов инструментов, безопасность токенов, stateless‑архитектура и планы на будущее.

## 2. Главные идеи
- Слишком много инструментов ухудшают работу агентов: переполняется контекстное окно, агенты путаются. Решение — сократить число инструментов по умолчанию, сгруппировать CRUD‑операции и дать пользователям возможность расширять набор.
- Оптимизация выдачи: сокращение контекста при старте на 49 %, уменьшение выходных токенов (пример — list pull requests потерял >75 % токенов).
- Безопасность: массовое использование долгоживущих plain‑text токенов, хранящихся там, где агент может их прочитать. Remote‑сервер поддерживает OAuth 2.1 и PKCE; динамическая регистрация клиентов отвергнута из‑за неконтролируемого роста и проблем с rate‑лимитами.
- Сервер полностью stateless: на каждый запрос создаётся новый экземпляр MCP‑сервера со сконфигурированными инструментами. Масштабируется до ≈7 млн tool calls в неделю.
- Insiders‑режим позволяет быстро выкатывать экспериментальные фичи (например, MCP‑apps с human‑in‑the‑loop).

## 3. Практические выводы
- Набор инструментов по умолчанию должен быть компактным и расширяемым. Группировка CRUD‑операций и фильтрация по скоупам токена снижают шум без ручной настройки.
- Описания инструментов нужно тестировать «друг против друга»: идеальное описание, вызывающее инструмент слишком часто, так же плохо, как и обратное.
- Step‑up OAuth и фильтрация инструментов по правам токена уменьшают число отказов и не требуют от пользователя правки JSON.
- Клиентам стоит использовать системные связки ключей и OAuth‑поток вместо plain‑text токенов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- GitHub MCP server (local/remote), LangChain (исследование про много инструментов), Invariant Labs (prompt injection), MCP specification, OAuth 2.1, PKCE
- Redis (сессии), VS Code (интеграция со step‑up OAuth)
- Claude Code (tool search tool API), OpenAI (похожий API), Cloudflare (code mode), Pi (harness)
- Динамическая регистрация клиентов (dynamic client registration) — сознательно отвергнута

## 5. Что полезно разработчику
- Как снижать контекстное давление: убирать лишние поля из ответов, группировать инструменты, пересоздавать экземпляр сервера под каждый запрос (stateless‑подход).
- Как повысить надёжность: кодировать «намерение агента» в серверную логику, делать несколько API‑вызовов на стороне сервера, чтобы уменьшить round‑trips.
- Идеи из Q&A: оборачивать read‑only‑инструменты в CLI с нормальной справкой — агенты работают на удивление хорошо.

## 6. Что можно применить в агентных продуктах
- Human‑in‑the‑loop через MCP‑apps: агент генерирует черновик issue, а человек редактирует перед отправкой — полезно для открытых проектов, где боятся «бот‑контента».
- Динамическая фильтрация инструментов по скоупам токена и поддержка step‑up OAuth снижают необходимость ручной конфигурации.
- Insiders‑канал для быстрой обкатки экспериментальных возможностей без нарушения стабильности.
