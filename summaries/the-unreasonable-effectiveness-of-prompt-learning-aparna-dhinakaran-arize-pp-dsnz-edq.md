## 1. О чём доклад

Доклад о практической методике **system prompt learning** — итеративном улучшении системных промптов кодинг-агентов на основе обратной связи на естественном языке. Вместо подхода reinforcement learning (RL), который требует больших объёмов данных и команд data science, автор предлагает собирать объяснения ошибок через LLM-as-a-judge и автоматически дополнять правила в промпте. Методика проверена на Claude Code и Cline (в докладе — Klein) с бенчмарками SWE-bench.

## 2. Главные идеи

- Итерация системного промпта с использованием **текстовых объяснений ошибок** может давать прирост качества без файнтюнинга модели.
- Процесс аналогичен тому, как человек выписывает свои ошибки и руководствуется ими в следующих попытках (отсылка к фильму «Помни» от Karpathy).
- Ключевую роль играет качество **eval-промпта** LLM-судьи: он должен не просто сказать «пройден/провален», а детально объяснить, *почему* агент ошибся.
- Для Claude Code удалось поднять долю решённых GitHub issues на SWE-bench lite на **5%**, для Cline — на **15%** всего на 150 обучающих примерах.

## 3. Практические выводы

- Prompt learning — быстрый и малозатратный способ улучшить агентов по сравнению с RL.
- Успех критически зависит от тщательно прописанных eval-промптов: качество объяснений прямо влияет на полезность новых правил.
- Методика хорошо ложится на реальные проекты, где уже используются файлы типа `CLAUDE.md` или `.clinerules`.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Claude Code**, **Cline (Klein)**
- **SWE-bench** (и SWE-bench lite), **BBH** и другие датасеты

- **DSPy** (промпт-оптимизатор **GEA**)
- **Arize** (компания спикера)
- Подходы: system prompt learning, meta-prompting, автоматическая генерация правил

## 5. Что полезно разработчику

- Можно повышать точность кодинг-агентов без привлечения data science команд, не меняя модель.
- Стоит инвестировать время в написание развёрнутых eval-промптов — именно они дают «learning signal».
- Итерация происходит быстро, а прирост достижим даже на малом количестве примеров.

## 6. Что можно применить в агентных продуктах

- Внедрить цикл «запуск агента → unit-тесты → LLM-судья с объяснениями → мета-промпт → обновлённые правила» для автоматического улучшения системных инструкций.
- Использовать динамически пополняемые правила в `.md` или `.rules` файлах как долговременную память агента.
