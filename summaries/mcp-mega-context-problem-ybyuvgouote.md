## 1. О чём доклад  
Доклад посвящён проблеме «MCP = Mega Context Problem»: как дать агенту доступ ко всей поверхности API (у Cloudflare ~2600 эндпоинтов), не взрывая контекстное окно моделей. Автор, Мэтт Кэри из Cloudflare, делится эволюцией их подхода — от раздутых списков инструментов к прогрессивному открытию возможностей, вплоть до создания песочницы, где модель сама генерирует и исполняет код.

## 2. Главные идеи  
- Наивное превращение каждого ендпоинта в инструмент MCP приводит к миллионам токенов — так работать нельзя.  
- Вместо загрузки всех инструментов в контекст нужно прогрессивное открытие (progressive discovery).  
- Три способа обойти проблему: CLI в песочнице, поисковый инструмент и *Code Mode* — позволить LLM писать TypeScript‑код, опираясь на автосгенерированные типы API.  
- *Code is actually a very compact plan* – один инструмент `code` даёт модели намного больше свободы, чем множество статичных вызовов.  
- Запуск ненадёжного кода пугает, но появляются безопасные «песочницы»: Workers на V8, Deno, Pydantic Monty; они изолируют окружение и дают программируемые ограничения.  
- MCP как протокол остаётся живым; важен не формат, а то, как мы подаём возможности модели.

## 3. Практические выводы  
- Если у вас большое API – используйте подход с автосгенерированными типами (из OpenAPI) и запуском кода в изолированной среде.  
- Не пытайтесь вручную нарезать API на множество маленьких MCP‑серверов – это даёт неполное покрытие.  
- Готовьте свои API к тому, что агенты будут «стучаться» в них в цикле – нужен качественный rate limiting.  
- Безопасное исполнение сгенерированного кода должно стать инфраструктурным примитивом.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- **Cloudflare Workers / WorkerD** – изоляция на базе V8, позволяющая безопасно исполнять строки кода.  
- **Cloudflare MCP server** – демо с доступом ко всем endpoint’ам только на чтение.  
- **Cloud Code** – пример поискового инструмента для подгрузки нужных операций.  
- **Deno** – альтернативная среда для исполнения ненадёжного кода.  
- **Pydantic Monty** – новый интерпретатор Python для ненадёжного кода.  
- **OpenAPI spec** – источник типов для генерации SDK.  
- **MCP, Wrangler, Next.js** – упомянуты как протокол, CLI и фреймворк с будущей нативной поддержкой MCP.

## 5. Что полезно разработчику  
- Как подключить всё API к агенту за ~1000 токенов: генерируем типы, даём модели возможность писать код, исполняем в изоляте.  
- Безопасный запуск кода теперь не обязательно требует контейнеров или ревью – подойдут лёгкие изоляты типа Workers.  
- MCP‑сервер может превратиться в middleware с флагом `mcp: true`, встраиваемый прямо в фреймворк (Next.js).  
- Чтобы клиенты не боялись запускать код, нужны программируемые ограничения: нет доступа к файловой системе, нет секретов, опциональный доступ в сеть только к разрешённым доменам.

## 6. Что можно применить в агентных продуктах  
- **Programmatic tool calling**: замена множества отдельных инструментов одним инструментом «code».  
- **Сохранение мини‑скриптов**: пользователь может сохранить сгенерированный скрипт и переиспользовать его, например, для кронов.  
- **Stateless agent loops**: при росте числа агентов на пользователя нужно уметь «засыпать» состояние в облаке.  
- Архитектура, при которой клиент сам управляет песочницей (WorkerD, Deno), делает интеграцию с любым API быстрой и универсальной.

## 7. Стоит ли смотреть полностью: yes / maybe / no  
**yes** – живое демо, наглядное сравнение подходов, чёткая концепция безопасного запуска кода как будущего агентов.
