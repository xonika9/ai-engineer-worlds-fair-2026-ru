1. **О чём доклад**  
Энджи Джонс (более двух лет руководила AI-трансформацией инженерной организации Block из 3500 человек) делится опытом построения автономной инженерной организации с помощью AI-агентов: от разочарования («90% используют AI, но доставка фич не ускорилась») до последовательного вывода на новые уровни зрелости, этапы делегирования, параллельной работы множества агентов и, наконец, создания оркестратора Builder Bot. Финал — горький вопрос: не привели ли эти успехи к сокращениям.

2. **Главные идеи**  
- Три фазы AI-внедрения: экспериментирование, принятие, и только затем измеримое влияние. У Block было принятие, но не было влияния.  
- Модель зрелости инженера: от нулевой (без AI) до пятой стадии, когда агент выдаёт готовый к поставке результат без помощи человека.  
- Правило 90-9-1: 1% создают, 9% взаимодействуют, 90% потребляют. Ставка делается на «1%» — AI-чемпионов из ключевых команд, а не на массовое обучение.  
- Подготовка репозиториев к AI: контекстные и rules-файлы, slash-команды, навыки агентов, AI-ревьюер кода, атрибуция AI в PR.  
- Делегирование задач агентам из привычных источников требований: Slack, Jira/Linear, GitHub issues. Пример: диагностика бага и PR прямо в Slack за 5 минут.  
- Мультиагентный параллелизм — переход к изолированным облачным рабочим пространствам, а затем к собственному оркестратору Builder Bot, который опирается на карту всей кодовой базы (~25 000 репозиториев).  
- Этический тупик: гордость за автономность организации обернулась вопросом — не ускорили ли мы собственные сокращения?

3. **Практические выводы**  
- Внедрять AI-дружественность не сверху, а через репозитории — это автоматически повышает качество работы агентов для всей команды.  
- Запустить программу AI-чемпионов, отбирая инженеров, готовых уделять не менее 30% времени AI-включению и не пасующих перед недетерминированностью.  
- Интегрировать делегирование в существующие рабочие потоки (назначать тикеты из трекера задач или @упоминание агента в Slack).  
- Обязательно включить AI-код-ревью с auto-fix loop, чтобы PR от ботов попадали к людям уже в приличном состоянии.  
- Для параллельной работы агентов обеспечить облачные workspace’ы; позднее построить оркестратор и мировую модель кодовой базы.

4. **Упомянутые инструменты, фреймворки, компании, продукты и подходы**  
- Компании: Block (Square, Cash App, Afterpay, Tidal).  
- AI-агенты/инструменты: Goose (внутренний кодинг-агент), Claude Code, Anthropic, MCP, Codex (AI-ревьюер), Builder Bot (собственный оркестратор).  
- Платформы для делегирования: Slack, Jira, Linear, GitHub issues.  
- Файлы репозитория: agents.md, claude.md, rules files, slash-команды, agent skills.  
- Инфраструктура: облачные workspace’ы для изолированного выполнения агентов; «мировая модель» кодовой базы на основе анализа 25 000 репозиториев.

5. **Что полезно разработчику: AI-агенты, браузерные и офисные агенты, harness, sandboxes, tool execution, developer tools, evals, observability, агентном продукте**  
- *AI-агенты* — модель зрелости и правило 1% для внедрения.  
- *Sandboxes* — изолированные облачные workspace’ы для параллельного запуска агентов.  
- *Tool execution* — интеграция агентов через MCP, slash-команды и навыки агентов.  
- *Developer tools* — подготовка репозиториев (контекстные файлы, правила), AI-код-ревью с автофиксом.  
- *Evals/observability* — метрики: AI-authored code +69%, экономия времени +37%, количество автоматических PR ×21.  
- *Harness* — программа AI-чемпионов как способ селективного разгона агентной культуры.  
- *агентном продукте* напрямую не упоминается, но принципы делегирования и интеграции в рабочие инструменты (Slack, трекеры) универсальны.

6. **Что можно применить в агентных продуктах**  
- Внедрить AI-дружественные компоненты в репозитории клиентов: agents.md, rules files, готовые slash-команды.  
- Запустить программу чемпионов/ранних последователей, фокусируясь на самых критичных репозиториях.  
- Реализовать интеграцию делегирования задач агенту из привычных источников (чаты, тикет-системы).  
- Обеспечить изолированные песочницы (sandboxes) для многозадачного выполнения агентов.  
- Построить метрики наблюдаемости: доля AI-кода, время цикла, количество PR, инициированных агентами.

7. **Возможная идея для поста**  
«Правило 1% в AI-внедрении: как 50 инженеров-чемпионов изменили всю организацию из 3500 человек — и почему массовое обучение не работает». Или более провокационно: «Мы построили автономную инженерию — и нас сократили. Кто виноват?»

8. **Стоит ли смотреть полностью:** yes  
Доклад содержит практическую дорожную карту трансформации, модель зрелости, реальные цифры и честный финальный рефлексивный удар, который редко встречается в технических выступлениях.
