## 1. О чём доклад
Доклад объясняет, как построить полноценную наблюдаемость (observability) для агентов на всех стадиях их жизненного цикла. На примере платформы Microsoft Foundry показано, как с помощью трассировки, оценок качества и безопасности, а также автоматизированного цикла «оценка–оптимизация» сократить разрыв между требованиями и реальным поведением агента в продакшене.

## 2. Главные идеи
- Аналогия *«Mind the gap»*: разрыв между платформой (ожиданиями) и агентом (реальностью) необходимо постоянно измерять, оценивать и устранять.
- Три столпа надёжности: evaluate (оценки), monitor (непрерывный мониторинг), optimize (оптимизация на основе данных).
- Observability должна быть встроена с первых шагов: трассировка через OpenTelemetry и привязанные к трассам оценки (trace-linked evals) позволяют быстро диагностировать проблемы.
- Универсальный подход: Foundry может управлять трассировкой и оценками агентов, созданных в любом фреймворке, если они передают телеметрию по стандарту OTel.
- Автоматизация цикла «eval→optimize» с помощью coding-агента (*Observe skill*) и человеческого контроля ускоряет доработку агентов.

## 3. Практические выводы
- Быстрый старт возможен через портал Foundry: создать проект, подключить модель и инструменты, включить встроенные оценки прямо в интерфейсе.
- Для программируемой разработки использовать SDK и ноутбуки из репозитория: пошагово создать агента с инструментами, workflow-агента, настроить трассировку и запустить оценки качества, безопасности и агентные (intent resolution, tool calling, task adherence).
- Локальная трассировка помогает отлаживать агента, а отправка телеметрии в Azure Monitor – мониторить его в облаке в контексте всей инфраструктуры.
- Red teaming (в том числе agentic-атаки) проверяет устойчивость к вредоносным промптам.
- Использовать навык *Observe* (в GitHub Copilot Chat) для автоматической генерации eval dataset, batch-оценок, оптимизации инструкций и выбора лучшей версии агента.
- Для анализа логов и трасс применять встроенных AI-агентов в портале Foundry и Azure Monitor.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
Azure AI Foundry, Azure Monitor, Application Insights, OpenTelemetry, GitHub Copilot (Codespaces, Copilot Chat), VS Code AI Toolkit, Jupyter notebooks, Azure CLI, GitHub Actions (подразумевается CI/CD), PyRIT (открытый репозитарий для red teaming), Microsoft Foundry MCP server, навыки Foundry (Observe skill), workflow agents, LangGraph, LangChain, GPT‑4, GPT‑4o mini, Claude (как модель coding-агента), Azure AI Search.

## 5. Что полезно разработчику
- Готовый репозиторий с пошаговыми лабораторными работами: создание агентов с нуля, добавление инструментов, построение workflow, настройка трассировки и проведение всех видов оценок.
- Возможность начать в портале, а затем перейти к SDK и автоматизировать всё через навык *Observe*, который генерирует данные для оценок, запускает batch evals, оптимизирует промпты и показывает сравнение версий.
- Единая трассировка через OTel позволяет подключать агенты из любых фреймворков и видеть полную картину в Foundry.
- Встроенные evaluators для качества, безопасности и agentic-метрик; можно добавлять собственные.
- Анализ логов и трасс естественным языком с помощью AI-агента в Azure Monitor.

## 6. Что можно применить в агентных продуктах
- Полный цикл observability: от трассировки единичных вызовов до централизованного мониторинга флота агентов.
- Trace-linked evals для быстрого поиска регрессий при смене модели или промпта.
- Workflow-агенты для декомпозиции задач и изоляции проблем.
- Red teaming для agentic-уязвимостей (запрещённые действия, утечка данных).
- Автоматическая оптимизация промптов с human-in-the-loop и откатом к лучшей версии.
- Ускорение eval-optimize цикла на порядок с помощью навыка *Observe*.
