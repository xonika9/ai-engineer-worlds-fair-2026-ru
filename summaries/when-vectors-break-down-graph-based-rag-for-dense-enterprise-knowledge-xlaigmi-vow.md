## 1. О чём доклад  
Практический кейс Writer — платформы для агентов в регулируемых отраслях — о том, как компания прошла путь от векторного поиска к гибридному графовому RAG и снизила галлюцинации при работе с плотными специализированными данными.

## 2. Главные идеи  
- Векторный поиск ломается на концентрации похожих терминов и наивном чанкинге, теряя связи.  
- Знание графовой структуры сохраняет отношения и даёт лучший контекст для LLM.  
- Fusion‑in‑Decoder (FiD) + граф знаний позволяют агрегировать пассажи без квадратичного роста затрат и резко снижают галлюцинации.  
- Мета‑уроки: решать проблемы клиентов, а не гнаться за хайпом; опираться на экспертизу команды; проверять предположения исследованиями.

## 3. Практические выводы  
- При сверхплотных данных использовать гибридный подход: графовое индексирование + текстовый поиск.  
- Графовую структуру можно хранить в JSON внутри Lucene‑поисковика, избегая узких мест специализированных графовых БД.  
- Для масштабируемого text‑to‑graph стоит обучить небольшую специализированную модель, работающую на CPU/небольших GPU.  
- Комбинация графа и FiD даёт лучшую точность и скорость, чем чистые векторные системы (проверено на Amazon Robust QA).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Writer, модели Palmyra (X5, Med, Finn), платформа для агентов.  
- Графовые БД, Cypher, Lucene‑based search engine.  
- Fusion‑in‑Decoder (FiD), Knowledge Graph + FiD, библиотека FiD от Facebook.  
- Amazon Robust QA (бенчмарк).  
- Векторный поиск, графовый RAG, гибридный поиск.

## 5. Что полезно разработчику  
- Пример поэтапного встраивания графового RAG в продукт без монолитных графовых БД.  
- Конкретный рецепт: спецмодель для картографирования в граф → JSON в Lucene → FiD для агрегации.  
- Возможность раскрыть цепочку рассуждений (сниппеты, подзапросы, источники) через API.  
- Справляется с многошаговыми вопросами и сложными форматами, где векторы дают сбой.

## 6. Что можно применить в агентных продуктах  
- Графовую основу для достоверного мульти‑хоп рассуждения без ручного связывания документов.  
- Прозрачность «мыслительного процесса» для аудита и доверия в регулируемых сценариях (финансы, медицина).  
- Архитектуру, снижающую галлюцинации и задержки, что критично для агентов, принимающих решения.

## 7. Стоит ли смотреть полностью: yes  
Да, содержит реальный инженерный опыт, сравнимые бенчмарки и практичные мета‑уроки; подходит и для начинающих, и для опытных разработчиков RAG.
