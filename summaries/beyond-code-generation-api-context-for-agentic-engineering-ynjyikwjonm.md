## 1. О чём доклад

Доклад посвящён контекстной инженерии в агентных системах — тому, как выбирать и комбинировать поисковые инструменты, чтобы агент получал нужный контекст из разных источников (файлы, базы данных, веб, память) и выдавал точные ответы. Спикер на примерах Elasticsearch и файловой системы показывает эволюцию от фиксированного RAG-пайплайна к гибкому набору поисковых тулов.

## 2. Главные идеи

- *«Context engineering is about 80% agentic search»* — ключевая роль поиска в формировании контекстного окна.
- Один универсальный поисковый инструмент не покрывает все сценарии; необходимо курировать сбалансированный набор специализированных и общих тулов.
- Агенты часто ошибаются: не вызывают инструмент, выбирают не тот или генерируют некорректные параметры. Важны продуманные описания тулов, error handling и agent skills.
- Сочетание инструментов с «низким порогом» (простые, быстрые, предсказуемые) и «высоким потолком» (гибкие, например shell tool или выполнение произвольных запросов) даёт лучшие результаты по точности и покрытию.

## 3. Практические выводы

- Если поведение агента неизвестно, начинайте с общего инструмента (например shell tool), логируйте вызовы и затем добавляйте специализированные.
- В описаниях тулов указывайте цель, триггеры, ограничения и связи с другими инструментами. При ошибках помогайте системным промптом.
- Для сложных параметров (SQL-подобные запросы) подключайте agent skills — дополнительную документацию, загружаемую по требованию.
- Обязательно встраивайте обработку ошибок в тулы и возвращайте сообщения агенту, чтобы он мог сам скорректировать запрос.
- Мощная модель снижает число ошибок, но не избавляет от необходимости грамотно спроектированного набора тулов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- Elasticsearch, ESQL (Elasticsearch query language), встраивание new genome v5, Gina CLI / Gina Grap.
- LangChain (tool decorator, встроенный shell tool, skill middleware).
- Shell tool (bash tool / exec tool), семантический поиск, гибридный поиск (grep + семантика).
- GPT‑4o nano, GPT‑4o mini.
- Agent skills с прогрессивным раскрытием (progressive disclosure), low floor / high ceiling принцип.
- Упомянуты llama index (sam tools), lighton (coal grap) как альтернативы семантического grep.

## 5. Что полезно разработчику

- Подход к проектированию тулов: начинать с простого, логировать, итеративно усложнять и добавлять специализированные инструменты.
- Приёмы улучшения надёжности: триггеры в описаниях («всегда загружай skill перед использованием»), системные промпты, возврат ошибок как ответа тула.
- Использование shell tool с песочницей и CLI‑инструментами (grep, Gina Grap) для быстрого прототипирования поиска по локальным файлам.
- Понимание, что даже слабые модели могут хорошо управлять shell‑командами, а для генерации сложных запросов может потребоваться более мощная LLM.

## 6. Что можно применить в агентных продуктах

- Комбинирование нескольких поисковых тулов (например, файловый поиск + выполнение запросов к БД) повышает точность — агент может верифицировать результаты.
- Agent skills как способ дать агенту глубокие знания о конкретном источнике данных без раздувания системного промпта.
- Паттерн «низкий пол / высокий потолок» для баланса между быстрыми стандартными ответами и способностью решать неожиданные сложные запросы.
- Семантический поиск через CLI‑обёртки (типа Gina Grap) как легковесная альтернатива полноценным векторным базам в файловых сценариях.
