## 1. О чём доклад
Дотта (Paperclip) о том, что «готово» для агентов — не чекбокс, а пучок утверждений, и как балансировать живость и верификацию.

## 2. Главные идеи
- *«Programming is solved, and agents can now produce more code and documentation faster than any human can ever verify.»* Риск: агенты создают больше работы, чем мы можем проверить.
- «Готово» — объект: артефакт, объём, рубрика, доказательства, верификатор, утверждающий, следующий шаг.
- Уровни: заявление автора → рецензия → стандарт → утверждение → реальная проверка.
- Конфликт «живость ↔ верификация»: без контроля — AI slop, сплошной контроль — застой.
- Три инварианта: продуктивная работа не останавливается без настоящих блокеров, бесконечные циклы ограничены.

## 3. Практические выводы
- Детально описывайте «готово»: артефакт, scope, критерии, доказательства, владелец следующего шага.
- Разделяйте автора и проверяющего (Claude → Codex).
- Дайте агентам инструменты: браузер, скриншоты, кастомные хуки — пусть доказывают завершённость.
- Выстраивайте цепочку передачи: агент знает, кому передать работу.
- *«Stop treating done as a Boolean and treat it more like an object.»* Галочку замените структурой.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
Paperclip, watchdogs (harness-agnostic), поддержка Py, OpenGL, Hermes, Claude Code, Codex, browser harness, блокеры, диаграмма состояний, идемпотентные checkout’ы, аудиторский след.

## 5. Что полезно разработчику
AI-агенты, браузерные агенты, harness, tool execution, developer tools затронуты напрямую; sandboxes не упоминались; evals/observability — косвенно через доказательства и цепочку; агентный продукт не упоминается, но уроки применимы.

## 6. Что можно применить в агентных продуктах
Структурированное «готово» (артефакт + критерии + доказательства + следующий ответственный). Разные агенты для генерации и верификации. Watchdog для предотвращения зависаний.

## 7. Возможная идея для поста
«Done — не галочка: как агентам подтверждать свою работу без AI slop» — разбор пяти компонентов с примерами для агентного продукта.
