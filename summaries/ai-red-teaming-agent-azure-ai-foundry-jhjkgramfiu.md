## 1. О чём доклад  
Презентация AI Red Teaming Agent в Azure AI Foundry — инструмента, который автоматически атакует AI‑приложения и модели, помогает выявлять уязвимости и встраивать защитные ограждения.

## 2. Главные идеи  
- Red teaming — обязательная часть инженерии безопасных AI‑систем (как тестирование мостов).  
- Агент построен на Python‑пакете pyrit от Microsoft AI Red Team, обёрнут в SDK и веб‑дашборд Foundry.  
- Генерирует вредоносные промпты с разными стратегиями (шифры, переворот строк, Base64, комбинации), отправляет целевому приложению или модели и оценивает ответы.  
- Вписывается в цикл: картирование рисков → расстановка guardrails → red teaming и другие проверки → итеративное улучшение.

## 3. Практические выводы  
- Без ограждений модели уязвимы: GPT‑4.1 без фильтров пропустил 25% атак в категории «насилие».  
- Контент‑фильтры и prompt shields резко снижают успешность атак (GPT‑4o с guardrails — 0% в примере).  
- Скан можно запускать на живое приложение (в т.ч. RAG) или напрямую на модель Azure OpenAI.  
- Результаты фильтруются, сохраняются и дают точечные указания для усиления защиты.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Azure AI Foundry, Microsoft AI Red Team, pyrit (pyate)  
- Semantic Kernel, Ollama, Azure OpenAI, GPT‑4o, GPT‑4.1, GPT‑5.3  
- Content filters, prompt shields, quality/safety evaluators, агентные evaluators  
- Стратегии атак: Flip, Base64, Caesar encoding, MARS‑конвертеры, композиция стратегий  
- Категории рисков: violence, hate and fairness и др.

## 5. Что полезно разработчику  
- Минимум кода для сканирования: указать проект, цель, категории, число запросов и стратегии.  
- Автоматический перебор атак без ручного составления промптов.  
- Интеграция в CI/CD, готовая ссылка на отчёт в дашборде.

## 6. Что можно применить в агентных продуктах  
- Сканирование агентов на Semantic Kernel: red team плагин встраивается как инструмент (tool call).  
- Проверка следования инструкциям и устойчивости к обходным атакам (пример с RAG‑агентом).  
- Ко‑пилотный режим: агент сам предлагает стратегии и отправляет атаки на целевого агента.
