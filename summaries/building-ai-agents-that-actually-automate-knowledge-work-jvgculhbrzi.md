## 1. О чём доклад
Доклад посвящён автоматизации умственного труда (knowledge work) с помощью AI-агентов, способных осмысленно работать с неструктурированными данными — PDF, PowerPoint, Word, Excel. Рассматриваются стек инструментов для документов, архитектурные паттерны assistive и automation-агентов и реальные кейсы внедрения.

## 2. Главные идеи
- *90% enterprise data lives within the form of documents whether it is PDFs, powerpoints, word and Excel*
- *RAG is just retrieval and then oneshot synthesis… a lot of what agents can do over your documents includes retrieval but also other operations like filebased search manipulation and more*
- Нужен «document toolbox», выходящий далеко за рамки наивного RAG: парсинг, индексация, семантический поиск, поиск по метаданным, манипуляции и структурированные запросы.
- Два типа агентов: assistive (чат-интерфейс, human-in-the-loop, нежёсткий реактивный цикл) и automation (фоновое выполнение, ограниченный control flow, пакетная обработка, human review в конце).
- Сложные документы (нерегулярные таблицы, Excel) требуют гибрида LLM/LVM с классическими парсерами и агентной валидацией.
- Automation-агенты могут играть роль ETL-слоя, структурируя сырые данные для последующего использования ассистивными агентами.

## 3. Практические выводы
- Без качественной предобработки (коннекторы, извлечение метаданных, точный парсинг) даже сильные LLM провалятся на сложных корпоративных документах.
- Для Excel-данных специализированный агент с RL‑обучением семантической карты и набором инструментов даёт точность ~95% против 70–75% у LLM с интерпретатором кода и превосходит человеческий бейзлайн в 90%.
- Архитектуру агента стоит выбирать под UX: для диалоговых ассистентов подходит неограниченный react-loop, для автоматизации — более контролируемый рабочий процесс с явной логикой и этапом human review.
- Автоматизация, замыкающая ручной просмотр только на финальное согласование, способна сокращать недели ручной работы до автоматического конвейра.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- LlamaIndex: managed-платформа, document‑toolbox, сервис парсинга, Excel‑агент.
- Концепция Document MCP server как обобщённого интерфейса инструментов для документов.
- Модели: Claude (Sonnet 3.5, 4.0), Gemini 2.5 Pro, OpenAI 4.1, LVM.
- Техники: Reinforcement Learning для построения семантической карты таблиц, code‑interpreter baseline, react-лупы, function‑calling.
- Клиентские кейсы: Carl (финансовый due diligence), SEMX (enterprise‑search), глобальная электронная компания (авто‑извлечение тех. даташитов).

## 5. Что полезно разработчику
- Понимание, что для рабочих агентов недостаточно простого RAG; требуется toolbox с операциями поиска, манипуляций и структурированных запросов.
- Рецепт точного парсинга: комбинация LLM/LVM с традиционными эвристиками и агентной валидацией.
- Подход к ненормализованным Excel‑файлам: RL‑обучение семантической карте → генерация специализированных инструментов → 95% точности.
- Разделение агентов на automation‑бэкенд и assistant‑фронтенд как устойчивый архитектурный паттерн.

## 6. Что можно применить в агентных продуктах
- Document MCP сервер как единый слой инструментов для всей неструктурированной документации компании.
- Конвейер «automation‑агент структурирует данные, assistant‑агент отвечает на вопросы»; особенно эффективен в фин.анализе, due diligence и обработке технических спецификаций.
- Пакетная обработка с финальным человеческим ревью вместо полного ручного контроля на каждом шаге.
- Специализированные агенты для нормализации Excel‑отчётов, извлечения ключевых полей и проверки по бизнес‑правилам.
