## 1. О чём доклад
Samuel Colvin, создатель Pydantic, показывает, что протокол MCP (Model Context Protocol) — это не только связка для coding-агентов, но и универсальный фундамент для коммуникации автономных агентов. Фокус — на вызовах инструментов (tool calling), а также на «семплинге» — возможности MCP-сервера запрашивать LLM через клиента, упрощая построение многоагентных систем.

## 2. Главные идеи
- Три примитива MCP: prompts, resources, tools. Для агентов важнее всего tools: динамическое появление/исчезновение инструментов, логирование во время выполнения, семплинг (обратные запросы к LLM) и трассировка.
- Семплинг позволяет вложенному агенту, реализованному как MCP-сервер, использовать LLM родительского агента — без настройки API-ключей и дополнительных расходов.
- Инкапсуляция логики (например, генерации SQL) внутри инструмента разгружает контекст основного агента.
- Протокол удобно работает через подпроцессы со стандартным вводом/выводом, что избавляет от необходимости OpenAPI.

## 3. Практические выводы
- MCP заменяет самодельные протоколы взаимодействия агентов стандартизированными вызовами инструментов.
- Логирование через `context.log` и прогресс-отчёты передаются клиенту до завершения вызова — полезно для long-running операций.
- Семплинг радикально упрощает совместный доступ к LLM в распределённых системах, не требуя раздачи ключей от API каждому компоненту.
- Наблюдаемость (tracing) внутри MCP даёт полную картину цепочек вызовов без дополнительной инструментации.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Pydantic** (валидация данных), **Pydantic AI** (агентный фреймворк), **Pydantic Logfire** (наблюдаемость)
- **MCP Python SDK**, **FastMCP**
- BigQuery (публичные данные PyPI)
- Модели: *GPT-4o*
- Подходы: output validation, model retry, dynamic tool calling, sampling, logging, observability/tracing

## 5. Что полезно разработчику
- Типобезопасное логирование через контекст MCP (`Context['log']`).
- Валидация результатов с автоматическими ретраями при ошибках — через `ModelRetry` в Pydantic AI.
- Пример упаковки SQL-логики внутрь MCP‑инструмента, снижающей потребление токенов основным агентом.
- Демонстрация клиент-серверного взаимодействия с проксированием LLM‑запросов (семплинг, поддержка появляется в PR прямо сейчас).
- Трассировка в Logfire показывает сквозной путь от внешнего агента до вложенного вызова LLM.

## 6. Что можно применить в агентных продуктах
- **Разделение обязанностей**: каждый инструмент — отдельный MCP‑сервер со своей логикой и доступом к LLM; основной агент только оркестрирует.
- **Интерактивные обновления**: прогресс и логи из инструментов можно отображать пользователю в реальном времени (веб‑приложения, deep research).
- **Централизованное управление LLM**: семплинг позволяет вложенным агентам разделять модель родителя, сохраняя единую точку конфигурации и контроля бюджета.
- **Глубокая наблюдаемость**: MCP‑трейсинг делает видимой всю цепочку вызовов — от пользовательского запроса до финального ответа, без доп. настроек.
