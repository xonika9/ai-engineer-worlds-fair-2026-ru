- Спикеры: Daniel Chalef, Zep
- Трек: Graphs @ AI Engineer (Context Graphs, Knowledge Graphs, GraphRAG, GNNs)

## 1. О чём доклад
Доклад о том, почему универсальная память на основе RAG (извлечения фактов в векторную БД) не работает для агентов, и как строить доменно-ориентированную память с помощью временного графа.

## 2. Главные идеи
- Не существует универсальной памяти: извлечение произвольных фактов и семантический поиск не гарантируют релевантности бизнес-контексту.
- Нерелевантные факты «загрязняют» память агента, вызывая галлюцинации и неточные ответы.
- Семантическое сходство ≠ бизнес-релевантность: векторные представления не передают причинных или отношенческих связей.
- Память нужно моделировать под предметную область: определять бизнес-объекты, их поля и связи в виде онтологии графа.
- Графовая память с фильтрацией по типам узлов позволяет извлекать строго то, что нужно предметной области.

## 3. Практические выводы
- Не стоит сбрасывать сырые факты в векторную БД и полагаться только на семантический поиск.
- Разработчик должен задавать явные бизнес-сущности (финансовые цели, долги, доходы и т.п.) и их атрибуты.
- Использовать схемы (pydantic, zod, go structs) и бизнес-правила для валидации данных.
- Регистрировать онтологию в графовой памяти, чтобы агент понимал структуру предметной области.
- При поиске применять фильтрацию по типам узлов для получения строго релевантных результатов.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Zep** (компания, SDK для Python, TypeScript, Go) — решение для доменной памяти на временных графах.
- **Graffiti** — open-source temporal graph framework от Zep для построения кастомных сущностей и связей.
- **Zep frontend** — визуализация графа знаний пользователя.
- Схемы: Pydantic (Python), Zod (TypeScript), Go structs.
- Упомянуты векторные БД и Redis (в контексте хранения фактов), а также ChatGPT и обобщённые агентные фреймворки.

## 5. Что полезно разработчику
- Возможность определять собственные типы сущностей, поля и бизнес-правила, используя привычные инструменты валидации.
- Простое SDK для работы с памятью из Python, TypeScript или Go.
- Готовые механизмы фильтрации по типам узлов и параллельного поиска (multi‑search) в графе.
- Интеграция с агентами через инструменты (tools), возвращающие снимок доменных данных.

## 6. Что можно применить в агентных продуктах
- Заменить «плоский» фактоидный RAG на графовую память с онтологией предметной области.
- Снизить количество галлюцинаций, вызванных нерелевантными или произвольными фактами.
- Повысить точность ответов в вертикальных агентах (финансовый коуч, медиа-ассистент и т.д.) за счёт жёсткой привязки памяти к домену.
- Использовать временную составляющую графа для учёта истории взаимодействий и изменений сущностей.
