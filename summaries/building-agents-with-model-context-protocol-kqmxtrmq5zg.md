## 1. О чём доклад

Доклад раскрывает философию и практическое применение Model Context Protocol (MCP) — открытого протокола от Anthropic для стандартизации взаимодействия AI-приложений и агентов с внешними источниками данных и инструментами. Mahesh Murag объясняет, как MCP решает проблему фрагментации при интеграциях, предоставляя три интерфейса: *промпты*, *инструменты* и *ресурсы*. Также освещаются первые результаты, архитектурные паттерны и дорожная карта развития.

## 2. Главные идеи

- Модели сильны лишь настолько, насколько хорош предоставляемый им контекст.
- MCP превращает N×M проблему кастомных интеграций в единый стандартный слой, подобно тому как API унифицировали веб-сервисы, а LSP — языковые инструменты в IDE.
- Протокол вводит три паттерна контроля: *инструменты* (model-controlled), *ресурсы* (application-controlled) и *промпты* (user-controlled), давая богатые интерфейсы разным частям системы.
- Главная цель — сделать MCP фундаментальным протоколом для агентов, позволяя им не просто вызывать тулы, а динамически *обнаруживать новые возможности* и саморазвиваться.

## 3. Практические выводы

- Разделяйте логику: серверы должны находиться ближе к конечным системам и владеть логикой взаимодействия (повторы, форматирование), а клиентские приложения — фокусироваться на агентном цикле.
- Для борьбы с перегрузкой контекста при сотнях инструментов внедряйте иерархические группы тулов или RAG-поиск по инструментам.
- Используйте `sampling` для случаев, когда серверу нужен интеллект без самостоятельного хостинга LLM: сервер запрашивает логический вывод у клиента, сохраняя контроль над расходами и приватностью.
- Композиция (клиент-сервер как логическая, а не физическая роль) позволяет строить многоуровневые системы специализированных агентов, где каждый узел может быть и клиентом, и сервером.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **MCP Agent** (от Last Mile AI) — open-source фреймворк для декларативной сборки агентов поверх MCP.
- **Клиенты**: Claude for Desktop, Cursor, WindSurf, Goose (Block), Zed.
- **Серверы и системы**: GitHub, Asana, Brave Search, файловая система, серверы для Slack, Shopify, Grafana, Cloudflare, Stripe; 1100+ комьюнити-серверов.
- **Протоколы**: OAuth 2.0, SSE (Server-Sent Events) для удалённых серверов, концепция `.well-known` для автообнаружения.
- **Инструменты разработки**: MCP Inspector, автогенерация серверов в IDE (Cline/Клин), пакетная система (npm, PyPI), Docker-образы.

## 5. Что полезно разработчику

- Можно с нуля написать простой MCP-сервер к своему API примерно за час, используя Claude в помощники.
- После реализации MCP-клиента вы сразу получаете доступ ко всем существующим серверам без доработок.
- В enterprise-среде MCP позволяет инфраструктурной команде выставить RAG-систему или базу данных как сервер, и все AI-команды подключатся к ней стандартно, без дублирования логики чанкинга и промптинга.

## 6. Что можно применить в агентных продуктах

- Сделать агентов саморазвивающимися: через реестр серверов агент сам ищет и устанавливает нужный сервер (например, Grafana), даже если не был запрограммирован на работу с ним изначально.
- Сочетать MCP с компьютерным управлением (Computer Use): сначала агент пытается работать поверх структурированного MCP-интерфейса (/.well-known), а при его отсутствии переходит к кликам по UI.
- Внедрять асинхронное поведение: сервер может уведомлять клиент об изменении ресурсов или запрашивать ввод у пользователя через цепочку вызовов без разрыва сессии.
