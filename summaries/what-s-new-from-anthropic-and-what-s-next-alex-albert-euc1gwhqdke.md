## 1. О чём доклад  
Алекс Альберт (Anthropic) рассказывает о новых возможностях Claude и о том, как переосмысливать продукты с нуля вокруг ИИ, а не просто добавлять «волшебную звёздочку». Представляет модель 3.5 Sonnet, функции Artifacts и Projects, улучшения API и направление будущих исследований.

## 2. Главные идеи  
- Аналогия с электрификацией: производительность выросла, только когда заводы перестроили под электричество. Так и с ИИ – вместо поверхностной интеграции нужно создавать новые продуктовые парадигмы. Мы сейчас в фазе *«magic star icon phase with respect to AI»*.  
- Claude 3.5 Sonnet – средняя по размеру модель, уже превосходящая Opus, с сильными навыками кодирования, визуального анализа и почти идеальным запоминанием в 200K-контексте.  
- Artifacts и Projects позволяют отделять контент от диалога и заземлять ответы на командных документах, делая LLM основой совместной работы.

## 3. Практические выводы  
- Модели дешевеют и умнеют за месяцы, не годы – продуктовые дорожные карты должны быть амбициозными. Спикер напоминает: *«models will become smarter cheaper and faster in orders of months not years»*.  
- Claude 3.5 Sonnet подходит для RAG, отладки кода, транскрипции таблиц и превращения скриншотов в код.  
- Tool use API даёт модели сотни инструментов и структурированный JSON-вывод.  
- Консоль разработчика упрощает создание промптов (генератор, переменные, evaluate).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- Claude 3.5 Sonnet, Claude 3 Opus, анонсированные 3.5 Haiku и 3.5 Opus.  
- Artifacts, Projects, tool use API, prompt generator, evaluate, steering API.  
- Golden Gate Claude (демо управления функциями модели).  
- Google Cloud, AWS Bedrock, Vertex AI.  
- Конкурс Build with Claude (призы – API-кредиты).

## 5. Что полезно разработчику  
- 3.5 Sonnet доступна на основных облачных платформах; цена $3/1M входных и $15/1M выходных токенов.  
- Значительный скачок в задачах по пулл-реквестам: 64% успешных решений против 38% у Opus.  
- Vision: точная транскрипция сложных таблиц и OCR.  
- Консоль: автоматическая генерация оптимизированных промптов, работа с переменными, бета-функция evaluate.  
- Tool use: до сотен инструментов и последовательный структурированный ответ.

## 6. Что можно применить в агентных продуктах  
- Tool use API – ключевой строительный блок для агентов: Claude самостоятельно выбирает и вызывает клиентские функции.  
- Steering API позволяет включать или приглушать «фичи» модели, точно настраивая поведение агента.  
- Artifacts и Projects показывают, как встраивать LLM в живой цикл совместной разработки и документирования.
