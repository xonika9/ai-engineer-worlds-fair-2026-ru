## 1. О чём доклад
Использование нескольких AI-агентов (локальных, фоновых и облачных) в VS Code как единой точке входа. Показано, как одновременно решать разные задачи в одной кодовой базе, варьируя степень контроля человека над агентом. Применимо не только к GitHub Copilot, но и к другим ИИ-сервисам.

## 2. Главные идеи
- Агенты не предназначены для всесильных «одношаговых» запросов — требуется осмысленное распределение задач и контроль расхода токенов.
- VS Code выступает центральной панелью для агентов: локальные (максимальный hands-on), фоновые (CLI с Git worktrees, 50/50) и облачные (удалённые, для автоматизации документации).
- Параллельная работа разнотипных агентов повышает продуктивность и снижает когнитивную нагрузку.
- Настройка агентов строится на общих механизмах: кастомные инструкции, prompt-файлы, навыки (skills), хуки, MCP-серверы — подход переносим между Copilot и Claude.
- *«You still see developers … who think we can do one shot prompts. They’ll create a wonderful application or solve all of their issues in one go. Absolutely not the case.»*

## 3. Практические выводы
- Разделяйте задачи по типу агента: локальный — для тестов (полный контроль), фоновый — для фронтенда (частичный контроль), облачный — для рутины вроде README.
- Для фоновых агентов применяйте Git worktrees — изолированные подпапки с отдельной веткой, чтобы не загрязнять основную кодовую базу.
- Осваивайте модальное окно настроек Copilot в VS Code: там задаются кастомные агенты, skills, MCP-серверы и хуки.
- Используйте Autopilot с осторожностью, чтобы отключить подтверждения для инструментальных вызовов при создании планов.
- Ориентируйтесь на готовые конфигурации из Awesome Copilot (aka.ms/awesomecopilot).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- GitHub Copilot, Copilot Chat, Copilot CLI, Autopilot (preview)
- VS Code, Git worktrees
- MCP (Model Context Protocol), Playwright MCP, GitHub MCP server
- Claude (Anthropic), Azure, GCP, AWS (контекст MCP)
- Awesome Copilot, custom agents, prompt files, agent skills, hooks
- Облачные агенты на базе GitHub Actions

## 5. Что полезно разработчику
- Параллельное управление тремя агентами в одном рабочем пространстве без смены инструмента.
- Гибкая настройка поведения агента под свою задачу через единый интерфейс VS Code.
- Постепенное снижение нагрузки: руками — только критические участки, остальное делегируется с желаемой глубиной контроля.
- Готовые открытые сценарии и интегрируемые MCP-серверы для тестирования и документации.

## 6. Что можно применить в агентных продуктах
- Модель «локальный / фоновый / облачный» как способ разделения ответственности и вовлечённости человека.
- Одновременный запуск нескольких специализированных агентов над разными задачами в одном проекте.
- Git worktrees как безопасная песочница для автономных агентских изменений.
- Расширение контекста агентов через MCP (например, доступ к UI-тестированию через Playwright MCP или к ресурсам облака).
