## 1. О чём доклад
Доклад посвящён трём методам файн-тюнинга, доступным через API OpenAI: SFT (Supervised Fine-Tuning), DPO (Direct Preference Optimization) и RFT (Reinforcement Fine-Tuning). Спикер на примерах и демонстрациях разбирает, чем они отличаются, когда какой применять, как готовить данные и каких результатов можно достичь — от дистилляции моделей до обучения агентов и решения научных задач.

## 2. Главные идеи
- Файн-тюнинг — это не замена промпт-инжинирингу, а специализированный инструмент для случаев, когда подсказок уже недостаточно. *«Избегайте файн-тюнинга, пока он действительно не понадобится»*.
- Поведение модели корректируется через дообучение LoRA-адаптеров: обновляется лишь малая часть весов, поэтому метод плох для запоминания больших объёмов новых знаний (для этого лучше использовать RAG).
- Три типа файн-тюнинга решают принципиально разные задачи:
  - **SFT** учится имитировать конкретные ответы (непрерывное отображение вход → выход).
  - **DPO** улавливает разницу между предпочтительным и нежелательным ответами, особенно полезен для тона, стиля и A/B-тестов.
  - **RFT** тренирует модель самостоятельно выстраивать цепочку рассуждений, получая награду от «грейдера».
- Проклятие данных: RFT крайне чувствителен к шуму и неоднозначности, требует очень чистых и верифицируемых примеров. Неудача с классификацией личной почты показала, что субъективные и плохо воспроизводимые оценки не подходят для RFT.

## 3. Практические выводы
- Всегда начинайте с промпт-инжиниринга и системы оценок (evals). Файн-тюнинг имеет смысл, когда исчерпаны возможности промптов и есть достаточно чистых данных.
- SFT отлично работает для жёстко ограниченных задач: классификации, извлечения структурированных данных, вызова функций. Можно «сжимать» промпт и ускорять модель, дистиллируя знания из большой модели в малую (например, Gpt 4 → Gpt 4o-mini).
- Для SFT часто хватает от 50–100 примеров для первых результатов и 500+ для продакшена. DPO более щадящий к количеству данных, а RFT может дать прирост уже на 40–80 высококачественных примерах.
- Грейдер для RFT критически важен: чем точнее он отражает понятие «хорошего» ответа, тем выше итоговое качество. Лучше всего работают однозначные задачи, где решение легко проверить (например, точные науки).

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Методы файн-тюнинга**: SFT, DPO, RFT, метод дообучения LoRA (Low-Rank Adaptation).
- **Модели**: Gpt 3.5, Gpt 4o, Gpt 4o-mini, o1, o3, o4-mini. Gpt 4.5 использовался для генерации синтетических шуток.
- **Инструменты**: OpenAI Agents SDK (упомянут релиз TypeScript), cookbook по файн-тюнингу для function calling и подбору грейдеров для RFT.

## 5. Что полезно разработчику
- Формат данных — JSONL с массивом messages, модель учится на последнем сообщении `assistant`. Для RFT дополнительно задаётся `reference` и структура грейдера.
- При смене системного промпта у файн-тюненной модели стоит либо переобучиться, либо использовать несколько вариантов промптов в обучающей выборке.
- Наблюдение за графиками train/validation loss в панели файн-тюнинга помогает вовремя заметить переобучение и остановить задание.

## 6. Что можно применить в агентных продуктах
- SFT позволяет дистиллировать сложное поведение агента в быструю и дешёвую модель: яркий кейс — точность вызова нужной функции из 120 вариантов на уровне Gpt 4 при минимальной задержке на Gpt 3.5.
- На данный момент RFT поддерживает только один ход (single turn) без возврата результатов функции, поэтому напрямую не применим для многошаговых агентных сценариев.
