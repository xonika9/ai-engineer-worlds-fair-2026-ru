## 1. О чём доклад

Доклад о том, как применять алгоритмы графовой аналитики (graph data science) на платформе Neo4j для повышения качества retrieval-augmented generation и reasoning в GraphRAG-приложениях. Рассматривается связка «документы – промпты – ответы – контекстные чанки», а затем используемые алгоритмы кластеризации, центральности и коллапсирования для управления данными на масштабе и улучшения ответов.

## 2. Главные идеи

*   Граф объединяет неструктурированные чанки, метаданные приложения (сессии, сообщения, контекст) и связи между ними, позволяя анализировать поведение системы и данных в одной модели.
*   Кластеризация (community detection) на основе эмбеддингов выявляет группы почти идентичных документов и проблемные кластеры (слишком короткие, чрезмерно похожие), что помогает чистить данные без потери связей.
*   Анализ центральности (PageRank, betweenness) и взвешенные связи совместной встречаемости документов в ответах дают возможность переранжировать результаты поиска для повышения разнообразия и релевантности.
*   Важно осознанно управлять усиливаемыми сигналами: *«мы должны отвечать за сигналы, которые усиливаем в наших системах»*, чтобы агентные цепочки не уходили в нежелательном направлении.

## 3. Практические выводы

*   Стройте граф документов и промптов, запускайте KNN по эмбеддингам для создания связей похожести, затем – Louvain или Label Propagation для выделения сообществ.
*   Анализируйте сообщества агрегированными метриками (средняя длина текста, среднее косинусное сходство). Высокое сходство (≈0.98) у десятков чанков означает избыточность.
*   Почти идентичные чанки схлопывайте через `apoc.nodes.collapse` – сохраняется одна нода, но остаются все исходные связи для трассировки.
*   Повышайте разнообразие выдаваемых документов при переранжировании: приоритизируйте чанки из разных сообществ, а не только ближайшие по вектору.
*   Создавайте связи `co-occurs` между документами, которые возвращались в одном ответе: это выявляет часто совместно используемые концепции.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

*   Neo4j (Graph DB, AuraDB, console.neo4j.io), Graph Data Science (GDS), Cypher, APOC.
*   Алгоритмы: KNN (по косинусному сходству), Louvain, Label Propagation, PageRank, Betweenness Centrality.
*   GraphRAG-пакет (graphrag.com), Knowledge Graph Builder (KG Builder) для автоматического построения графов.
*   Co-occurrence-граф на основе контекстных документов.

## 5. Что полезно разработчику

*   Метод коллапса почти идентичных чанков с сохранением всех источников – удобная чистка данных без страха потерять информацию.
*   Готовые шаблоны запросов для быстрого построения графа схожести и выполнения кластеризации прямо в Jupyter-ноутбуках.
*   Простой способ выявлять «мусорные» документы (пустые/однословные чанки) через статистики сообщества.
*   Стратегия начальной настройки: запускать автоматический Knowledge Graph Builder, смотреть статистики, затем уточнять схему.

## 6. Что можно применить в агентных продуктах

*   Application graph (граф сессий, промптов и ответов) позволяет видеть, по каким сообществам данных движется пользователь, и предсказывать следующие шаги диалога.
*   Аналитика центральности и co-occurrence помогает агентам выбирать не самые очевидные, но семантически связанные документы, снижая риск зацикливания на одной теме.
*   Контроль усиливаемого сигнала: мониторинг цепочек «агент-агент» через графовую аналитику не даёт системе уверенно повторять ошибочный или устаревший контекст.
