## 1. О чём доклад
Доклад посвящён проектированию памяти AI-агентов — как превратить stateless‑приложения в stateful, уменьшить prompt‑engineering за счёт персистентности и сделать агентов believable, capable и reliable. Richmond Alake (MongoDB) даёт высокоуровневый обзор типов памяти, управления памятью (memory management) и показывает, как документная база данных может служить единым провайдером памяти для агентных систем.

## 2. Главные идеи
- Память — фундаментальный компонент агента, без неё невозможны reflective, proactive и autonomous свойства.
- Спектр агентности (agenticity): от минимального агента с LLM‑петлёй до автономной системы с инструментами и минимальным контролем.
- Агентная память делится на множество типов: persona, toolbox, conversation, workflow, episodic, long‑term, entity memory, agent registry.
- Memory management — систематический процесс генерации, хранения, retrieval, интеграции, обновления и забывания (не удаления) памяти.
- Retrieval (RAG) — ключевой этап, где нужны не только векторный, но и другие виды поиска (текстовый, графовый, геопространственный).
- MongoDB позиционируется как «memory provider» с гибкой документной моделью и всеми retrieval‑механизмами в одной базе.
- Нейронаука — источник вдохновения: понимание работы человеческой памяти помогает строить более совершенные агентные архитектуры.

## 3. Практические выводы
- Для масштабируемой памяти агента стоит использовать документную БД, позволяющую хранить разнородные структуры (persona, workflow, инструменты и т.д.) в едином контуре.
- Вместо удаления воспоминаний внедрять механизмы забывания (recency, ассоциативные сигналы).
- Toolbox memory помогает масштабировать число инструментов: хранить JSON‑схемы в БД и извлекать релевантные перед вызовом LLM, не забивая контекстное окно.
- Workflow memory даёт агенту учиться на ошибках: сохранение неудачных траекторий позволяет избегать их в следующих исполнениях.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- MongoDB Atlas, MongoDB как memory provider
- Voyage AI (приобретение MongoDB) — embedding‑модели и реранкеры
- Open‑source библиотека «Memoriz» (Richmond Alake) — экспериментальные design‑patterns для разных типов памяти
- MEGPT, ME Zero, Zep — внешние инструменты memory management
- Подходы: persona memory, toolbox memory, conversation memory, workflow memory, episodic/long‑term memory, agent registry, memory signals (recency, association), forgetting mechanisms
- Аналогия с нейронаукой: Hubel & Wiesel, convolutional neural networks; консультации с нейробиологами для построения агентов

## 5. Что полезно разработчику
- Понимание, что память — не только контекст диалога, а целый спектр типов, каждый из которых требует своего моделирования.
- Примеры схем данных в MongoDB для persona, toolbox, conversation и workflow memory.
- Идея автоматического чанкинга и реранкинга через интегрированные модели Voyage AI (упрощает построение RAG‑пайплайнов).
- Практический совет: не удалять память, а реализовывать forgetting‑сигналы для поддержания актуальности.

## 6. Что можно применить в агентных продуктах
- Persona memory для создания believable взаимодействия и долгосрочных отношений с пользователем.
- Toolbox memory для безопасного масштабирования числа инструментов без переполнения контекстного окна.
- Workflow memory для накопления опыта и автономного улучшения цепочек действий.
- Episodic/long‑term memory для сохранения важных событий и фактов о пользователе или окружении.
