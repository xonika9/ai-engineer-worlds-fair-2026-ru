## 1. О чём доклад  
Доклад о том, почему браузер — единственный универсальный мост между AI-агентами и «обычным» интернетом, где нет API и MCP‑серверов. Автор объясняет, как безбраузерные (headless) браузеры и браузерный MCP‑сервер становятся «интеграцией последней надежды», и разбирает подходы, инструменты и практики создания надёжных веб‑агентов.

## 2. Главные идеи  
- Браузерный MCP‑сервер — горизонтальный шлюз для всей сети: один сервер покрывает множество сайтов.  
- Веб‑агенты (одна задача → много недетерминированных действий) и браузер‑инструменты (одно действие за вызов) решают разные классы задач.  
- Критически важны собственные evals, а не публичные бенчмарки, и полная наблюдаемость (запись сессий и действий).  
- Агент должен быть «хорошим гражданином» интернета, иначе его заблокируют, независимо от стелс‑технологий.

## 3. Практические выводы  
- Включайте браузерный инструмент в каждый агент, если нет готовой интеграции.  
- Для сайтов без API используйте горизонтальный браузерный MCP‑сервер вместо реверс‑инжиниринга API.  
- Стройте собственные evals для выбора модели под конкретную задачу; бенчмарки от вендоров необъективны.  
- Обеспечьте запись и просмотр действий агента — это упрощает отладку и контроль.  
- Для human‑in‑the‑loop встраивайте live‑view (iframe) с возможностью вмешательства пользователя.  
- Капчи решаются временными средствами, но будущее — за агентской аутентификацией; соблюдайте robots.txt и лимиты действий.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы  
- **Browserbase** — облачная инфраструктура headless‑браузеров, MCP‑сервер, Stagehand, live‑view.  
- **Stagehand** — фреймворк‑надстройка над Playwright, превращающий высокоуровневые шаги в детерминированные действия.  
- **MCP‑сервер для браузера** (browser MCP server) — горизонтальный сервер с примитивами «кликни», «введи текст».  
- **Web Voyager, Adept (Fuji), OpenAI Operator, proxy от Convergence** — ранние веб‑агенты.  
- **Модели**: Claude (Sonnet 4), в целом подход model‑agnostic.  
- **Brain Trust** — платформа для собственных evals.  
- **Подходы**: vision‑based агенты (скриншоты + set‑of‑marks), DOM‑based агенты (Accessibility Tree, XPath), обучение на web trajectories.

## 5. Что полезно разработчику  
- Писать агентов, которые через браузер кликают, навигируют и реагируют на модальные окна, используя примитивы MCP‑сервера.  
- Встраивать запись сессий и live‑view в своё приложение для отладки и демонстрации пользователю.  
- Использовать Stagehand, чтобы преобразовывать намерения пользователя в надёжные цепочки действий.  
- Обрабатывать капчи через инфраструктуру, но помнить про легитимность ботов.  
- Строить собственные evals для выбора наилучшей модели под конкретный веб‑флоу.

## 6. Что можно применить в агентных продуктах  
- Браузерный MCP‑сервер как «клей» для интеграций с CRM, устаревшими системами и любыми сайтами без API.  
- Live‑view и записи браузерных сессий для прозрачности перед пользователем и ревью операций.  
- Режим human‑in‑the‑loop: встроенный браузер, где человек может перехватить управление и поправить действия агента.  
- Автоматизацию рутинных операций (например, для небольших компаний) без найма разработчиков.
