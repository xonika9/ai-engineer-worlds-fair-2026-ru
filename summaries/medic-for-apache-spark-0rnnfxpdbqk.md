# Структурированное саммари доклада «Medic for Apache Spark»

## 1. О чём доклад

Драшко Проферович (Pinterest) рассказывает о создании Medic — агентного диагностического инструмента для Apache Spark. Система по запросу «почему упала задача» автоматически проводит глубокое исследование, находит корневую причину сбоя и предлагает конкретные исправления. Доклад прослеживает путь от прототипа на одном React‑агенте до мульти‑агентной архитектуры, останавливаясь на улучшениях обработки логов, метрик и тестирования.

## 2. Главные идеи

- Автоматизация поддержки Spark‑задач LLM‑агентами, чтобы снять с инженеров бесконечный поток запросов.
- Переход от одного монолитного промпта к специализированным агентам: *«prompt tuning became unsustainable»*, один промпт должен был делать всё и терял качество.
- Борьба с шумными логами: классификатор исключений, отделяющий безобидные ошибки от настоящих, чтобы не сбивать LLM.
- Замена сырых метрик изображениями (как Grafana‑дашборд) для экономии контекстного окна.
- Внедрение e2e test harness с режимами записи/воспроизведения и офлайн‑оценок для измерения качества вместо догадок.

## 3. Практические выводы

- Один промпт не масштабируется; мульти‑агентная архитектура даёт контроль и упрощает поддержку.
- Контекстное окно — ценный ресурс. Конвертируйте временные ряды в графики, а логи сжимайте через классификатор.
- Тестовый харнес со снэпшотами продакшн‑состояния и codified evals позволяет ловить регрессии и настраивать промпты уверенно.
- MCP оказался удобным способом подключения данных к LLM.
- React‑агенты оказались надёжнее детерминированных рабочий процесс: *«рабочий процессs... proved to be brittle compared to the reasoning and acting agent paradigm»*.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы

- **Инструменты/фреймворки**: MCP, React‑агенты, OpenTelemetry, LangFuse, LangGraph Deep Agent library, Grafana (упоминание), Cloud Code/Codex.
- **Продукты**: Medic (внутренний инструмент Pinterest).
- **Подходы**: e2e test harness (record/playback), exception classifier конвейер, конвертация сырых метрик в аннотированное изображение, multi‑agent arch (triage → research агенты параллельно → supervisor → healer agent с runbooks из векторной БД), классификация интента на простой ответ или глубокую диагностику.

## 5. Что полезно разработчику

- **Мульти‑агентный шаблон** с триажем, параллельным исследованием гипотез и «лекарем» напрямую применим к офисным/бразуерным агентам.
- **Test harness** со снэпшотами и eval-скриптами — отличный способ воспроизводимо тестировать поведение AI-агентов и избегать регрессов.
- **Observability** через LangFuse и OpenTelemetry даёт waterfall‑диаграмму шагов агента.
- **Подход с изображениями** (графики вместо сырых метрик) помогает вписаться в контекстное окно — полезно при аналитике в песочницах.
- **Классификатор исключений** улучшает signal‑to‑noise, как и для логов выполнения в браузере или офисных скриптах.

## 6. Что можно применить в агентных продуктах

- Внедрить **e2e test harness** с записью ответов инструментов как fixtures и офлайн evals, чтобы измерять качество агентов до релиза.
- Для браузерных/офисных агентов — **мульти‑агентную декомпозицию**: triage‑агент, исследователи гипотез, supervisor‑составитель отчёта.
- Обрабатывать большие логи через **классификатор**, а метрики — **визуализировать** перед подачей в LLM.
- Использовать **MCP** для унификации инструментов агентов.
