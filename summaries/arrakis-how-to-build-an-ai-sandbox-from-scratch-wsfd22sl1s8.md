## 1. О чём доклад
Обзор архитектуры и устройства открытой AI-песочницы Arachus: как с нуля построить безопасную среду для выполнения кода и управления компьютером агентами с помощью microVM, моментальных снимков и сетевой изоляции.

## 2. Главные идеи
- Полноценная Linux-песочница даёт агенту возможность самостоятельно отлаживать сбои (`ps`, `lsof`), возвращаться назад и перепланировать задачи.
- Для многошаговых агентных сценариев критичны снэпшоты: агент может сохранить контрольную точку, а при ошибке восстановиться без потери прогресса.
- Безопасность достигается не контейнерами (общее ядро), а microVM — каждое «виртуальное обслуживание» запускается в изолированном ядре, написанном на Rust с jail-устройств.
- Скорость загрузки песочницы должна быть менее секунды; microVM дают загрузку ~7 с (Roadmap — <1 с).
- Overlay FS защищает базовый образ: общий read-only слой плюс приватный read-write слой для каждой песочницы.

## 3. Практические выводы
- Нельзя полагаться только на ограничение capabilities/seccomp в контейнерах — уязвимость ядра всё равно даст root.
- Выбор microVM (Cloud Hypervisor) вместо QEMU оправдан малой кодовой базой, безопасностью и быстрой загрузкой.
- Снэпшот требует паузы VM, дампа гостевой памяти, ручного сохранения read-write слоя overlay FS и последующего возобновления.
- Настройка сети через tap-устройства, Linux bridge и iptables позволяет прозрачно пробрасывать порты для VNC/код-сервера без ручной возни с IP-таблицами.

## 4. Упомянутые инструменты, фреймворки, компании, продукты и подходы
- **Arachus**: открытая self-hosted песочница на microVM.
- **Cloud Hypervisor**: выбранный VMM (Rust, горячее добавление/удаление памяти, поддержка снэпшотов).
- **CrossVM/Firecracker**: родоначальники microVM; Firecracker — основа AWS Lambda.
- **QEMU, GVisor**: альтернативы с разным балансом безопасности и производительности.
- **Контейнеры** (namespaces, cgroups), **mini jail**.
- **Overlay FS, tap device, Linux bridge, iptables, btrfs**.
- **Docker**-tooling для кастомизации образа.
- **MCP server, Claude Desktop, Manas AI, OpenAI Canvas**.

## 5. Что полезно разработчику
- Код инициализации overlay FS внутри гостевой виртуалки для защиты базового образа.
- Функции настройки bridge, tap и правил iptables для проброса портов.
- Схема снэпшотирования на Go: пауза VM → копия read-write диска → дамп гостевой памяти → возобновление.
- Открытое API (REST, Python SDK, MCP) позволяет генерировать клиенты на любом языке.

## 6. Что можно применить в агентных продуктах
- Безопасное исполнение сгенерированного ИИ кода для мультитенантных сценариев.
- Откаты через снапшоты для надёжного выполнения длинных цепочек задач.
- Полноценное Linux-окружение для кодинг-агентов: установка любых пакетов, запуск серверов, отладка процессов.
- GUI-доступ через встроенный VNC для задач Computer Use.
